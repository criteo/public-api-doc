To provide our developer community with a reliable and stable API experience, we are announcing the enforcement of our rate limit policy.

Rate limits for the Criteo Marketing API are implemented at the app level and allow for a maximum of 100 requests per minute. Exceeding this limit will result in a 429 HTTP response, with additional information in the header.

We strongly encourage using throttling mechanisms that allow spreading the number of calls in a larger timeframe.

See the [rate limits](/marketing-solutions/docs/rate-limits-1) page for full details.
