## About this document

Complete this document with Criteo before implementation to validate your proposed selling model and architecture. It serves as a blueprint for a successful MPO API integration.

It helps to:

- Confirm whether your proposed requirements are supported by Criteo MPO.
- Align requirements with Criteo best practices.
- Recommend a more stable and scalable operating model where needed.

<Note>
  This is a reusable master template for multiple markets and advertisers. Select the Activation Model first, then define Campaign, Budget, Tracking, and Reporting accordingly.
</Note>

## How to use this template

Fill in the placeholders in each section to produce a working PRD draft. Follow the recommended sequence:

1. Activation Model \+ Executive Summary \+ Entity Mapping
2. Common Core Requirements (Campaign, Budget, Tracking, Reporting)
3. Scenario-Specific Requirements
4. Open Issues \+ MVP finalization

## 9. Criteo review checklist

**Activation model**

- Is it clearly defined whether the model is Bundle or Standalone?
- Does the selected model match the actual operating structure?

**Entity mapping**

- Is the customer structure clearly mapped to the MPO structure?
- Is there a clear foundation for Budget, Tracking, and Reporting design?

**Campaign**

- Are ownership and responsibilities for creation, activation, pause, and end-of-life clearly defined?
- Are lifecycle assumptions realistic?

**Budget**

- Is the budget type clearly defined?
- Is shared budget usage clearly documented?
- Are timezone and refresh rules defined?
- Is there an overspend protection method?

**Tracking**

- Are catalog field requirements sufficiently defined?
- Are seller identification rules clear?
- Are tracking parameter requirements specific enough?
- Is there a clear distinction between feed-based and runtime-based values?

**Reporting**

- Is the reporting level clearly defined?
- Is there a metrics dictionary?
- Are attribution differences documented?
- Are latency expectations clearly described?
