# Beaconing

<table>
  <thead>
    <tr>
      <th>Level</th>
      <th>Name</th>
      <th>Response field</th>
      <th>Explanation</th>
      <th>Mandatory</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Format</td>
      <td>Impression</td>
      <td><code>onLoadBeacon</code></td>
      <td>URL to be called when the whole placement is loaded onto the page. For display ads, the placement is considered "loaded" when the entire unit is rendered.<br />This beacon should only be fired once.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Format</td>
      <td>View</td>
      <td><code>onViewBeacon</code></td>
      <td>URL to be called when the placement is viewed according to IAB's viewable ad impression measurement guidelines. At least 50% of the unit must be in the user's viewport for at least 1 uninterrupted second.<br />This beacon should only be fired once.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Format</td>
      <td>Click</td>
      <td><code>onClickBeacon</code></td>
      <td>URL to be called when the user clicks on the placement itself.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Format</td>
      <td>File download</td>
      <td><code>onFileClickBeacon</code></td>
      <td>URL to be called when a click is made by the user on a CTA button, or from the creative image of the format, which leads to a file download.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product</td>
      <td>Impression</td>
      <td><code>onLoadBeacon</code></td>
      <td>URL to be called when the individual product tile is loaded onto the page. If leveraging <a href="https://developers.criteo.com/retail-media-delivery/docs/universal-beacons">universal beacons</a> , this field will be blank.<br />This beacon should only be fired once.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product</td>
      <td>View</td>
      <td><code>onViewBeacon</code></td>
      <td>URL to be called when the individual product tile is viewed according to IAB's viewable ad impression measurement guidelines. At least 50% of the placement must be in the user's viewport for at least 1 uninterrupted second.<br />This beacon should only be fired once per product.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product</td>
      <td>Click</td>
      <td><code>productPage</code><br />and/or<br /><code>onClickBeacon</code></td>
      <td><strong>productPage</strong><br />This element is a URL that will send an <code>OnClickBeacon</code> beacon to Criteo and then redirect the shopper to the item's PDP link (pulled from the product feed).<br />It is possible to use this as the <code>href</code> value for the sponsored product as a way to track clicks instead of calling the <code>OnClickBeacon</code>URL.<br /><strong>onClickBeacon</strong><br />URL to be called when the individual product tile is clicked. This refers to &#42;any&#42;<br />click interaction with the product tile outside of adding to cart or adding to wishlist (see next two rows).<br />This beacon should fire on all of the range of clicks; left click, middle click, right click to open in a new tab and keyboard shortcuts that simulate a click.<br /><strong>Both</strong><br />This beacon should be fired on every click event. Criteo will deduplicate clicks for billing purposes internally.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product</td>
      <td>Add-to-cart</td>
      <td><code>onBasketChangeBeacon</code></td>
      <td>URL to be called when a product is added to or removed from the basket/cart. You can add the quantity of products as a query string parameter. If no <code>quantity</code> parameter is provided, it defaults to <code>1</code>.<br />This beacon should be fired on &#42;every&#42; add-to-cart click event. Criteo will deduplicate clicks for billing purposes internally.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product</td>
      <td>Add-to-wishlist</td>
      <td><code>onWishlistBeacon</code></td>
      <td>URL to be called when the product is added to the wishlist/favorites.<br />This beacon should be fired on &#42;every&#42; add-to-wishlist click event. Criteo will deduplicate clicks for billing purposes internally.</td>
      <td>Yes</td>
    </tr>
  </tbody>
</table>

# Ad design

<table>
  <thead>
    <tr>
      <th>Check</th>
      <th>Explanation</th>
      <th>Mandatory</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Breakpoint adherence</td>
      <td><ul><li><strong>1.</strong> Ensure the breakpoints for the banners are correct.</li><li><strong>2.</strong> For the desktop version, verify the image cropping is as specified.</li><li><strong>3.</strong> Ensure the banner switches to its mobile version at the appropriate breakpoint.</li><li><strong>4.</strong> Use mock-ups to verify the correct spacing around the banner.</li><li><strong>5.</strong> Ensure the "sponsored" badge is displayed correctly for every breakpoint, as shown in the mock-ups provided by Criteo.</li></ul></td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product button formatting</td>
      <td>Ensure that the button's shape, font, font size, and position are aligned with the provided mock-ups.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product button text</td>
      <td>Ensure the name can fit on two lines and supports at least 20 characters of text</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Redirection button formatting</td>
      <td>Ensure that the button's shape, font, font size, and position are aligned with the provided mock-ups.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Redirection button text</td>
      <td>Ensure that the button text supports up to 20 characters in length.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Mock-up adherence</td>
      <td>The design of the mock-ups should be followed, ensuring the following:<br /><ul><li><strong>Background image ratio:</strong> The image resizes proportionally on mobile, maintaining the correct ratio.</li><li><strong>Backgrounds per product:</strong> Up to 5 different backgrounds per product are displayed.</li><li><strong>Image cropping:</strong> The image is cropped correctly for different screen resolutions, keeping the safe zone visible.</li><li><strong>CSS colors:</strong> Extra CSS colors are displayed correctly.</li><li><strong>Branding and product zones:</strong> The ratio between the branding zone and the product zone follows the mock-ups.</li><li><strong>Product information:</strong> The layout, price, rating, text orientation, and text centering match the mock-ups.</li><li><strong>Title:</strong> The font style matches the mock-ups.</li><li><strong>Text:</strong> Text placement, font family, font size, and colors match the mock-ups.</li><li><strong>Contour:</strong> A contour around the solution follows the mock-up design.</li><li><strong>Sponsored text:</strong> The text placement, font family, font size, and color match the mock-ups</li></ul></td>
      <td>Yes</td>
    </tr>
  </tbody>
</table>
