비디오 광고 구현을 위해서는 몇 가지 주요 단계를 고려해야 합니다.

1. VAST tag 요청 및 구문 분석,
2. DOM에 주입할 플레이어의 HTML 구조 생성,
3. 비디오 재생 중 적절한 시간에 beacons 실행,
4. 추가 검증 스크립트(OMID) 통합.

크리테오는 **VAST 4.2**를 지원합니다(IAB 사양은 [여기](https://iabtechlab.com/wp-content/uploads/2019/06/VAST_4.2_final_june26.pdf)에서 확인할 수 있습니다).

***

## 1단계: VAST 태그 요청 및 구문 분석

비디오 플레이어에 대한 데이터를 얻으려면 먼저 크리테오 API 광고 응답에서 VAST 정보를 가져옵니다.

다음은 VAST를 요청하고(필요한 경우) 구문 분석하는 방법의 예입니다.

```javascript
const response = await fetch('VAST TAG URL');  
const DOMXML = await response.text();  
const xmlParser = new window.DOMParser();  
const data = xmlParser.parseFromString(DOMXML, 'text/xml');
```

\<\<: VAST 태그를 구문 분석하면, 다음을 포함하여 이 문서에 설명된 필요한 비콘을 추출할 수 있습니다.

<Note>
  **VAST 구문 분석 라이브러리**

  일부 고객은 VAST 구문 분석 라이브러리를 사용하여 필요한 모든 정보를 추출하는 쪽을 선택합니다. `DOMParser`의 사용을 원치 않는 경우 대체 라이브러리의 예는 다음과 같습니다.

  * [VAST client JS](https://github.com/dailymotion/vast-client-js)
  * [JQuery](https://github.com/jquery/jquery)
</Note>

## 2단계: 비디오 플레이어 HTML 구조 생성

크리테오의 비디오 플레이어는 비디오 HTML 태그를 중심으로 구축되었습니다. 이 태그는 `HTMLMediaElement`로부터 상속받으며 `play()` 및 `pause()`와 같은 메소드와 `volume`, `duration`, `currentTime`과 같은 속성을 제공합니다.

이러한 속성은 플레이어를 음소거하거나 음소거 해제하고 재생 진행 상황을 추적하는 데 도움이 됩니다.

<Note>
  **[HTML 태그](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/video), [HTMLMediaElement](https://developer.mozilla.org/en-US/docs/Web/API/HTMLMediaElement) 및 [비디오 컨트롤](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/video#controls)에 대해 자세히 알아보려면 Mozilla 개발자 설명서를 참조하세요.**
</Note>

이러한 기능을 관리하기 위해 전체 광고 게재위치를 둘러싸도록 `<div>`로 묶고 재생/일시 중지 및 음소거/음소거 해제, 자막 버튼에 대한 추가 HTML을 통합합니다.

재생, 일시 중지, 볼륨 제어를 처리하기 위해 이러한 버튼에 클릭 리스너가 추가되었습니다. `<video>` 태그에는 기본 컨트롤 속성이 포함되어 있지만, 이는 탐색을 허용하기 때문에 사용하지 않습니다.

또한, 현지 규정에 따라 필수 공지사항의 경우 **콜아웃**이나 **팝업**이 필요할 수 있습니다.

## 3단계: 비디오 재생 중 비콘 실행

<Note>
  **브라우저 내에서 비콘을 실행하는 권장 방법에 대해 자세히 알아보려면 [Mozilla 개발자 문서의 다음 페이지](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/sendBeacon)를 참조하세요. 다른 방법(아래 설명)에서는 100%의 데이터를 전송하는 데 문제가 발생했습니다.**
</Note>

실행해야 할 첫 번째 비콘은 **노출 비콘**입니다.

이에 대한 트리거를 설정하려면 `<video>` 요소에 `can play` 이벤트에 대한 이벤트 리스너를 추가합니다. 이 이벤트는 브라우저가 비디오가 재생할 준비가 되었다고 판단하고 페이지에 그려지면 뷰포트에서의 위치와 관계없이 즉시 전송됩니다.

<Warning>
  정확한 비콘 추적을 보장하기 위해 **재생 비콘**(시작 및 사분위수 비콘)\*\*은 비디오를 처음 재생할 때 한 번만 방출되어야 합니다. 중복을 방지하기 위해 비디오가 방출된 각 비콘에 대한 상태를 유지하는 것이 중요합니다.

  하지만 **상호작용 비콘** (클릭, 음소거, 음소거 해제, 일시 중지, 재개 등)은 **해당 사용자 동작이 발생할 때마다 방출되어야 합니다.**
</Warning>

start, `firstQuartile`, `midpoint`, `thirdQuartile` 비콘은 `<video>` 태그의 `timeupdate 이벤트`에서 리스너가 트리거하는 함수에서 관리됩니다.

이 재생 핸들러 함수 내부에서, 먼저 비디오 재생이 첫 번째 루프에 있는지 확인합니다. 그렇다면 `currentTime` 속성을 duration 속성으로 나누어서 시청된 비디오의 백분율을 계산한 다음, 결과에 100을 곱하여 이 값을 0\~100 범위로 정규화합니다.

0이 아닌 백분율을 얻고 첫 번째 루프에 진입하면 `start` 비콘을 삭제합니다. 25%가 조회되었고 여전히 첫 번째 루프 내에 있는 경우 `firstQuartile` 비콘을 삭제하고, 50%와 75%에도 동일하게 적용합니다.

이 핸들러는 초당 여러 번 호출되므로([`timeupdate` 사양 참조](https://developer.mozilla.org/en-US/docs/Web/API/HTMLMediaElement/timeupdate_event)), 다음과 같이 핸들러를 빌드할 수 있습니다.

```javascript expandable
// 초기화 과정 중
this.videoPlayer.addEventListener('timeupdate', this.handlePlaybackEvents.bind(this));

function handlePlaybackEvents() {
        const percent = (this.videoPlayer.currentTime / this.videoPlayer.duration) * 100;
// 첫 번째 루프 동안만 비콘 실행
        if (this.currentLoop === 1 ) {
            if (percent >= 0.01 && !this.adStarted.sent) {
// 마크 전송
              this.adStarted.sent = true;
// start 비콘 실행
              const adStartedBeacon = new URL(this.adStarted.beacon);
              adStartedBeacon.drop();
// OMID로 이벤트 전달(문서의 다음 섹션과 관련)
              if (this.mediaEvents) {
                  this.mediaEvents.start(this.videoPlayer.duration, this.videoPlayer.muted ? 0 : this.videoPlayer.volume);
              }
            }

            if (percent >= 25 && !this.adFirstQuartile.sent) {
// 마크 전송
              this.adFirstQuartile.sent = true;
// firstQuartile 비콘 실행
              const adFirstQuartileBeacon = new URL(this.adFirstQuartile.beacon);
              adFirstQuartileBeacon.drop();
// OMID로 이벤트 전달(문서의 다음 섹션과 관련)
              if (this.mediaEvents) {
                  this.mediaEvents.firstQuartile();
              }
            }

            if (percent >= 50 && !this.adMidpoint.sent) {
// 마크 전송
              this.adMidpoint.sent = true;
// midPoint 비콘 실행
              const adMidpointBeacon = new URL(this.adMidpoint.beacon);
              adMidpointBeacon.drop();
// OMID로 이벤트 전달(문서의 다음 섹션과 관련)
              if (this.mediaEvents) {
                  this.mediaEvents.midpoint();
              }
          }

            if (percent >= 75 && !this.adThirdQuartile.sent) {
// 마크 전송
              this.adThirdQuartile.sent = true;
// thirdQuartile 비콘 실행
              const adThirdQuartileBeacon = new URL(this.adThirdQuartile.beacon);
              adThirdQuartileBeacon.drop();
// OMID로 이벤트 전달(문서의 다음 섹션과 관련)
              if (this.mediaEvents) {
                  this.mediaEvents.thirdQuartile();
              }
          }
        }
```

완료 비콘을 처리하려면 비디오 플레이어에서 실행된 [종료 이벤트](https://developer.mozilla.org/en-US/docs/Web/API/HTMLMediaElement/ended_event)를 수신합니다.

이 이벤트를 수신하면 비디오가 첫 번째 루프에 있는지 확인하고, 그렇다면 완료 비콘을 삭제합니다. 그런 다음 더 이상 필요하지 않으므로 `timeupdate 이벤트` 리스너와 관련 재생 핸들러를 제거합니다.

그 후 디버깅 및 로깅 목적으로 내부 루프 수를 증가시키고 비디오에서 `play()`를 호출하여 루핑 효과를 만듭니다. 이후의 "ended" 이벤트는 루프 카운트만 증가시키며 비디오를 다시 시작합니다.

```javascript
// 초기화 과정 중
this.videoPlayer.addEventListener('ended', this.handleLoop.bind(this));

handleLoop() {
    if (this.currentLoop === 1) {
        Logger.info('Sent video ad completed beacon');
// 완료 사분위수 비콘 방출
        const adCompletedBeacon = new URL(this.adCompleted.beacon);
        adCompletedBeacon.drop();
// OMID로 이벤트 전달(문서의 다음 섹션과 관련)
        if (this.mediaEvents) {
            this.mediaEvents.complete();
        }
    }
// 비디오 진행에 대한 모니터링 중지
    this.videoPlayer.removeEventListener('timeupdate', this.handlePlaybackEvents);
// 루프를 증가 & 비디오 재시작
    this.currentLoop++;
    this.videoPlayer.play();
}

```

모든 재생 비콘은 플레이어가 재생 중일 때만 실행됩니다.

<Warning>
  IAB 표준에 따라 광고가 표시되도록 하려면 **비디오 플레이어의 최소 50%가viewport에 있어야 합니다.** 임계값을 0.5로 설정한 [IntersectionObserver](https://developer.mozilla.org/en-US/docs/Web/API/IntersectionObserver)를 사용하여 비디오 플레이어에서 "50% 표시” 여부를 관찰합니다.
</Warning>

```javascript
handleVideoPlayPauseObserver(entries, observer) {
    for (let i = 0; i < entries.length; i++) {
        let entry = entries[i];
        let isVisible = entry.isIntersecting;
        if (isVisible) {
            this.playVideo();
        } else if (!isVisible && this.isPlaying) {
            this.pauseVideo();
        }
    }
}

startVideoAdObserver() {
    try {
// 비디오가 뷰포트에서 50% 이상이 아닌 경우 비디오 재생
        const observer = new IntersectionObserver(this.handleVideoPlayPauseObserver.bind(this), {threshold: 0.5});
        observer.observe(this.videoPlayerContainer);
    } catch (error) {
        throw new Error(`Unable to start video ad or observer. Error:  ${error}`);
    }
}
// 플레이어가 생성 및 초기화된 후
this.player.startVideoAdObserver();
```

마지막으로 `click` 이벤트에 대한 이벤트 리스너를 `<video>` 태그에 연결하여 `ClickTracking` 비콘을 처리합니다. `handler` 함수에서 비콘을 삭제하고 VAST tag에서 `ClickThrough` URL을 열도록 브라우저에 지시합니다(리디렉션 URL이 제공된 경우에만 해당).

```javascript
clickThroughHandler() {
// 클릭 비콘 방출
    if (this.clickThrough.beacon) {
        Logger.info('Sent video ad Click Through beacon');
        const clickThroughBeacon = new URL(this.clickThrough.beacon);
        clickThroughBeacon.drop();
    }
// 사용자를 클릭스루 URL로 리디렉션
    window.open(this.clickThroughURL, '_self');
}

handlePlayPause() {
    if (this.isPlaying) {
// 비디오를 일시 중지하고 일시 중지 비콘 방출
        this.pauseVideo();
    } else {
// 비디오 재생을 시작하고, 재개 비콘 방출
        this.playVideo();
    }
}

handleClickThrough() {
// ClickThroughURL이 없는 경우, 비디오 플레이어를 클릭하면 비디오를 재생하거나 일시 중지
    if (this.clickThroughURL) {
        this.videoPlayer.addEventListener('click', this.clickThroughHandler.bind(this));
    } else {
        this.videoPlayer.addEventListener('click', this.handlePlayPause.bind(this));
    }
}
```

## 4단계: OMID 호환 스크립트를 사용한 추가 검증(비디오 뷰)

클라이언트는 플레이어가 추가 스크립트를 실행할 수 있도록 OMID을 지원해야 하며, 이를 통해 크리테오가 비디오viewability를 추적할 수 있습니다. 이는 크리테오가 VAST XML 내에서 자체적인 검증 스크립트를 전송하여 비디오의 뷰어빌리티를 추적할 수 있도록 합니다.

<Info>
  **OMID 소개**

  OMID([오픈 측정 인터페이스 정의)](https://iabtechlab.com/standards/open-measurement-sdk/)는 IAB에서 제공하는 표준화된 기술로, 다양한 플랫폼과 디바이스에서 디지털 광고, 특히 비디오 광고의 뷰어빌리티를 측정하고 검증합니다.

  OMID는 업계 표준을 준수하여 조회 가능한 노출 수, 조회 가능한 비디오 재생 수 등 뷰어빌리티 지표를 측정하기 위한 일관된 프레임워크를 제공합니다. OMID를 구현하면 광고 성과와 사용자 참여에 대한 정확한 인사이트를 얻을 수 있으며, 모든 유통사에서 동일한 방식으로 비디오 뷰어빌리티를 추적하여 네트워크 전반에서 일관성을 보장할 수 있습니다.

  이 문서에서 OM SDK는 **웹 비디오용 OM SDKJS**를 의미합니다.
</Info>

[VAST 사양](https://www.iab.com/wp-content/uploads/2016/04/VAST4.0_Updated_April_2016.pdf)의 섹션 3.16은 이러한 검증이 있는 선택적 `AdVerifications` 요소를 설명합니다. `AdVerifications` 내부에 중첩된 각 `Verification` 요소는 검증에 필요한 리소스를 자세히 설명합니다.

<Note>
  **OM SDK**

  이러한 리소스를 처리하기 위해, 크리테오는 IAB에서 제공하는 OM SDK에 이를 전달합니다. 공식 OM OM SDK 구현 가이드는 [여기](https://interactiveadvertisingbureau.github.io/Open-Measurement-SDKJS)에서 확인할 수 있습니다. 이 문서에서는 뷰어빌리티 통합을 위한 각 단계에 필요한 코드 스니펫을 제공합니다.
</Note>

그 과정은 크게 다음과 같은 단계로 나뉩니다.

1. OMID 서비스 스크립트 및 세션 클라이언트를 로드
2. 필요한 세션 클라이언트 클래스 추출
3. `VerificationParameters` 정보를 세션 클라이언트의 광고 `Context`에 전달
4. 세션 시작
5. 비디오 및 광고 관련 이벤트가 발생하면 OM SDK로 `adEvents` 및 `mediaEvents` 이벤트 트리거

<Warning>
  [IAB 구현 가이드](https://interactiveadvertisingbureau.github.io/Open-Measurement-SDKJS/)는 크리테오의 비디오 플레이어 사양(버퍼링 이벤트 지원, 트래킹 이벤트 1회 방출, 재생/일시 중지 이벤트가 OM SDK에 알리지 않음 등)을 모두 다루기에는 충분하지 않습니다. 이러한 이유로 [IAB 문서](https://interactiveadvertisingbureau.github.io/Open-Measurement-SDKJS/validation.html)와 해당 문서에서 제공하는 [검증 스크립트](https://github.com/InteractiveAdvertisingBureau/Open-Measurement-JSClients/blob/master/creatives/video/omid_video_creative_session.js)를 사용하여 통합을 확인하는 것이 **적극** 권장됩니다.\*\*
</Warning>

<Note>
  **이러한 파일은 자체적으로 호스팅할 수도 있고 사본을 호스팅하는 크리테오의 CDN에서 가져올 수도 있습니다. 다음 URL에서 파일을 찾을 수 있습니다.**

  [https://static.criteo.net/banners/js/omidjs/stable/omid-session-client-v1.js](https://static.criteo.net/banners/js/omidjs/stable/omid-session-client-v1.js)

  [https://static.criteo.net/banners/js/omidjs/stable/omweb-v1.js](https://static.criteo.net/banners/js/omidjs/stable/omweb-v1.js)

  Omid-session-client-v1.js는 통합 파트너가 JavaScript 계층에서 광고 세션 활동을 수행하는 데 사용됩니다.

  Omweb-v1.js는 웹 컨텍스트를 위한 OMID JS 라이브러리입니다.
</Note>

### 3. `&lt;VerificationParameters&gt;` 정보를 세션 클라이언트의 광고 `Context`에 전달

광고 세션 초기화 전에 OM SDK JS는 크리테오 통합과 관련된 일부 파라미터를 지정해야 합니다.

```javascript
const CONTENT_URL = window.location.href;
const PARTNER_NAME = 'criteo';
const PARTNER_VERSION = Object.keys(OmidSessionClient)[0];

const partner = new Partner(PARTNER_NAME, PARTNER_VERSION);
```

이 단계는 크리테오의 검증 스크립트를 참조하여 `AdSession`이 시작될 때 자동으로 로드될 수 있도록 합니다.

```javascript expandable
// VAST 정의에 있는 &lt;AdVerifications&gt;으로부터
const vastVerifications = [{
    javaScriptResource: 'https://static.criteo.net/banners/js/omidjs/stable/omid-validation-verification-script-for-retail-media.js',
    vendor: 'criteo.com-omid',
// VAST 정의에 있는 &lt;VerificationParameters&gt;로부터
    verificationParameters: JSON.stringify({
        "beacons": {
            "omidTrackView": `${bbtUrlomidTrackView}`,
            "start": `${bbtUrlStart}`,
            "firstQuartile": `${bbtUrlFirstQuartile}`,
            "midpoint": `${bbtUrlMidpoint}`,
            "thirdQuartile": `${bbtUrlThirdQuartile}`,
            "complete": `${bbtUrlComplete}`,
            "twoSecondsFiftyPercentView": `${bbtUrlTwoSecondsFiftyPercentView}`
        }
    }),
}];

const resources = vastVerifications.map((verification) => {
    const scriptUrl = verification.javaScriptResource;
// 액세스 모드 값:
// https://interactiveadvertisingbureau.github.io/Open-Measurement-SDKJS/accessmodes.html
    const accessMode = 'full';
    return new VerificationScriptResource(scriptUrl, verification.vendor, verification.verificationParameters, accessMode);
});

const OMSDK_SERVICE_WINDOW = window.top;
const VIDEO_ELEMENT = document.getElementById('video-creative-element');
// 크리테오의 검증 스크립트에 대한 참조 설정
const context = new Context(partner, resources, CONTENT_URL);
context.setVideoElement(VIDEO_ELEMENT);
context.setServiceWindow(OMSDK_SERVICE_WINDOW);
```

### 4. 세션 시작

광고 세션이 시작되면 크리테오의 검증 스크립트가 `Context` 정보를 사용하여 자동으로 로드됩니다([IAB 구현 가이드의 3단계](https://interactiveadvertisingbureau.github.io/Open-Measurement-SDKJS/#3-create-the-context)).

이 시점에서 크리테오의 검증 스크립트는 OM SDK에 대해 트리거되는 모든 `AdEvents` 및 `MediaEvents`를 관찰하기 시작합니다.

```javascript expandable
const adSession = new AdSession(context);
adSession.setCreativeType('video');
adSession.setImpressionType('beginToRender');

if (!adSession.isSupported()) {
    const errorMessage = "Ad session is not supported."
    Logger.error(errorMessage)
    throw errorMessage
}

const adEvents = new AdEvents(adSession);
const mediaEvents = new MediaEvents(adSession);
// AdSession 시작
adSession.start();
// 세션이 시작되었음을 OM SDK JS에 알리고 수명 주기 지원(오류/완료)
adSession.registerSessionObserver((event) => {
    const {adSessionId, timestamp, type, data} = event;
    switch (type) {
        case 'sessionStart':
            state.omidState.sessionActive = true;

            const isSkippable = false;
            const skipOffset = 0;
            const vastProperties = new VastProperties(isSkippable, skipOffset, /* isAutoPlay= */ true, /* position= */ 'standalone');
// 로드 이벤트
            adEvents.loaded(vastProperties);
            adEvents.impressionOccurred();
            break;
        case 'sessionError':
            Logger.error("An OMID session error occured", event);
            throw event;
        case 'sessionFinish':
// 클린 업
            Logger.log("The OMID session finished", event);
            break;
    }
});
```

### 5. 오픈 측정 이벤트 트리거

<Note>
  **사용 가능한 `AdEvents` 및 `MediaEvents`의 전체 목록은 [IAB의 Github](https://github.com/InteractiveAdvertisingBureau/)에서 확인할 수 있습니다.**
</Note>

이러한 호출을 통해 크리테오의 검증 스크립트는 고급 뷰어빌리티 이벤트를 계산할 수 있습니다.

* 크리테오는 사분위수 이벤트를 추적할 수 있습니다(비디오가 viewport의 100% 부분인 경우).
* 고급 이벤트는 2초 연속으로 뷰포트에서 비디오의 50%가 재생되고 표시되는지 추적하기 위해 계산됩니다.

`AdEvents` 및 `MediaEvents`에 대한 모든 호출은 사분위수 및 디스플레이 이벤트에 대한 기존 추적 비콘과 별도로 트리거되어야 합니다.

또한 비디오 버퍼링 이벤트는 IAB 코드 샘플에 설명된 대로 트리거되어야 합니다.

```javascript
this.isBuffering = false;

this.videoPlayer.addEventListener("waiting", () => {
    this.isBuffering = true;
    this.mediaEvents.bufferStart();
});

this.videoPlayer.addEventListener("timeupdate", () => {
    if (this.isBuffering) {
        this.isBuffering = false;
        this.mediaEvents.bufferFinish();
    }
});
```

### 6. 여러 영상이 제시되는 경우 올바른 영상 선택

<Info>
  **타겟 규격**은 비디오가 배치될 HTML 요소의 너비와 높이입니다.
</Info>

VAST XML에 여러 개의 비디오가 표시되는 경우 `loadAd()`를 호출할 때 타겟 규격을 파라미터로 제공하는 것이 중요합니다. 다음과 같이 `clientWidth` 및 `clientHeight`를 사용하여 규격을 동적으로 가져올 수 있습니다.

```javascript
const targetContainer = document.querySelector('.targetContainerSelector');

const targetDimension = {
    width: targetContainer.clientWidth,
    height: targetContainer.clientHeight,
};

await videoAdPlayer.loadAd(targetDimension);
```

타겟 규격에 가장 일치하는 규격을 기준으로 올바른 비디오가 선택됩니다.

<Warning>
  유효한 타겟 규격이 제공되지 않거나 유효하지 않은 경우, 목록의 첫 번째 비디오가 자동으로 선택됩니다.
</Warning>

유효한 타겟 규격이 지정되면 라이브러리는 종횡비를 고려하고 비율 차이가 가장 작은 비디오를 선택합니다.

<Info>
  **예시**

  수직, 수평, 정사각형 비디오와 수직 타겟 규격이 제공되면 라이브러리는 수직 비디오를 선택합니다. 세로 2배, 가로 2배, 정사각형 1배의 비디오가 있고, 세로 타겟 규격이 있는 시나리오에서는 비율 차이가 가장 작은 두 개의 세로 비디오 중 하나가 선택됩니다.
</Info>
