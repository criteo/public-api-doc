# Beaconing

<table>
  <thead>
    <tr>
      <th>Level</th>
      <th>Title</th>
      <th>Response field</th>
      <th>Explanation</th>
      <th>Mandatory</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Placement</td>
      <td>Impression</td>
      <td><code>onLoadBeacon</code></td>
      <td>URL to be called when the whole placement is loaded onto the page. For sponsored products, the placement is considered "loaded" when at least one product is rendered.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Placement</td>
      <td>View</td>
      <td><code>onViewBeacon</code></td>
      <td>URL to be called when at least 50% of the first product is in the user's viewport for at least 1 uninterrupted second.<br />This beacon should only be fired once.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product</td>
      <td>Impression</td>
      <td><code>onLoadBeacon</code></td>
      <td>URL to be called when the individual product tile is loaded onto the page. If leveraging <a href="https://developers.criteo.com/retail-media-delivery/docs/universal-beacons">universal beacons</a>, this field will be blank.<br />This beacon should only be fired once.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product</td>
      <td>View</td>
      <td><code>onViewBeacon</code></td>
      <td>URL to be called when the individual product tile is viewed according to IAB's viewable ad impression measurement guidelines. At least 50% of the placement must be in the user's viewport for at least 1 uninterrupted second.<br />This beacon should only be fired once per product.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product</td>
      <td>Click</td>
      <td><code>productPage</code><br />and/or<br /><code>onClickBeacon</code></td>
      <td><strong>productPage</strong><br />This element is a URL that will send an <code>OnClickBeacon</code> beacon to Criteo and then redirect the shopper to the item's PDP link (pulled from the product feed).<br />It is possible to use this as the <code>href</code> value for the sponsored product as a way to track clicks instead of calling the <code>OnClickBeacon</code>URL.<br /><strong>onClickBeacon</strong><br />URL to be called when the individual product tile is clicked. This refers to &#42;any&#42;<br />click interaction with the product tile outside of adding to cart or adding to wishlist (see next two rows).<br />This beacon should fire on all the range of clicks; left click, middle click, right click to open in a new tab and keyboard shortcuts that simulate a click.<br /><strong>Both</strong><br />This label refers to cases where both <code>OnClick</code> and <code>ProductPage</code> beacons are triggered.<br /><ul><li>Important&#92;&#95;: These beacons are treated as separate events and are not deduplicated against each other.</li></ul><br />Deduplication is performed only when multiple beacons of the same type are received. Criteo will handle this deduplication internally for billing purposes.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product</td>
      <td>Add-to-cart</td>
      <td><code>onBasketChangeBeacon</code></td>
      <td>URL to be called when a product is added to or removed from the basket/cart. You can add the quantity of products as a query string parameter. If no <code>quantity</code> parameter is provided, it defaults to <code>1</code>.<br />This beacon should be fired on &#42;every&#42; add-to-cart click event. Criteo will deduplicate clicks for billing purposes internally.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Product</td>
      <td>Add-to-wishlist</td>
      <td><code>onWishlistBeacon</code></td>
      <td>URL to be called when the product is added to the wishlist/favorites.<br />This beacon should be fired on &#42;every&#42; add-to-wishlist click event. Criteo will deduplicate clicks for billing purposes internally.</td>
      <td>Yes</td>
    </tr>
  </tbody>
</table>

# Ad design

<table>
  <thead>
    <tr>
      <th>Check</th>
      <th>Explanation</th>
      <th>Mandatory</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Breakpoint adherence</td>
      <td>For each window size (breakpoint), ensure the sponsored products are positioned properly and that all "sponsored" badges are displayed correctly, as shown in the mock-ups provided by Criteo.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Mock-up adherence</td>
      <td>The design of the mock-ups should be followed as specified:<br /><ul><li><strong>Product information:</strong> Layout, price, rating, text orientation, and text centering.</li><li><strong>Navigation elements:</strong> Include any arrows or scrolls that are part of the product.</li><li><strong>Title:</strong> Font style should match the mock-ups.</li><li><strong>Text:</strong> Placement, font family, font size, and colors should match the mock-ups.</li><li><strong>Sponsored text:</strong> Placement, font family, font size, and color should match the mock-ups.</li><li><strong>Contour:</strong> Ensure there is a contour around the solution, following the mock-up design.</li></ul></td>
      <td>Yes</td>
    </tr>
  </tbody>
</table>

    <tr>
      <td>Filters leveraged in ad calls</td>
      <td><ul><li><strong>Format:</strong> Ensure the filters in the ad call match the product feed.</li><li><strong>Multiple values:</strong> Ensure filters with multiple values are passed correctly using the 'in' operator.</li><li><strong>Removal:</strong> Ensure that when filters are removed, updated ad calls are made to Criteo reflecting on site.</li></ul></td>
      <td>Yes, if filters have been implemented</td>
    </tr>
    <tr>
      <td>Leveraging regiondata</td>
      <td><ul><li><strong>Format:</strong> Ensure that the region IDs in the ad call match what is passed in the product feed</li><li><strong>Adherence:</strong> Ensure that Criteo is adhering properly to the region in our ad response</li></ul></td>
      <td>Yes, if regiondata has been implemented</td>
    </tr>
