## Filters

When a user has filtered a product listing page, the filter value(s) may be included in the Criteo API call to ensure a relevant response. Three specific filters are supported natively by Criteo, utilizing required data fields in the product feed. They are “brand”, “price”, and “rating”. All other filters can be set up by your Technical Account Manager via additional, optional fields in the feed.

A fully qualified filter consists of 3 properties: (filter-name,operator,value)

• filter-name: Name of the filter\
• operator: The applied operator\
• value: This is the value of the filter. It can be a list of values depending on your use case. For example, if you use the operator ‘in’, you can specify a comma delimited list of values.

Finally, the filter value must be URL encoded.

Examples:

1. This would set a price filter for skus that are ‘less than or equal to’ \$500.\
    filters=(price,le,C500)

2. This would set a color filter for skus whose color is ‘blue’.\
    filters=(color,eq,blue)

3. This would set a brand filter that would return skus from any of ‘sony’, ‘hp’, or ‘dell’ brands.\
    filters=(brand,in,sony,hp,dell)

## Sort

Criteo supports the following options for sorting.\
• featured\
• default\
• price\_desc\
• price\_asc\
• name\_desc\
• name\_asc\
• rating\_desc\
• rating\_asc
