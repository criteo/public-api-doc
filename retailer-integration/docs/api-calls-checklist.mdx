# Introduction

This page presents a breakdown of all the parameters needed for the API calls by common parameters (that can be used on all page calls) as well as by page type.

***

## Common parameters

<table>
  <thead>
    <tr>
      <th>Title</th>
      <th>Name</th>
      <th>To Check</th>
      <th>Mandatory</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>Account ID</code></td>
      <td><code>criteo-partner-id</code></td>
      <td>Your Criteo Account ID, will be provided by your technical contact<br />Ex: <code>criteo-partner-id=102302</code></td>
      <td>Yes</td>
    </tr>
    <tr>
      <td><code>Retailer Visitor ID</code></td>
      <td><code>retailer-visitor-id</code></td>
      <td>Unauthenticated User ID, consistent across sessions<br />Ex: <code>retailer-visitor-id=c9623-aaaa</code></td>
      <td>Yes</td>
    </tr>
    <tr>
      <td><code>Customer ID</code></td>
      <td><code>customer-id</code></td>
      <td>After logging in, whether there is a unique user ID<br />Can also log in with a different device or log-in with the same account on another browser/incognito.<br />Ex: <code>customer-id=c9621-bbbb</code></td>
      <td>Yes</td>
    </tr>
    <tr>
      <td><code>Email</code></td>
      <td><code>email</code></td>
      <td>Hash value of the user email address SHA256<br />Ex: <code>3786ae0aa4a0655c</code></td>
      <td>Only if discussed with your Criteo team</td>
    </tr>
    <tr>
      <td><code>Test call</code></td>
      <td><code>nolog</code></td>
      <td>To be used when the log of this call shouldn't be counted as a real page view. (used for our cached API response scenario)<br />Ex: <code>nolog=1</code><br />Can be used on preprod accounts.</td>
      <td>Only if discussed with your Criteo team</td>
    </tr>
    <tr>
      <td><code>Item white list</code></td>
      <td><code>item-whitelist</code></td>
      <td>A parameter to be used to return the specific products that are set in the call, only for sponsored products. To be used with the cached API response scenario or to use the recommendation engine.</td>
      <td>Only if discussed with your Criteo team</td>
    </tr>
    <tr>
      <td><code>Region ID</code></td>
      <td><code>regionId</code></td>
      <td>The region IDs that are sent are accurate to the regions IDs that are shared in the daily feed.<br />If multiple region IDs are to be sent, then they need to be separated by the pipe symbol.<br />Ex: <code>regionId=123-abc|456-def|789-ghi</code></td>
      <td>Only if discussed with your Criteo team</td>
    </tr>
  </tbody>
</table>

    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>viewCategory</code></td>
      <td><code>page-id</code></td>

    </tr>

      <td>Recommended</td>
    </tr>
    <tr>
      <td><code>viewCategory</code></td>
      <td><code>category</code></td>
      <td>The category path of the current category, either numerical or alphanumerical. The path provided in the tag needs to match "product&#95;type&#95;key" from the feed<br />Ex: <code>category=1001</code></td>
      <td>Yes</td>
    </tr>
    <tr>
      <td><code>viewCategory</code></td>
      <td><code>filters</code></td>
      <td>Used to restrict the response to products which have a certain attribute with the exact value in the Feed<br />Ex: <code>filters=(brand,eq,nike)</code></td>
      <td>Only if discussed with your Criteo team</td>
    </tr>
    <tr>
      <td><code>viewCategory</code></td>
      <td><code>list-size</code></td>
      <td>The total number of organic products that show on the page.<br />Ex: <code>list-size=24</code></td>
      <td>Recommended</td>
    </tr>
  </tbody>
</table>

    <tr>
      <td><code>addToCart</code></td>
      <td><code>page-id</code></td>
      <td>For the <code>page-id</code>, you must use the ID of the page where the <code>add-to-cart</code> event took place. For example, if the user added a SKU to the cart directly from the product tile on a Search results page, the <code>pageID</code> for this call would be:<br />EMEA: <code>viewSearchResultApi[Environment]</code><br />AMER: <code>viewSearchResult&#95;API&#95;[Environment]</code><br />Ex: <code>viewSearchResultApiDesktop</code></td>
      <td>Yes</td>
    </tr>
    <tr>
      <td><code>addToCart</code></td>

      <td>We are using the <code>addToCart</code> for the add to cart page</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td><code>addToCart</code></td>
      <td><code>item</code></td>
      <td>The ID of the product that was added to the basket, the SKU that has to match the ID provided in the feed.<br />Ex: <code>item=123</code></td>
      <td>Yes</td>
    </tr>
    <tr>
      <td><code>addToCart</code></td>
      <td><code>parent-item</code></td>
      <td>Parent Item of the products currently in the users' basket. This ID needs to match with the ID provided in the feed. We can only use this to update all the variants (child products)<br />Ex: <code>parent-item=123</code></td>
      <td>Only if discussed with your Criteo team</td>
    </tr>
    <tr>
      <td><code>addToCart</code></td>
      <td><code>price</code></td>
      <td>Array of products in the basket single unit price<br />Ex: <code>price=3.99</code></td>
      <td>Yes</td>
    </tr>
    <tr>
      <td><code>addToCart</code></td>
      <td><code>quantity</code></td>
      <td>Array of quantity of each product in the basket<br />Ex: <code>quantity=3</code></td>
      <td>Yes</td>
    </tr>
    <tr>
      <td><code>addToCart</code></td>
      <td><code>page-uid</code></td>
      <td>Unique ID present at the bottom of the API call of the current page.<br />Ex: <code>page-uid=545d9a70-f096-4568-b4b9-8f2f32a452d4</code></td>
      <td>Yes</td>
    </tr>
  </tbody>
