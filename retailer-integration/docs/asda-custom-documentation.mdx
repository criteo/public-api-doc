## Beacon Description

Beacons that are returned once per format

<table>
  <thead>
    <tr>
      <th>Beacon key</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>onLoadBeacon</td>
      <td>Indicates that the ad has been rendered (even if not viewable yet)</td>
    </tr>
    <tr>
      <td>onViewBeacon</td>
      <td>Indicates that the ad has been viewed (viewed = 50% of the ad in viewport for 1 second)</td>
    </tr>
    <tr>
      <td>OnFileClickBeacon</td>
      <td>Indicates that a click was made on the file option</td>
    </tr>
    <tr>
      <td>OnBundleBasketChangeBeacon</td>
      <td>add multiple products at ones</td>
    </tr>
  </tbody>
</table>

Beacons that are returned once per product:

<table>
  <thead>
    <tr>
      <th></th>
      <th></th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>onLoadBeacon</td>
      <td>Indicates that the product tile was rendered (even if not viewable yet)</td>
    </tr>
    <tr>
      <td>onViewBeacon</td>
      <td>Indicates that the ad has been viewed (viewed = 50% of the ad in viewport for 1 second)</td>
    </tr>
    <tr>
      <td>ProductPage</td>
      <td>Clicktracking that redirects to the product detail page<br />Should be fired every time for every single click<br />(only applicable for web environment)</td>
    </tr>
    <tr>
      <td>onClickBeacon</td>
      <td>Clicktracking without redirection.<br />Should be used for clicks in case ProductPage is not a viable option. (e.g. InApp environment, AddtoCart)<br />Should be fired every time for every single click</td>
    </tr>
    <tr>
      <td>onBasketChangeBeacon</td>
      <td>Added/removed quantity needs to be added as query string parameter (optional, defaults to 1)<br />Example: &#92;&amp;quantity=2 or &#92;&amp;quantity=-1</td>
    </tr>
    <tr>
      <td>onWishlistBeacon</td>
      <td>Indicates that a click was made on the Add to Wishlist button on the product tile</td>
    </tr>
  </tbody>
</table>
