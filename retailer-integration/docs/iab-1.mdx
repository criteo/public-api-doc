# Beaconing

<table>
  <thead>
    <tr>
      <th>Level</th>
      <th>Name</th>
      <th>Response field</th>
      <th>Explanation</th>
      <th>Mandatory</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Format</td>
      <td>Impression</td>
      <td><code>onLoadBeacon</code></td>
      <td>URL to be called when the whole placement is loaded onto the page. For display ads, the placement is considered "loaded" when the entire unit is rendered.<br />This beacon should only be fired once.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Format</td>
      <td>View</td>
      <td><code>onViewBeacon</code></td>
      <td>URL to be called when the placement is viewed according to IAB's viewable ad impression measurement guidelines. At least 50% of the unit must be in the user's viewport for at least 1 uninterrupted second.<br />This beacon should only be fired once.</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Format</td>
      <td>Click</td>
      <td><code>onClickBeacon</code></td>
      <td>URL to be called when the user clicks on the placement itself.</td>
      <td>Yes</td>
    </tr>
  </tbody>
</table>

# Ad design

<table>
  <thead>
    <tr>
      <th>Check</th>
      <th>Explanation</th>
      <th>Mandatory</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Breakpoint adherence</td>
      <td><ul><li><strong>1.</strong> Ensure the breakpoints for the banners are correct.</li><li><strong>2.</strong> For the desktop version, verify the image cropping is as specified.</li><li><strong>3.</strong> Ensure the banner switches to its mobile version at the appropriate breakpoint.</li><li><strong>4.</strong> Use mock-ups to verify the correct spacing around the banner.</li><li><strong>5.</strong> Ensure the "sponsored" badge is displayed correctly for every breakpoint, as shown in the mock-ups provided by Criteo.</li></ul></td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>Mock-up adherence</td>
      <td>The design of the mock-ups should be followed, ensuring the following:<br /><ul><li><strong>Background image:</strong> The image ratio is correct and resizes proportionally.</li><li><strong>Cropping:</strong> The image is cropped correctly for different screen resolutions, with the safe zone remaining visible.</li><li><strong>Product information:</strong> The layout, price, rating, text orientation, and text centering align with the mock-ups.</li><li><strong>Sponsored text: </strong>The text placement, font family, font size, and color should match the mock-ups.</li></ul></td>
      <td>Yes</td>
    </tr>
  </tbody>
</table>
