### 커머스 비디오 포맷 비콘 레벨

커머스 비디오 포맷의 경우, **비콘에는 3가지 레벨**이 있으며, 이는 각 비콘이 다음 중 하나에 연결됨을 의미합니다.

* **포맷/배치** 자체와 관련된 이벤트
* 포맷 내 **개별 SKU**와 관련된 이벤트(포맷에서 SKU를 사용할 수 있는 경우)
* **비디오**와 관련된 이벤트 (VAST 비디오 플레이어를 통해 발생)

아래 표에는 이 세 가지 레벨의 비콘 간의 차이점과 비콘의 이름 및 설명이 요약되어 있습니다.

<Warning>
  달리 명시되지 않는 한 비콘은 **페이지 로드당 한 번만** 실행되어야 합니다.
</Warning>

<Info>
  모든 비디오 트래킹 비콘(위에서 언급한 3번째 종류의 이벤트와 관련)은 VAST XML 파일을 로드한 후 **클라이언트 측 비디오 플레이어에 의해 직접 실행됩니다**.
</Info>

<Info>
  크리테오 비콘은 브라우저에서 직접 **클라이언트 측에서 트리거**되도록 하는 것이 좋습니다. 서버 측 비콘을 구현하려는 경우, 먼저 크리테오 기술팀에 문의해 주시기 바랍니다.
</Info>

## 포맷 레벨 비콘

<table>
  <thead>
    <tr>
      <th>비콘 이름</th>
      <th>비콘 설명</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>OnLoadBeacon</code></td>
      <td>포맷(해당되는 경우 비디오 + SKU)이 아직 볼 수 없더라도 페이지에 렌더링됨을 나타냅니다.</td>
    </tr>
    <tr>
      <td><code>OnViewBeacon</code></td>
      <td>사용자가 해당 포맷을 보고 있다는 것을 나타냅니다 (뷰어빌리티 = 1초 동안 50% 이상 보이는 경우).</td>
    </tr>
    <tr>
      <td><code>OnClickBeacon</code> (200)</td>
      <td>사용자가 비디오나 제품 타일이 아닌 클릭 가능한 요소(CTA 등)를 클릭했음을 나타냅니다.</td>
    </tr>
    <tr>

      <td>이 비콘을 무시하세요.</td>
    </tr>

      <td>이 비콘을 무시하세요.</td>
    </tr>

## SKU 레벨 비콘

<table>
  <thead>
    <tr>
      <th>비콘 이름</th>
      <th>비콘 설명</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>OnLoadBeacon</code></td>
      <td>(아직 볼 수 없더라도) 제품 타일이 페이지에 렌더링됨을 나타냅니다.</td>
    </tr>
    <tr>
      <td><code>OnViewBeacon</code></td>
      <td>사용자가 제품 타일을 보고 있다는 것을 나타냅니다 (뷰어빌리티 = 1초 동안 50% 이상 보이는 경우).</td>
    </tr>
    <tr>
      <td><code>OnClickBeacon</code> (200) 또는 <code>ProductPage</code> (302)</td>
      <td>사용자가 제품 페이지로 이동하는 요소(이미지, 텍스트)를 클릭할 때 트리거됩니다.</td>
    </tr>
    <tr>
      <td><code>OnBasketChangeBeacon</code></td>
      <td>사용자가 장바구니에 제품을 추가할 때 트리거됩니다.</td>
    </tr>
    <tr>
      <td><code>OnWishListBeacon</code></td>
      <td>사용자가 위시리스트에 제품을 추가할 때 트리거됩니다.</td>
    </tr>
  </tbody>
</table>

## VAST XML 레벨 비콘

<table>
  <thead>
    <tr>
      <th>비콘 이름</th>
      <th>설명</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>Impression</code></td>
      <td>(아직 볼 수 없더라도) 비디오가 페이지에 렌더링됨을 나타냅니다.</td>
    </tr>
    <tr>
      <td><code>Tracking event="start"</code></td>
      <td>비디오가 시작되었으며, 비디오가 사용자 뷰포트의 50%에 들어오면 트리거되어야 합니다.</td>
    </tr>
    <tr>
      <td><code>Tracking event="firstQuartile"</code></td>
      <td>비디오의 25%가 재생되면 트리거됩니다.</td>
    </tr>
    <tr>
      <td><code>Tracking event="midpoint"</code></td>
      <td>비디오의 50%가 재생되면 트리거됩니다.</td>
    </tr>
    <tr>
      <td><code>Tracking event="thirdQuartile"</code></td>
      <td>비디오의 75%가 재생되면 트리거됩니다.</td>
    </tr>
    <tr>
      <td><code>Tracking event="complete"</code></td>
      <td>비디오의 100%가 재생되면 트리거됩니다.</td>
    </tr>
    <tr>
      <td><code>ClickTracking</code></td>
      <td>사용자가 비디오를 클릭하여 VAST XML의 <code>&lt;ClickThrough id="Criteo"&gt;</code> 노드에 저장된 랜딩 페이지로 이동할 때 트리거됩니다.</td>
    </tr>
    <tr>
      <td><code>Tracking event="mute"</code></td>
      <td>사용자가 비디오 플레이어의 <code>음소거</code> 버튼을 클릭할 때 트리거됩니다.</td>
    </tr>
    <tr>
      <td><code>Tracking event="unmute"</code></td>
      <td>사용자가 비디오 플레이어의 <code>음소거 해제</code> 버튼을 클릭할 때 트리거됩니다.</td>
    </tr>
    <tr>
      <td><code>Tracking event="pause"</code></td>
      <td>사용자가 비디오 플레이어의 <code>일시 중지</code> 버튼을 클릭할 때 트리거됩니다.</td>
    </tr>
    <tr>
      <td><code>Tracking event="resume"</code></td>
      <td>사용자가 비디오 플레이어의 <code>재생</code> 버튼을 클릭할 때 트리거됩니다.</td>
    </tr>
  </tbody>
</table>
