## Version 1.2. (Q2 2025)
### New video format
A new format is now available: the **Branding Video Standout**. This format combines a branded video with a branded image and a CTA. Learn more about it [on the dedicated page](https://developers.criteo.com/retailer-integration/docs/branding-video-standout).

### Commerce Video is now in general availability
Commerce Video is not in beta anymore. You can find the dedicated documentation [here](https://developers.criteo.com/retailer-integration/docs/commerce-video-integration).

### Update of the feed parameter page
We have adjusted the formatting of the [feed parameter page](https://developers.criteo.com/retailer-integration/docs/product-feed-parameters) for better readability. The Product Importer API documentation will now be located in the Retail Media API documentation [here](https://developers.criteo.com/retail-media/v2025-preview/docs/product-importer-api) to avoid duplicates.

### New implemented examples
New implemented examplesYou can now find embedded examples of implemented ads for [Flagship](https://developers.criteo.com/retailer-integration/v1.2/docs/flagship#implementation-example), [Butterfly](https://developers.criteo.com/retailer-integration/v1.2/docs/butterfly#implementation-example), [Branded Header](https://developers.criteo.com/retailer-integration/v1.2/docs/branded-header#implementation-example), [Interactive Header](https://developers.criteo.com/retailer-integration/v1.2/docs/interactive-header#implementation-example), [Digital Shelf Talker](https://developers.criteo.com/retailer-integration/v1.2/docs/digital-shelf-talker#implementation-example) at the end of each guide in the Ad rendering section.

## Version 1.1. (Q1 2025)
### New Product Importer API guides
The Product Importer API enables users to upload, update, and manage product datasets on the Criteo platform. We have added an entire new section dedicated to the Product Importer API in our "**Feed & Product Data**" section, where you can now find:

- [The list of product dataset parameters](https://developers.criteo.com/retailer-integration/v1.1/docs/dataset-parameters) (for both *onsite* and *offsite*),
- [Product dataset examples](https://developers.criteo.com/retailer-integration/v1.1/docs/product-dataset-examples).

### New Google Ad Manager section
The GAM or Google Ad Manager is a comprehensive ad management platform that enables publishers to manage, sell, and optimize both direct and programmatic ad inventory across web, mobile and app environments. Retailers already using GAM can now also integrate with Retail Media by adding custom creatives to their Line items. Learn how by visiting the new GAM guide available [here](https://developers.criteo.com/retailer-integration/v1.1/docs/gam-overview).


### Legacy vs. universal beacons
We have worked on [clarifying the distinction between "legacy" and "universal/standard" beacons](https://developers.criteo.com/retailer-integration/v1.1/docs/legacy-universal-beacons) in the Ad Tracking section. We strongly encourage all new integrations to be done using universal beacons. We also provide in [this section](https://developers.criteo.com/retailer-integration/v1.1/docs/legacy-universal-beacons#migrating-from-legacy-to-universal-beacons) a guide to migrate from legacy to universal beacons.

### Integration best practices
We have incorporated integration best practices guides into each of the main sections of the documentation ([Product feed](https://developers.criteo.com/retailer-integration/v1.1/docs/feed-best-practices), [Requesting Ads](https://developers.criteo.com/retailer-integration/v1.1/docs/requesting-ads-best-practices), [Ad rendering](https://developers.criteo.com/retailer-integration/v1.1/docs/ad-rendering-best-practices), and [Ad Tracking](https://developers.criteo.com/retailer-integration/v1.1/docs/ad-tracking-best-practices)) to better accommodate different Retailer technical teams working on separate sections of the integration.

### OneTag Offsite for hybrid apps
We have added a complementary page to the main [OneTag for Retail Media Offsite page](https://developers.criteo.com/retailer-integration/v1.1/docs/onetag-for-offsite), for hybrid apps. You will find on this page the information to be able to (1) retrieve the advertising ID from the OS (Android or iOS), to then (2) pass the advertising ID to OneTag for both Android and iOS. We have also added a page about [Hybrid apps](https://developers.criteo.com/retailer-integration/v1.1/docs/onetag-offsite-for-hybrid-apps#introduction).

### OneTag integration flow diagram
We have added a new diagram giving a more detailed overview of the integration flow for OneTag - Retail Media Offsite [on this page.](https://developers.criteo.com/retailer-integration/v1.1/docs/onetag-for-offsite#integration-flow), for a more comprehensive understanding of the process.

### New implementation examples for Flagship & Butterfly
We are now providing a couple of new implementation examples for [the Flagship](https://developers.criteo.com/retailer-integration/v1.1/docs/flagship#implementation-example) and [Butterly](https://developers.criteo.com/retailer-integration/v1.1/docs/butterfly#implementation-example) formats, so users can get a better sense of what a real implementation of those formats look like and access the code samples (HTML, CSS, Javascript) for each.
