# Tracking ad activity

Ad activity is tracked through beacons. Those are GET HTTPS calls on the client side that need to be triggered both when a set of standard actions take place, as well as when specific actions take place 

The list of available beacons will depend on the format of the ad.  

Some beacons are tied to the whole ad, while others are tied to specific products. Note that any user interaction with the ad should only trigger one call.\
If the user repeats the same action multiple times (e.g. increasing the quantity of a given product in his basket), the corresponding beacon needs to be fired the same number of times. 

Below are the beacons to be triggered for the Shelf Carousel format:

<table>
  <thead>
    <tr>
      <th>Beacon key</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>onLoadBeacon</td>
      <td>Indicates that the ad has been rendered (even if not viewable yet)</td>
    </tr>
    <tr>
      <td>onViewBeacon</td>
      <td>Indicates that the ad has been viewed (viewed = 50% of the ad in viewport for 1 second)</td>
    </tr>
    <tr>
      <td>onClickBeacon</td>
      <td>Indicates that a click was made on the format</td>
    </tr>
  </tbody>
</table>
