# New feed setup

We have different ways of handling multiple language with one retailer. In the case of Mediamarkt we propose to follow the proven Walmart Canada integration as example. For this to work we will still need two feeds but it will be one primary and one secondary one ( MMS choice to which one). Those feeds can be used by all criteo products and will be up to specs for all retail media products.

## Primary feed

The primary feed should contain all the fields that can be found at: [Feed parameters](/retailer-integration/docs/feed-parameters). It will be the feed considered as the source of truth and data with no cultural impact will be neglected if present in the secondary feed.  

The Current feed can be used but we will need some extra fields to be able too function correctly

### Specifications field

To be able to show the correct specs we need them to be in the feed. For that we think that a list of key value pairs would be best. As an example there is a Samsung phone below but if you prefer we can handle any other format that you can embed in the XML.

### Shipping

We need to show some extra shipping information on the ad : `shipping_cost` and `shipping_information`. Those will give us the possibility to show the correct information on the ad. Those should exactly show what we should print.

### Mediamarkt internal ID's

To be able to recreate some features such as the add to basket we saw that we would need some extra ID's. The following ID's will be needed if they are not identical then other provided information :

* `data-modelnumber`
* `storeId`
* `langlId`
* `categoryId_0`
* `catEntryId_0`

Those have been found by reverse engineering the add to cart button. If you think information is needed to recreate the buttons feel free to reach out

## Secondary feed

The secondary feed is only there to translate fields with visible text to the secondary language. It is a slimmed down version of the primary feed. For this feed it is important that the `id` exactly match the one in the primary feed.

The current feed can be used but it can be slimmed down if needed because most fields will be neglected. Here is a list of fields that need to stay and the one that need too be added.
