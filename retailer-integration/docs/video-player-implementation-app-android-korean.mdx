# 소개

이 문서의 목적은 **모바일 플랫폼, Android, iOS에서 온사이트 커머스 비디오 광고를 구현하는 방법**에 대한 자세한 지침을 제공하는 것입니다.

이 페이지는 모바일 구현에 중점을 둡니다. 더 광범위한 맥락에서 알아보려면 [데스크톱용 비디오 플레이어 구현 가이드](/retailer-integration/docs/video-player-implementation-guide-desktopmobile-korean)를 방문하세요.

<Note>
  **Android OM SDK 문서**

  Android OM SDK에 대한 자세한 내용은 [IAB의 이 페이지](https://interactiveadvertisingbureau.github.io/Open-Measurement-SDKAndroid/)에서도 확인할 수 있습니다.
</Note>

<Info>
  IAB에서 제공하는 OM SDK가 사용되며, 라이브러리에 대해 `minSdkVersion="16"`과 `targetSdkVersion="30"`가 표시됩니다. SDK를 사용하는 모든 Android 애플리케이션은 최소 Android 4.1(API 레벨 16) 이상을 실행하는 디바이스와 호환되어야 하며, 앱은 Android 11(API 레벨 30)에 최적화되어 있습니다.
</Info>

***

# 비디오 플레이어 구현 가이드

비디오 광고 구현을 위해서는 몇 가지 주요 단계를 고려해야 합니다.

1. VAST tag 요청 및 구문 분석,
2. DOM에 주입할 플레이어의 HTML 구조 생성,
3. 비디오 재생 중 적절한 시간에 beacons 실행,
4. 추가 검증 스크립트(OMID) 통합.

<Info>
  크리테오는 **VAST 4.2**를 지원합니다(IAB 사양은 [여기](https://iabtechlab.com/wp-content/uploads/2019/06/VAST_4.2_final_june26.pdf)에서 확인할 수 있습니다).
</Info>

***

## 1단계: VAST 태그 요청 및 구문 분석

비디오 플레이어에 대한 데이터를 얻으려면 먼저 크리테오 API 광고 응답에서 VAST 정보를 가져옵니다.

다음은 VAST를 요청하고(필요한 경우) 구문 분석하는 방법의 예입니다.

```java
DocumentBuilderFactory dbFactory = DocumentBuilderFactory.newInstance();  
DocumentBuilder dBuilder = dbFactory.newDocumentBuilder();  
Document doc = dBuilder.parse('VAST TAG URL');  
doc.getDocumentElement().normalize();
```

<Warning>
  `dBuilder`는 VAST URL을 내부적으로 쿼리합니다. 이는 네트워크 쿼리이므로 메인 UI 스레드에서 실행되어서는 안 됩니다. (예: [Android에서 네트워크 요청을 실행하는 방법](https://developer.android.com/develop/background-work/background-tasks/asynchronous))
</Warning>

VAST 태그를 구문 분석하면, 다음을 포함하여 이 문서에 설명된 필요한 정보를 추출할 수 있습니다.

* 비디오 광고의 링크,
* VAST XML 트래킹 비콘,
* OMID 검증 스크립트 및 `VerificationParameters`
* 자막 파일.

***

## 2단계: 비디오 플레이어 생성

크리테오의 비디오 플레이어는 네이티브 모바일 비디오 플레이어(Android의 경우 `ExoPlayer` 사용)를 기반으로 구축되었으며, `play()`, `pause()`와 같은 메소드와 `volume`, `duration`, `currentPosition`과 같은 속성을 제공합니다.

이러한 속성은 비디오 플레이어와 상호작용하여 음소거 또는 음소거 해제하고 재생 진행 상황을 추적하는 데 도움이 됩니다.

<Note>
  **Android용 ExoPlayer**

  Android용 ExoPlayer에 대해 자세히 알아보려면 [이 페이지](https://developer.android.com/media/media3/exoplayer)를 참조하세요.
</Note>

이러한 기능을 관리하기 위해 크리테오는 재생/일시 중지, 음소거/음소거 해제, CC(자막) 버튼에 대한 추가 제어 요소를 통합했습니다.

재생, 일시 중지, 볼륨 제어를 처리하기 위해 이러한 버튼에 클릭 리스너가 추가되었습니다.

또한, 현지 규정에 따라 필수 공지사항의 경우 **콜아웃**이나 **팝업**이 필요할 수 있습니다.

플레이어의 UI 레이아웃을 설정해야 합니다. 광고용 비디오 플레이어와 음소거 버튼만 포함되어 있습니다.

```xml
 <androidx.media3.ui.PlayerView
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/videoView"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:padding="16dp">

    &lt;LinearLayout
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:gravity="center"
        android:orientation="horizontal"&gt;

        &lt;TextView
            android:id="@+id/muteTextView"
            android:layout_width="0dp"
            android:layout_height="match_parent"
            android:layout_weight="1"
            android:gravity="center"
            android:text="Mute" /&gt;
    &lt;/LinearLayout&gt;

</androidx.media3.ui.PlayerView>
```

레이아웃이 준비되면 다음 단계는 다음을 위해 구성 요소를 인스턴스화하는 것입니다.

* UI 요소 제어
* 클릭 리스너 설정
* VAST 쿼리 및 구문 분석
* 플레이어를 준비하고 광고를 시작합니다.

> \*\* Comment 3.1:\*\* VAST 쿼리 및 구문 분석

```java expandable
public class VideoAdNativeActivity extends Activity implements Player.Listener, View.OnClickListener {

    private static final int PLAYER_UNMUTE = 1;
    private static final int PLAYER_MUTE = 0;
    private PlayerView playerView;
    private ExoPlayer player;
    private TextView muteTextView;

    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.video_ad_native_detail);

        playerView = findViewById(R.id.videoView);
        muteTextView = findViewById(R.id.muteTextView);

        playerView.requestFocus();
        playerView.setOnClickListener(this);
        muteTextView.setOnClickListener(this);
// VAST 쿼리 및 구문 분석
        ...
// 비디오 광고 플레이어를 VAST의 미디어 항목으로 인스턴스화
        initializePlayer(videoUrl);
    }

    private void initializePlayer(String videoUrl) {
// ExoPlayer를 생성한 후 콘텐츠 및 광고의 플레이어로 설정
        player = new ExoPlayer.Builder(this)
                .build();
// 재생할 MediaItem 생성
        MediaItem mediaItem =
                new MediaItem.Builder()
                        .setUri(Uri.parse(videoUrl))
                        .build();
        player.setMediaItem(mediaItem);
// 플레이어 이벤트에 청취자 추가
        player.addListener(this);

        player.setVolume(PLAYER_VOLUME);
        playerView.setPlayer(player);
// 기본 플레이어 컨트롤 비활성화(기본 컨트롤은 검색 허용)
        playerView.setUseController(false);
        player.setPlayWhenReady(true);
        player.prepare();
    }
}

```

## 3단계: OMID 광고 세션 설정

클라이언트는 플레이어가 추가 스크립트를 실행할 수 있도록 OMID를 지원해야 하며, 이를 통해 크리테오는 비디오의 뷰어빌리티를 추적할 수 있습니다. 이는 크리테오가 VAST XML 내에서 자체적인 검증 스크립트를 전송하여 비디오의 뷰어빌리티를 추적할 수 있도록 합니다.

<Info>
  **OMID 소개**

  OMID([오픈 측정 인터페이스 정의)](https://iabtechlab.com/standards/open-measurement-sdk/)는 IAB에서 제공하는 표준화된 기술로, 다양한 플랫폼과 디바이스에서 디지털 광고, 특히 비디오 광고의 뷰어빌리티를 측정하고 검증합니다.

  OMID는 업계 표준을 준수하여 조회 가능한 노출 수, 조회 가능한 비디오 재생 수 등 뷰어빌리티 지표를 측정하기 위한 일관된 프레임워크를 제공합니다. OMID를 구현하면 광고 성과와 사용자 참여에 대한 정확한 인사이트를 얻을 수 있으며, 모든 유통사에서 동일한 방식으로 비디오 뷰어빌리티를 추적하여 네트워크 전반에서 일관성을 보장할 수 있습니다.

  이 문서에서 OM SDK는 **모바일 플랫폼(Android/iOS)용 OM SDK**를 의미합니다.
</Info>

[VAST 사양](https://www.iab.com/wp-content/uploads/2016/04/VAST4.0_Updated_April_2016.pdf)의 섹션 3.16은 이러한 검증이 있는 선택적 `AdVerifications` 요소를 설명합니다. `AdVerifications` 내부에 중첩된 각 검증 요소는 검증에 필요한 리소스를 자세히 설명합니다.

<Info>
  **OM(Open-measurement) SDK**

  이러한 리소스를 처리하기 위해, 크리테오는 IAB에서 제공하는 OM SDK에 이를 전달합니다. Android용 공식 OM SDK 구현 가이드는 [여기](https://interactiveadvertisingbureau.github.io/Open-Measurement-SDKAndroid/)에서 확인할 수 있습니다. 이 문서에서는 뷰어빌리티 통합을 위한 각 단계에 필요한 코드 스니펫을 제공합니다.
</Info>

그 과정은 크게 다음과 같은 단계로 나뉩니다.

* OMID 서비스 스크립트를 로드합니다.
* 광고 세션 구성과 광고 세션 자체를 생성합니다.
* 광고를 표시하는 플레이어를 등록하고 광고 세션을 시작합니다.
* 비디오 및 광고 관련 이벤트가 발생하면 OM SDK에 `adEvents` 및 `mediaEvents` 이벤트를 트리거합니다.

<Warning>
  [IAB 구현 가이드](https://interactiveadvertisingbureau.github.io/Open-Measurement-SDKAndroid)는 크리테오의 비디오 플레이어 사양(버퍼링 이벤트 지원, 트래킹 이벤트 1회 방출, 재생/일시 중지 이벤트가 OM SDK에 알리지 않음 등)을 모두 다루기에는 충분하지 않습니다. 이러한 이유로 [IAB 문서](https://interactiveadvertisingbureau.github.io/Open-Measurement-SDKAndroid/validation.html)에 따라 통합을 검증하길 권합니다.
</Warning>

### 1. OMID 서비스 스크립트를 로드합니다.

제공된 IAB OM SDK는 광고 소재의 뷰어빌리티를 지원하기 위해 포함되어야 하는 스크립트(`omweb-v1.js`)와 모바일 라이브러리로 구성됩니다.

<Note>
  **스크립트는 자체 호스팅할 수도 있고, 사본을 호스팅하는 크리테오의 CDN에서 가져올 수도 있습니다. 해당 파일은 [이 페이지에서](https://static.criteo.net/banners/android/1.5.2/omsdk-v1_1.5.2.js) 찾을 수 있습니다.**

  Omweb-v1.js는 웹 컨텍스트를 위한 OMID JS 라이브러리입니다.
</Note>

***

### 2. 광고 세션 구성과 광고 세션 자체를 생성합니다.

광고 세션을 초기화하기 전에 OM SDK를 활성화해야 합니다. 광고 세션에는 크리테오 통합과 관련된 일부 파라미터도 지정해야 합니다.

<Info>
  OMID `VerificationScript`, `VerificationParameters` 및 `VendorKey`는 VAST 내부에서 제공됩니다.
</Info>

```java expandable
public AdSession getNativeAdSession(Context context) {
    Omid.activate(context.getApplicationContext());

    AdSessionConfiguration adSessionConfiguration =
        AdSessionConfiguration.createAdSessionConfiguration(
            CreativeType.VIDEO,
            ImpressionType.VIEWABLE,
            Owner.NATIVE,
            Owner.NATIVE, false);

    Partner partner = Partner.createPartner("criteo", "<version name>");
    final String omidJs = getOmidJs(context);

    VerificationScriptResource verificationScripts =
        VerificationScriptResource.createVerificationScriptResourceWithParameters(
            vendorKey,
            omidVerificationScript,
            omidVerificationParameters);

    AdSessionContext adSessionContext = AdSessionContext.createNativeAdSessionContext(
        partner,
        omidJs,
        Collections.singletonList(verificationScripts),
        null,
        null);

    AdSession adSession = AdSession.createAdSession(adSessionConfiguration, adSessionContext);
    return adSession;
}

public static String getOmidJs(Context context) {
    Resources res = context.getResources();
    try (InputStream inputStream = res.openRawResource(R.raw.omsdk_v1)) {
        byte[] b = new byte[inputStream.available()];
        final int bytesRead = inputStream.read(b);
        return new String(b, 0, bytesRead, "UTF-8");
    } catch (IOException e) {
        throw new UnsupportedOperationException("OM SDK JS library resource not found", e);
    }
}
```

### 4. 비디오 및 광고 관련 이벤트가 발생하면 OM SDK로 `adEvents` 및 `mediaEvents` 이벤트를 트리거합니다.

<Note>

</Note>

이러한 호출을 통해 크리테오의 검증 스크립트는 고급 뷰어빌리티 이벤트를 계산할 수 있습니다.

* 크리테오는 사분위수 이벤트를 추적할 수 있습니다(비디오가 뷰포트의 100%인 경우)
* 고급 이벤트는 2초 연속으로 뷰포트에서 비디오의 50%가 재생되고 표시되는지 추적하기 위해 계산됩니다.

`AdEvents` 및 `MediaEvents`에 대한 모든 호출은 사분위수 및 디스플레이 이벤트에 대한 기존 추적 비콘과 별도로 트리거되어야 합니다. 또한, 비디오 버퍼링 이벤트가 트리거되어야 합니다.

OMID 이벤트를 트리거하는 것은 VAST 트래커를 트리거하는 것과 비슷합니다. 두 가지 모두 다음 섹션에서 설명합니다.

## 4단계: 비디오 재생 중 비콘 실행

실행해야 할 첫 번째 비콘은 **VAST 노출 트래킹 비콘**(배치, 각 제품, VAST의 비디오용)입니다.

이에 대한 트리거를 설정하려면 플레이어 요소에 `STATE_READY` 이벤트에 대한 이벤트 리스너를 추가합니다. 이 이벤트는 플레이어가 뷰포트에서의 위치에 관계 없이 비디오를 즉시 재생할 수 있을 때 전달됩니다.

<Info>
  시간 경과에 따른 플레이어 상태의 변화를 수신하려면 `VideoAdNativeActivity`가 `Player.Listener`를 상속해야 합니다. `onPlayerStateChanged`를 구현하면 개발자는 비디오 플레이어 관련 이벤트에 대한 알림을 받게 됩니다.
</Info>

```java expandable
public class VideoAdNativeActivity extends Activity implements Player.Listener, View.OnClickListener {
    [...]

    private final OkHttpClient client = new OkHttpClient();
    private final Callback emptyCallback = new Callback() {
        @Override
        public void onFailure(Call call, IOException e) { }

        @Override
        public void onResponse(Call call, Response response) { }
    };

    private void emitBeacon(String url) {
        Request request = new Request.Builder().url(url).build();
        client.newCall(request).enqueue(emptyCallback);
    }

    @Override
    public void onPlaybackStateChanged(@Player.State int playbackState) {
        switch (playbackState) {
            case ExoPlayer.STATE_READY:
                onStateReady();
                break;
            case ExoPlayer.STATE_ENDED:
                onStateEnded();
                break;
            case ExoPlayer.STATE_BUFFERING:
                onStateBuffering();
                break;
            case ExoPlayer.STATE_IDLE:
                break;
            default:
                throw new IllegalStateException("Unknown playbackState: " + playbackState);
        }
    }

    private boolean loaded = false;

    private void onStateReady() {
        if (!loaded) {
            emitBeacon(adImpression);
            loaded = true;
// OMID로 이벤트 전달
            VastProperties vastProperties = VastProperties.createVastPropertiesForNonSkippableMedia(false, Position.STANDALONE);
            adEvents.loaded(vastProperties);
            adEvents.impressionOccurred();
        }
// OMID로 이벤트 전달
        mediaEvents.bufferFinish();
// 비디오 진행 상황을 모니터링하는 폴링 스레드 시작
        postProgress();
    }

    private void onStateBuffering() {
// OMID로 이벤트 전달
        mediaEvents.bufferStart();
    }

    [...]
}

```

<Warning>
  \*\*[정확한 비콘 추적](/retailer-integration/docs/tracking-ad-related-activity-korean)\*\*을 보장하려면 **재생 비콘**(시작 및 사분위수 비콘)은 **비디오를 처음 재생할 때 한 번만 방출되어야 합니다**. 중복을 방지하기 위해 비디오가 방출된 각 비콘에 대한 상태를 유지하는 것이 중요합니다.

  하지만 **상호작용 비콘** (클릭, 음소거, 음소거 해제, 일시 중지, 재개 등)은 **해당 사용자 동작이 발생할 때마다 방출되어야 합니다.**
</Warning>

start, `firstQuartile`, `midpoint`, `thirdQuartile` 비콘은 폴링 스레드에 의해 트리거되는 함수에서 관리됩니다.

`onProgress` 메서드는 광고 세션이 여전히 활성 상태이며 비디오가 아직 한 번 완료되지 않았는지 확인한 후 `updateQuartile` 메서드를 호출하여 현재 사분위수를 계산하고 해당 비콘을 삭제합니다. 또한 `play` 및 `pause` 이벤트를 모니터링하는 `updatePlayPause` 메서드도 호출합니다.

마지막으로 `postProgress` 메서드를 실행하여 `ProgressRunnable`의 다른 실행을 대기열에 넣어 루프를 닫습니다.

```java expandable
public class VideoAdNativeActivity extends Activity implements Player.Listener, View.OnClickListener {
    [...]

    private final Handler handler = new Handler(Looper.getMainLooper());
    private final ProgressRunnable progressRunnable = new ProgressRunnable(this);
    private static final int PROGRESS_INTERVAL_MS = 100;

    private static class ProgressRunnable implements Runnable {
        private final WeakReference&lt;VideoAdNativeActivity&gt; videoAdNativeActivityWeakReference;

        ProgressRunnable(VideoAdNativeActivity videoAdNativeActivity) {
            this.videoAdNativeActivityWeakReference = new WeakReference<>(videoAdNativeActivity);
        }

        @Override
        public void run() {
            VideoAdNativeActivity videoAdNativeActivity = videoAdNativeActivityWeakReference.get();
            if (videoAdNativeActivity == null) {
                return;
            }
            videoAdNativeActivity.onProgress();
        }
    }

    private void onProgress() {
        if (adSession == null) { 
            return; 
        }
        if (complete) { 
            return; 
        }

        updateQuartile();
        postProgress();
    }

    private void postProgress() {
        handler.removeCallbacks(progressRunnable);
        handler.postDelayed(progressRunnable, PROGRESS_INTERVAL_MS);
    }

    [...]
}

```

`updateQuartile` 함수 내부에서 비디오 광고가 아직 완료되지 않았는지 확인합니다. 완료되었다면, 현재 재생 중인 사분위수를 계산하고, 이전에 이벤트로 전송된 적이 없는지 확인한 후 전송합니다. 먼저 start 비콘을 보내고, 비디오 길이의 25%가 지나면 `firstQuartile`를 보내고, 그다음 `midpoint`를 보내고, 마지막으로 `thirdQuartile`를 보냅니다.

```java expandable
// updateQuartile 방법은 현재 재생 사분위수를 얻기 위해 몇 가지 계산을 수행합니다.
// 일부 안전 체크를 실행하고 사분위수 이벤트를 트리거합니다.
private Quartile lastSentQuartile = Quartile.UNKNOWN;

private void updateQuartile() {
    final long duration = player.getDuration();
    final long currentPosition = player.getCurrentPosition();

    if (duration != 0 && !complete) {
        final Quartile currentQuartile = getQuartile(currentPosition, duration);
// 이미 전송했거나 통과한 오래된 사분위수 통계는 보내지 마십시오.
        if (currentQuartile != lastSentQuartile && currentQuartile.ordinal() > lastSentQuartile.ordinal()) {
            sendQuartile(currentQuartile);
            lastSentQuartile = currentQuartile;
        }
    }
}

private void sendQuartile(Quartile quartile) {
    switch (quartile) {
        case START:
// start 비콘 실행
            emitBeacon(adStarted);
// OMID로 이벤트 전달
            mediaEvents.start(player.getDuration(), PLAYER_VOLUME);
            break;

        case FIRST:
// firstQuartile 비콘 실행
            emitBeacon(adFirstQuartile);
// OMID로 이벤트 전달
            mediaEvents.firstQuartile();
            break;

        case SECOND:
// midpoint 비콘 실행
            emitBeacon(adMidpoint);
// OMID로 이벤트 전달
            mediaEvents.midpoint();
            break;

        case THIRD:
// thirdQuartile 비콘 실행
            emitBeacon(adThirdQuartile);
// OMID로 이벤트 전달
            mediaEvents.thirdQuartile();
            break;

        case UNKNOWN:
        default:
            break;
    }
}
// 사분위수를 나타내는 Enum과 비디오 위치를 가장 가까운 사분위수로 반올림하는 일부 헬퍼 방법입니다.
public enum Quartile { UNKNOWN, START, FIRST, SECOND, THIRD }
// 사분위수 이벤트가 일어날 때 감지하는 방법입니다.
private static Quartile getQuartile(long position, long duration) {
    final double completionFraction = position / (double) duration;

    if (lessThan(completionFraction, 0.01)) {
        return Quartile.UNKNOWN;
    }

    if (lessThan(completionFraction, 0.25)) {
        return Quartile.START;
    }

    if (lessThan(completionFraction, 0.5)) {
        return Quartile.FIRST;
    }

    if (lessThan(completionFraction, 0.75)) {
        return Quartile.SECOND;
    }
// completionFraction > 1로 의도한 경우 크리테오는Quartile.THIRD를 보고합니다.
// 트랙이 기술적으로 완료 후 경과 시간을 보고할 수 있기 때문입니다.
// Quartile.Third가 이미 보고되지 않은 경우, 손실됩니다.
    return Quartile.THIRD;
}

private static boolean lessThan(double a, double b) {
    return b - a > EPSILON;
}

```

완료 비콘을 처리하려면 비디오 플레이어에서 실행된 종료 이벤트를 수신합니다. 이 이벤트를 수신하면 비디오가 아직 완료되지 않았는지 확인하고, 완료되었다면 완료된 비콘을 삭제합니다.

그런 다음 `completed`를 true로 설정하고 비디오의 `play()`를 호출하여 반복 효과를 만듭니다. 이후의 "종료된" 이벤트는 비디오를 다시 시작합니다.

> \*\* Comment 9.1:\*\* OMID로 이벤트 전달

**모든 재생 비콘은 플레이어가 재생 중일 때만 실행됩니다.**

<Warning>
  IAB 표준에 따라 광고가 표시되도록 하려면 **비디오 플레이어의 최소 50%가 뷰포트에 있어야 합니다.** UI가 표시되려고 할 때 호출되는 UI 스레드에서 리스너가 사용됩니다. 이 리스너에서 플레이어가 뷰포트에 50% 이상 있는지 확인하고, 그렇지 않으면 비디오를 일시 중지합니다.
</Warning>

```java expandable
private void initializePlayer(String videoUrl) {
    [...]

    playerView.getViewTreeObserver().addOnPreDrawListener(() -> {
// 비디오의 50% 이상이 보이는지 확인
// 사용자가 명시적으로 재생을 일시 중지하지 않았음
        if (isPlayingWhen50Visible && isVideoViewAtLeast50Visible(playerView)) {
            playVideo();
        } else {
            pauseVideo();
        }
        return true;
    });
}

private boolean isVideoViewAtLeast50Visible(View videoView) {
// 뷰에 대한 가시적 사각형 확보
    Rect visibleRect = new Rect();
    boolean isVisible = videoView.getGlobalVisibleRect(visibleRect);
    if (!isVisible) {
        return false;
    }
// 비디오 뷰에 대한 전체 영역 확보
    int totalArea = videoView.getWidth() * videoView.getHeight();
    if (totalArea == 0) {
        return false;
    }
// 가시적 영역 확보
    int visibleArea = visibleRect.width() * visibleRect.height();
    return (visibleArea >= totalArea / 2);
}
```
