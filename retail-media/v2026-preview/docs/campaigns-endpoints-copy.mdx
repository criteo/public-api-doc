## **Campaign Attributes**

<ParamField path="id" type="string">
  **Is Nullable?** No

  Campaign ID

  **Values:** int64

  **Default Values:** None

  **Write?** No
</ParamField>

<ParamField path="accountId" type="string" required>
  **Is Nullable?** No

  Account ID

  **Values:** int64

  **Default Value:** None

  **Write?** No
</ParamField>

<ParamField path="name" type="string" required>
  **Is Nullable?** No

  Campaign name; must be unique within an [account](/retail-media/docs/account)

  **Values:** 255 char limit

  **Default Value:** None

  **Write?** Yes
</ParamField>

<ParamField path="type" type="enum">
  *Optional.*

  **Is Nullable?** No - if attribute is passed in the call, a value must be specified

  Campaing type

  **Values:** `auction`,`preferred`

  **Default Value:** `auction`

  **Write?** None other than the accepted values
</ParamField>

<ParamField path="budget" type="number">
  *Optional.*

  **Is Nullable?** Yes

  Campaign lifetime spend cap; uncapped if omitted or set to null. Note that preferred campaign types cannot have budgets as these campaings types must be uncapped.

  **Values:** at least 0

  **Default Value:** `null`

  **Write?** Yes
</ParamField>

<ParamField path="budgetSpent" type="number">
  *Optional.*

  **Is Nullable?** No

  Amount the campaign has already spent

  **Values:** at least 0

  **Default Value:** `0.0`

  **Write?** No
</ParamField>

<ParamField path="budgetRemaining" type="number">
  **Is Nullable?** Yes

  Amount the campaign has remaining until cap is hit; `null` if budget is uncapped

  **Values:** between 0 and `budget` Amount

  **Default Value:** `null`

  **Write?** No
</ParamField>

<ParamField path="clickAttributionWindow" type="enum" required>
  **Is Nullable?** No

  Post-click attribution window

  **Values:** `7D`,`14D`, `30D`,`unknown`

  **Default Value:** `30D`

  **Write?** Yes
</ParamField>

<ParamField path="viewAttributionWindow" type="enum" required>
  **Is Nullable?** No

  Post-view attribution window

  **Values:** `None`,`7D`,`14D`, `30D`,`unknown`

  **Default Value:** `None`

  **Write?** Yes
</ParamField>

<ParamField path="clickAttributionScope" type="enum">
  **Is Nullable?** No

  Post-click attribution scope

  **Values:** `samesku`, `sameskucategory`, `sameskucategorybrand`,`unknown`

  **Default Value:** `sameskucategory`

  **Write?** None other than the accepted values
</ParamField>

<ParamField path="viewAttributionScope" type="enum">
  **Is Nullable?** No

  Post-view attribution scope

  **Values:** `samesku`, `sameskucategory`, `sameskucategorybrand`

  **Default Value:** `sameskucategory`

  **Write?** None other than the accepted values
</ParamField>

<ParamField path="promotedBrandIds" type="string array">
  **Is Nullable?** No

  Set of [brands](/retail-media/docs/brands) represented in line items of the campaign; note this set is append only

  **Values:** list of `brandId`

  **Default Value:** `[ ]`

  **Write?** No
</ParamField>

<ParamField path="drawableBalanceIds" type="string array">
  **Is Nullable?** No

  List of [balances](/retail-media/docs/balances) the campaign is able to draw from; at least one balance is required for a campaign to begin

  **Values:** list of `balanceId`

  **Default Value:** `[ ]`

  **Write?** No
</ParamField>

<ParamField path="status" type="enum">
  **Is Nullable?** No

  Campaign status, derived from the status of [line items](/retail-media/docs/line-items) it holds; active if at least one line item is active. To understand the conditions that will cause a status to change, check out our [status page](/retail-media/docs/status)

  **Values:** `active`, `inactive`

  **Default Value:** `inactive`

  **Write?** No
</ParamField>

<ParamField path="monthlyPacing" type="decimal">
  *Optional.*

  **Is Nullable?** Yes

  The maximum monthly spend allowed for the campaign in the currency of the account. The spend is constrained by remaining account balance and total budget of the campaign. Monthly budget spend reset monthly at the start of the month based on the account timezone.

  **Values:** `null` or a value that is greater or equal to 0.01

  **Default Value:** `null`

  **Write?** Yes
</ParamField>

<ParamField path="dailyPacing" type="decimal">
  *Optional.*

  **Is Nullable?** Yes

  The maximum daily spend allowed for the campaign in the currency of the account. As long as the parameter is not set to `null` and auto daily pacing is not set to `true` then monthly pacing will be applied to the campaign.

  - If monthly pacing is set to `null` and auto daily pacing is set to `false`, then no daily spend limit will be enforced.

  - The maximum daily spend allowed for the campaign in the currency of the account.

  **Values:** `null` or a value that is greater or equal to 0.01

  **Default Value:** `null`

  **Write?** Yes
</ParamField>

<ParamField path="isAutoDailyPacing" type="boolean" required>
  **Is Nullable?** No

  Auto daily pacing flag for the campaign budget. The daily pacing value is automatically calculated with respect to the days left in the month for the account. The campaign's remaining account balance, total budget, and monthly pacing value is constrained by the account timezone and the campaign status. If auto daily pacing is enabled, dailyPacing value should be left empty (null). Note the value may be affected by budget override. Daily budget spend is reset daily at the start of the day based on the account timezone.

  **Values:** `true`, `false`

  **Default Value:** N/A

  **Write?** Yes
</ParamField>

<ParamField path="startDate" type="date-time">
  *Optional.*

  **Is Nullable?** Yes

  Campaign start date. The campaign start inactive if invalid start date is not today or end date is in previous day.

  **Values:** YYYY-MM-DD

  **Default Value:** `Null`

  **Write?** Yes
</ParamField>

<ParamField path="endDate" type="date-time">
  *Optional.*

  **Is Nullable?** Yes

  Campaign end date

  **Values:** YYYY-MM-DD

  **Default Value:** `Null`

  **Write?** Yes
</ParamField>

<ParamField path="createdAt" type="timestamp">
  **Is Nullable?** No

  Timestamp in UTC of campaign creation

  **Values:** ISO-8601

  **Default Value:** N/A

  **Write?** No
</ParamField>

<ParamField path="updatedAt" type="timestamp">
  **Is Nullable?** No

  Timestamp in UTC of last campaign update

  **Values:** ISO-8601

  **Default Value:** N/A

  **Write?** No
</ParamField>

 

 

## **Create a Campaign**

This endpoint creates a new campaign in the specified account.

<EndpointBadge method="post">
```http
https://api.criteo.com/preview/retail-media/accounts/{accountId}/campaigns
```
</EndpointBadge>

-H 'Accept: application/json' \
-H 'Authorization: Bearer <MY_ACCESS_TOKEN>' \

		"type": "RetailMediaCampaignv202301",

<EndpointBadge method="put">
```http

```
</EndpointBadge>

**Sample Request** 

<CodeGroup>

```bash cURL

-H 'Content-Type: application/json' \
-H 'Accept: application/json' \
-H 'Authorization: Bearer <MY_ACCESS_TOKEN>' \
--data-raw '{
	"data": {
		"id": "405429491681951744",
		"type": "RetailMediaCampaignv2023001",
		"attributes": {
			"name": "API Campaign New ",
			"isAutoDailyPacing": "true",
			"startDate": "2023-01-23",
			"endDate": "2023-01-31",
			"type": "auction",
			"drawableBalanceIds": [
				"18"
			],
			"clickAttributionWindow": "30D",
			"viewAttributionWindow": "30D",
			"budget": "150",
			"monthlyPacing": "10",
			"dailyPacing": null,
			"clickAttributionScope": "sameSkuCategory",
			"viewAttributionScope": "sameSkuCategory"
		}
	}
}'
```
