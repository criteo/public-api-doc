## Step 1: Submit the Product Import Batch

**Endpoint:**

```http
POST /preview/retail-media/catalog/products/batch
```

**Headers:**

- `Authorization: Bearer {ACCESS_TOKEN}`
- `Content-Type: application/json`

**Request Body Example:**

```json
{
  "requests": [
    {
      "retailerId": "retailer-example-id",
      "marketplaceId": "marketplace-example-id",
      "products": [
        {
          "sku": "SKU-12345",
          "attributes": {
            "title": "Elegant Velvet Chair",
            "description": "A luxurious navy blue velvet chair.",
            "price": 159.99
          }
        }
      ]
    }
  ]
}
```

***

## Step 2: Retrieve the `operationToken`

**Success Response:**

```json
{
  "operationToken": "abc123-operation-token"
}
```

<Info>
  Save the `operationToken` to check the status and results of the import.
</Info>

***

## Step 3: Retrieve the Import Report

**Endpoint:**

```http
GET /preview/retail-media/catalog/products/batch/report/{operation-token}
```

**Example:**

```http
GET /preview/retail-media/catalog/products/batch/report/abc123-operation-token
```

**Success Response Example:**

```json
{
  "status": "validated",
  "successfulCount": 1,
  "failedCount": 0,
  "errors": []
}
```

<Info>
  This response includes the processing status and a detailed report of any validation or transformation errors.
</Info>

> 👍 
>
> The report should be **available a few minutes after submitting the product batches**. In the meantime, if you request the report, the response status will be `In progress` until the report is ready.

<Warning>
  When a report is marked as `validated`, `validated_with_errors`, or `failed,` it becomes available for download and remains accessible **for only 4 days**. You can call the `batch-report/{operation-token}` endpoint to retrieve it. No notification is sent or required.
</Warning>
