This page covers common issues that may arise during early MCP setup and testing.

## Authentication or Consent Issues

Examples:

* Invalid credentials.
* Missing redirect URI registration.
* Consent flow not completing successfully.
* Using a production server without required production access approval.

## Scope or Permissions Mismatch

Examples:

* The partner can connect, but the returned accounts or tools do not match expectations.
* The credentials do not include the scopes needed for the workflow being tested.

## Tool Discovery Issues

Examples:

* The MCP client connects, but expected tools are not visible.
* The client platform has limitations in how it surfaces tools to end users.

## Async Reporting Confusion

Examples:

* Reporting workflows require polling or multiple steps.
* The partner expects immediate synchronous output for workflows that are asynchronous underneath.
