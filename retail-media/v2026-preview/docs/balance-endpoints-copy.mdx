## **Request Body Parameters**

#### `id` Required for POST requests

- **Data Type:** string

**Description:** Campaign ID to which balance should be appended or deleted from
- **Values:** int64

## **Balance Response Attributes**

#### `id`

- **Data Type:** string
- **Values:** int64

**Description:** Balance ID

#### `name`

- **Data Type:** string
- **Values:** -

**Description:** Balance name

#### `poNumber`

- **Data Type:** string
- **Values:** -

**Description:** Purchase order number

#### `deposited`

- **Data Type:** number
- **Values:** -

**Description:** Amount of funds deposited, uncapped if `null`

#### `spent`

- **Data Type:** number
- **Values:** Amount of funds already spent

**Description:** Amount of funds already spent

#### `remaining`

- **Data Type:** number
- **Values:** between 0 and `deposited`

**Description:** Amount of funds remaining until cap is hit, `null` if not set

#### `startDate`

- **Data Type:** date
- **Values:** YYYY-MM-DD

**Description:** Balance start date in the [account](/retail-media/docs/accounts-endpoints) timeZone if not set

#### `endDate`

- **Data Type:** date
- **Values:** YYYY-MM-DD

**Description:** Balance end date in the [account](/retail-media/docs/accounts-endpoints) timeZone available indefinitely if `null`

#### `status`

- **Data Type:** enum
- **Values:** `active`, `scheduled`, `ended`

**Description:** Balance status

#### `createdAt`

- **Data Type:** timestamp
- **Values:** ISO-8601

**Description:** Timestamp in UTC of balance creation

#### `updatedAt`

- **Data Type:** timestamp
- **Values:** ISO-8601

**Description:** Timestamp in UTC of balance updated

 

## **Get All Balances**

This endpoint lists all balances in an account.  Response results will be provided in paginated form.

<Badge color="green" size="sm" shape="pill">GET</Badge>

```text
https://api.criteo.com/preview/retail-media/accounts/{accountId}/balances
```

**Sample Request**

<CodeGroup>

```bash cURL
curl -X GET "https://api.criteo.com/preview/retail-media/accounts/18446744073709551616/balances" \
    -H "Authorization: Bearer <MY_ACCESS_TOKEN>"
```

## **Add Campaigns to a Specific Balance**

This endpoint adds one or more campaigns to the specified balance.  The resulting state of the balance is returned as a single page.  In this example, a campaign had already existed on the balance before two new additions.

<Badge color="blue" size="sm" shape="pill">POST</Badge>

```text
https://api.criteo.com/preview/retail-media//balances/{balanceId}/campaigns/append
```

**Sample Request** 

<CodeGroup>

```bash cURL
curl -X GET "https://api.criteo.com/preview/retail-media/balances/14094543095747588032/campaigns/append" \
    -H "Authorization: Bearer <MY_ACCESS_TOKEN>" \
    -H "Content-Type: application/json" \
    -d '{
            "data": [
                {
                    "type": "RetailMediaCampaign",
                    "id": "3683145960016759663"
                },
                {
                    "type": "RetailMediaCampaign",
                    "id": "16108177282234788969"
                }
            ]
        }'
```

```python Python
import requests
import json

payload = json.dumps({
  "data": [
    {
      "id": "1280",

    }
  ]
```
