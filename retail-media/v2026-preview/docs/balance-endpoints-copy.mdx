## **Request Body Parameters**

<ParamField path="id" type="string" required>
  *Required for POST requests.*

  Campaign ID to which balance should be appended or deleted from

  **Values:** int64
</ParamField>

## **Balance Response Attributes**

<ParamField path="id" type="string">
  Balance ID

  **Values:** int64
</ParamField>

<ParamField path="name" type="string">
  Balance name

  **Values:** -
</ParamField>

<ParamField path="poNumber" type="string">
  Purchase order number

  **Values:** -
</ParamField>

<ParamField path="deposited" type="number">
  Amount of funds deposited, uncapped if `null`

  **Values:** -
</ParamField>

<ParamField path="spent" type="number">
  Amount of funds already spent

  **Values:** Amount of funds already spent
</ParamField>

<ParamField path="remaining" type="number">
  Amount of funds remaining until cap is hit, `null` if not set

  **Values:** between 0 and `deposited`
</ParamField>

<ParamField path="startDate" type="date">
  Balance start date in the [account](/retail-media/docs/accounts-endpoints) timeZone if not set

  **Values:** YYYY-MM-DD
</ParamField>

<ParamField path="endDate" type="date">
  Balance end date in the [account](/retail-media/docs/accounts-endpoints) timeZone available indefinitely if `null`

  **Values:** YYYY-MM-DD
</ParamField>

<ParamField path="status" type="enum">
  Balance status

  **Values:** `active`, `scheduled`, `ended`
</ParamField>

<ParamField path="createdAt" type="timestamp">
  Timestamp in UTC of balance creation

  **Values:** ISO-8601
</ParamField>

<ParamField path="updatedAt" type="timestamp">
  Timestamp in UTC of balance updated

  **Values:** ISO-8601
</ParamField>

 

## **Get All Balances**

This endpoint lists all balances in an account.  Response results will be provided in paginated form.

<EndpointBadge method="get">
```http
https://api.criteo.com/preview/retail-media/accounts/{accountId}/balances
```
</EndpointBadge>

**Sample Request**

<CodeGroup>


## **Add Campaigns to a Specific Balance**

This endpoint adds one or more campaigns to the specified balance.  The resulting state of the balance is returned as a single page.  In this example, a campaign had already existed on the balance before two new additions.

<EndpointBadge method="post">
```http
https://api.criteo.com/preview/retail-media//balances/{balanceId}/campaigns/append
```
</EndpointBadge>

**Sample Request** 

<CodeGroup>

```bash cURL
curl -X GET "https://api.criteo.com/preview/retail-media/balances/14094543095747588032/campaigns/append" \
    -H "Authorization: Bearer <MY_ACCESS_TOKEN>" \
    -H "Content-Type: application/json" \
    -d '{
            "data": [
                {
                    "type": "RetailMediaCampaign",
                    "id": "3683145960016759663"
                },
                {
                    "type": "RetailMediaCampaign",
                    "id": "16108177282234788969"
                }
            ]
        }'
```

```python Python
import requests
import json

url = "https://api.criteo.com/preview/retail-media/balances/13/campaigns/append"

payload = json.dumps({
  "data": [
    {
      "id": "1280",
      "type": "RetailMediaCampaign"
    }
  ]
})

  'Content-Type': 'application/json',
  'Accept': 'application/json',
  'Authorization': 'Bearer <MY_ACCESS_TOKEN>'
```
