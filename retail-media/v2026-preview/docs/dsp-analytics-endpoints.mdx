## Endpoints

Those endpoints support requests to create campaign, line item and account reports and allow retrieving the report data and status.

<Info>
  Among the endpoints presented on this page, four are available in [the Stable version.](/retail-media/docs/demand-side-analytics-get-started#/) Only the Accounts level report request is currently in `Preview`. For more details on this endpoint, visit our [dedicated guide](/retail-media/v2025-preview/docs/accounts#/).
</Info>

## Create a Report Request

Reporting endpoints provide two separate endpoints that allow downloading reports at the campaign or line item level. Each of the following requests can be repeated for each individual report type.

### Campaigns Level Report

<Badge color="red" size="sm" shape="pill">POST</Badge>

```text
https://api.criteo.com/preview/retail-media/reports/campaigns
```

**Sample Request**

<CodeGroup>

```bash cURL
curl -X POST "https://api.criteo.com/preview/retail-media/reports/campaigns" \
    -H "Authorization: Bearer <MY_ACCESS_TOKEN>" \
    -H "Content-Type: application/json" \
    -d '{
            "data": {
                "type": "RetailMediaReportRequest",
                "attributes": {
                    "id": "8343086999167541140",
                    "reportType": "summary",
                    "startDate": "2020-04-06",
                    "endDate": "2020-06-04",
                    "timeZone": "America/New_York"
                }
            }
        }'
```

```python Python

import json

      "endDate": "2022-07-04",
      "timeZone": "America/New_York"
    }
  }
})

  'Content-Type': 'application/json',
  'Accept': 'application/json',
  'Authorization': 'Bearer <MY_ACCESS_TOKEN>'

```

<Badge color="blue" size="sm" shape="pill">GET</Badge>

```text

```

**Sample Request**

    -H "Authorization: Bearer <MY_ACCESS_TOKEN>"
