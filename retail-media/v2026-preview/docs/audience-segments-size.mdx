## Introduction

The "audience-segments sizes" endpoints allow estimating (`estimate` endpoint) and retrieving (`compute`endpoint) the size of one or more audience-segments.

***

### Comparative use of `estimate` VS `compute` endpoints

<table>
  <thead>
    <tr>
      <th>Feature</th>
      <th><code>compute</code> Segment Endpoint</th>
      <th><code>estimate</code> Segment Endpoint</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Purpose</strong></td>
      <td>Computes size for one or more previously saved segments by ID.</td>
      <td>Provides a quick size estimation for a specific segment.</td>
    </tr>
    <tr>
      <td><strong>Input Definition</strong></td>
      <td>Requires a list of saved segment IDs.</td>
      <td>A single segment ID is provided inline.</td>
    </tr>
    <tr>
      <td><strong>Use Case</strong></td>
      <td>Used for analyzing reach of known, existing segments.</td>
      <td>Ideal for testing or previewing a specific segment.</td>
    </tr>
    <tr>
      <td><strong>Segment Source</strong></td>
      <td>Segment definitions are already stored in the system.</td>
      <td>Segment reference is passed directly in the request payload.</td>
    </tr>
    <tr>
      <td><strong>Response Format</strong></td>
      <td>Returns an array of segment sizes, one per provided ID.</td>
      <td>Returns a single segment size object.</td>
    </tr>
    <tr>
      <td><strong>Channel Specification</strong></td>
      <td>Required in the request payload.</td>
      <td>Required in the request payload.</td>
    </tr>
  </tbody>
</table>

## Compute Audience Segment(s) sizes

This endpoint returns the size computation for an audience-segment (if available and if supported). If the size cannot be estimated, an error is returned.

<Badge color="red" size="sm" shape="pill">POST</Badge>

```text
https://api.criteo.com/preview/retail-media/accounts/{accountId}/audience-segments/compute-sizes
```

***

### **Audience Segment Size Computation Attributes**

<table>

    </tr>
    <tr>
      <td><code>channel</code>&#42;</td>

      <td><code>size</code></td>
      <td>integer</td>

    </tr>
    <tr>

      <td>decimal</td>
      <td>Reach in number of users relative to the retailer’s total audience (e.g., 0.5523 = 55.23%).<br />Accepted values: 0.0 ≤ <code>relativeSize</code> ≤ 1.0<br />Writeable? N / Nullable? N</td>
    </tr>
  </tbody>
</table>

_(*) Required in body when requesting computation_

  * **Primary Key**: A unique, immutable identifier of the entity, generated internally by Criteo. Primary keys are typically ID fields (e.g., `retailerId`, `campaignId`, `lineItemId`) and are usually required in the URL path.
</Info>

## Estimate Audience Segment size

Returns the size estimation for an Audience Segment (if available and if supported). If the size cannot be estimated, an error is returned.

<Warning>
  The `Estimate` endpoints are resource-intensive; therefore, bulk processing is not supported.
</Warning>

<Badge color="red" size="sm" shape="pill">POST</Badge>

```text
https://api.criteo.com/preview/retail-media/accounts/{accountId}/audience-segments/estimate-size
```

## Errors and warnings

#### `segment-size-not-available`

**Error**: Segment size not available  
**Message**: The segment size is currently not available.

#### `segment-size-too-small`

**Error**: Segment size is too small  
**Message**: Size is below the minimum: 20.
