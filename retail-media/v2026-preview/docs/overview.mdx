title: Overview
---

* Reports are requested & retrieved through **asynchronous endpoints**

* All reports are at the **daily granularity** except for the **Attributed Transactions** report, which is at the transaction level.

* The report date window has a max of **100 days**; data is retained for up to **3 years**.

* Report attribution windows and `timezone` are fully configurable.

* Data latency is typically 2-8 hours but may have minor adjustments through the 120th hour before finalization.

* Data is processed and batched **hourly**. Same-day data may be partially available.

* While reports with an end date equal to the current date have been cached for **1 hour**, reports with an end date earlier than the current date are cached for **24 hours**.

* Exact expiration is indicated by the `expiresAt` field in the status response.

## Responses

| Code | Description |
|------|-------------|
| `200` | Request executed successfully. |
| `400` | Validation error. Common causes: `endDate` is more than 100 days from `startDate`; `timezone` is not a valid IANA identifier; `format` is not `json` or `csv`; `winRate` metric requested without `campaignTypeName` in dimensions (Sponsored Products only). |
| `410` | Report expired, lost, or failed to create. Submit a new request to regenerate. |
