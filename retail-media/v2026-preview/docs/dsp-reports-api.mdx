## Overview

This page documents the redesigned DSP reporting surface, available in Preview and targeting Stable 2026.07.

The three aggregation-level endpoints (`/reports/campaigns`, `/reports/line-items`, `/reports/accounts`) are replaced by a single flexible endpoint: `POST /reports/performance`. The `reportType` field is removed — metrics and dimensions are now specified explicitly in every request.

Two report types previously available via `/reports/campaigns` are promoted to their own dedicated endpoints:

| Old surface | New surface |
| :---------- | :---------- |
| `POST /reports/campaigns` with `reportType: summary` (campaign level) | `POST /reports/performance` with `campaignIds` filter |
| `POST /reports/campaigns` with `reportType: summary` (line-item level) | `POST /reports/performance` with `lineItemIds` filter |
| `POST /reports/accounts` | `POST /reports/performance` with `accountIds` filter |
| `POST /reports/campaigns` with `reportType: capout` | `POST /reports/missed-opportunities` |
| `POST /reports/campaigns` with `reportType: attributedTransactions` | `POST /reports/attributed-transactions` |

<Warning>
  Legacy endpoints (`/reports/campaigns`, `/reports/line-items`, `/reports/accounts`) remain functional in the current Stable version. They are excluded from future versions and will return `Deprecation` and `Sunset` HTTP headers once the new version ships. No automatic migration — new integrations should target the new endpoints.
</Warning>

All three new endpoints are **asynchronous**: submit a request to receive a `reportId`, poll `/reports/{reportId}/status` until `status: "Success"`, then retrieve results from `/reports/{reportId}/output`.

## Endpoint: Unified performance report

Replaces the three aggregation-level report endpoints. At least one of `accountIds`, `campaignIds`, or `lineItemIds` must be provided in `filters`.

<EndpointBadge method="post">
```http
https://api.criteo.com/preview/retail-media/reports/performance
```
</EndpointBadge>

### Filters

All filter fields are optional arrays, but at least one of `accountIds`, `campaignIds`, or `lineItemIds` is required.

| Filter | Type | Allowed values |
| ------ | ---- | -------------- |
| `accountIds` | `string[]` | Account IDs |
| `campaignIds` | `string[]` | Campaign IDs |
| `lineItemIds` | `string[]` | Line item IDs |
| `campaignTypes` | `string[]` | `sponsoredProducts`, `onSiteDisplays` |
| `buyTypes` | `string[]` | `auction`, `preferredDeals`, `sponsorship` |
| `mediaTypes` | `string[]` | `unknown`, `video`, `display` |
| `salesChannels` | `string[]` | `online`, `offline` |
| `activationPlatforms` | `string[]` | Activation platform IDs |
| `budgetModels` | `string[]` | Budget model IDs |
| `searchTermTypes` | `string[]` | Search term type IDs |
| `searchTermTargetings` | `string[]` | Search term targeting IDs |
| `targetedKeywordTypes` | `string[]` | Keyword type IDs |

### Request attributes

| Attribute | Type | Required | Description |
| --------- | ---- | :------: | ----------- |
| `filters` | object | Yes | At least one of `accountIds`, `campaignIds`, or `lineItemIds` must be provided. See Filters table above. |
| `dimensions` | `string[]` | Yes | One or more dimensions from the Dimensions table above. |
| `metrics` | `string[]` | Yes | One or more metrics from the Metrics table above. |
| `startDate` | `string` (date) | Yes | Inclusive start date (`YYYY-MM-DD`). |
| `endDate` | `string` (date) | Yes | Inclusive end date (`YYYY-MM-DD`). |
| `timezone` | `string` | No | IANA timezone name. Defaults to account timezone. |
| `clickAttributionWindow` | `string` | No | `none`, `7D`, `14D`, `30D`. |
| `viewAttributionWindow` | `string` | No | `none`, `1D`, `7D`, `14D`, `30D`. |
| `format` | `string` | No | `json`, `json-compact`, `json-newline`, `csv`. Default: `json-compact`. |

## Endpoint: Missed opportunities report

Dedicated endpoint replacing `reportType: capout` from `/reports/campaigns`. Provides visibility into budget exhaustion events and the resulting missed traffic, spend, and sales.

<Warning>
  Capout metrics have been renamed in this endpoint. The `capout` prefix has been dropped from all metrics except `capoutHour`:

  | Old name (legacy) | New name |
  | :---------------- | :------- |
  | `capoutMissedClicks` | `missedClicks` |
  | `capoutMissedImpressions` | `missedImpressions` |
  | `capoutMissedSales` | `missedSales` |
  | `capoutMissedSpend` | `missedSpend` |
  | `capoutMissedTraffic` | `missedTraffic` |
  | `capoutHour` | `capoutHour` *(retained)* |
</Warning>

<EndpointBadge method="post">
```http
https://api.criteo.com/preview/retail-media/reports/missed-opportunities
```
</EndpointBadge>

### Filters

| Filter | Type | Allowed values |
| ------ | ---- | -------------- |
| `accountIds` | `string[]` | Account IDs |
| `campaignIds` | `string[]` | Campaign IDs |
| `lineItemIds` | `string[]` | Line item IDs |
| `mediaTypes` | `string[]` | `unknown`, `video`, `display` |
| `salesChannels` | `string[]` | `online`, `offline` |

### Dimensions

`date`, `campaignId`, `campaignName`, `lineItemId`, `lineItemName`, `retailerId`, `retailerName`, `buyType`, `bidStrategy`

### Metrics

`daypartingScheduled`, `totalSpend`, `roas`, `capoutHour`, `attributedSales`, `impressions`, `clicks`, `cpc`, `cpm`, `ctr`, `missedTraffic`, `missedSpend`, `missedClicks`, `missedImpressions`, `missedSales`

## Endpoint: Attributed transactions report

Dedicated endpoint replacing `reportType: attributedTransactions` from `/reports/campaigns`. Provides hourly logs of each transaction attributable to an impression or click, with advanced attribution dimensions.

<EndpointBadge method="post">

```http
https://api.criteo.com/preview/retail-media/reports/attributed-transactions
```
</EndpointBadge>

| Filter | Type | Allowed values |
| ------ | ---- | -------------- |
| `accountIds` | `string[]` | Account IDs |
| `campaignIds` | `string[]` | Campaign IDs |
| `lineItemIds` | `string[]` | Line item IDs |
| `mediaTypes` | `string[]` | `unknown`, `video`, `display` |

### Dimensions

`purchasedDate`, `purchasedHour`, `advertisedDate`, `advertisedHour`, `daysDifference`, `campaignId`, `campaignName`, `lineItemId`, `lineItemName`, `advertisedProductId`, `advertisedProductGtin`, `advertisedProductMpn`, `advertisedProductName`, `advertisedProductCategory`, `purchasedProductId`, `purchasedProductGtin`, `purchasedProductMpn`, `purchasedProductName`, `purchasedProductCategory`, `advertisedEngagement`, `advertisedToPurchasedProductRelationship`, `salesChannel`, `retailerId`, `retailerName`, `pageType`, `keyword`, `attributionWindow`, `saleSellerId`, `saleSellerName`, `activitySellerId`, `activitySellerName`

## Report status and output

All three endpoints follow the same async pattern. Use the shared status and output endpoints to retrieve results.

| Verb | Endpoint | Description |
| :--- | :------- | :---------- |
| `GET` | `/preview/retail-media/reports/{reportId}/status` | Poll until `status: "Success"` |
| `GET` | `/preview/retail-media/reports/{reportId}/output` | Download report data |
