The balance management endpoints are additional sets of balance apps that allow users to have more control of their balances, along with the existing balance endpoints, which help to link existing balances to a campaign. Balance management will provide flexibility to create and update balances. Balance managing also provides additional `POST` and `PATCH` endpoints to provide the ability to edit specific balance attributes, depending on the end-user access level.

***

<Note>
  ### Learn more about campaign management with our API [**here**](/retail-media/v2026-preview/docs/balances)!
</Note>

## Balance response attributes

#### `id` Required

- **Data Type:** string
- **Values:** int64

**Description:** Balance ID

#### `name` Required

- **Data Type:** string
- **Values:** string

**Description:** The name of the balance

#### `poNumber` Required

- **Data Type:** string
- **Values:** string

**Description:** Purchase order number

#### `deposited` Required

- **Data Type:** decimal
- **Values:** decimal value or `null` for uncapped balances

**Description:** The amount of billable funds allotted to the balance. The amount should be based on the currency types of an account that the balance link to. User may want to confirm currency with their account settings.For uncapped budgets pass `null` as the deposited amount.

#### `spent` Required

- **Data Type:** number
- **Values:** Amount of funds already spent

**Description:** The total lifetime spend on this balance

#### `remaining` Required

- **Data Type:** number
- **Values:** between 0 and `deposited`

**Description:** Amount of funds remaining until the balance cap is hit. The balance will return a `null` value if no funds have been used.

#### `startDate` Required

- **Data Type:** date
- **Values:** YYYY-MM-DD

**Description:** The balance start date in the [account](/retail-media/docs/accounts-endpoints) timeZone if not set

#### `endDate`

- **Data Type:** date
- **Values:** YYYY-MM-DD

**Description:** Balance end date in the [account](/retail-media/docs/accounts-endpoints) timeZone. If the balance has a `null` end date then it indicates that there is no expiration date and the balance will not stop spending until an end date is specified.

#### `status`

- **Data Type:** enum
- **Values:** `active`, `scheduled`, `ended`, `unknown`

**Description:** The current balance status

#### `createdAt`

- **Data Type:** timestamp
- **Values:** ISO-8601

**Description:** Timestamp in UTC of balance creation

#### `updatedAt`

- **Data Type:** timestamp
- **Values:** ISO-8601

**Description:** Timestamp in UTC of balance updated

## Get all balances

<Badge color="green" size="sm" shape="pill">GET</Badge>

## Readjust funds for an existing balance

Adjust an existing balance funds without changing other balance attributes. 

<Badge color="blue" size="sm" shape="pill">POST</Badge>

```text
https://api.criteo.com/preview/retail-media/accounts/{accountId}/balances/{balanceid}/add-funds
```

**Sample Request** 

In the example below, the call requests a $10 deduction from an existing balance. The original balance was $100, and after the call, the new balance amount is now $90. To remove funds from a balance set the `deltaAmount` to a negative number, whereas set a positive amount to add funds. 

```bash
curl -L 'https://api.criteo.com/preview/retail-media/accounts/4/balances/475388270283501568/add-funds' \
-H 'Content-Type: application/json' \
-H 'Accept: application/json' \
-H 'Authorization: Bearer <MY_ACCESS_TOKEN>' \
-d '{
	"data": {
		"attributes": {
			"deltaAmount": "-10",
			"memo": "Balance re-adjustment. Removing $10 from balance",
			"poNumber": "123456"
		},
		"id": "475388270283501568",
		"type": "<string>"
	}
}'
```

{

        "attributes": {
