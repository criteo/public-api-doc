## Introduction

The "audiences sizes" endpoints allow estimating (`estimate` endpoint) and retrieving (`compute`endpoint) the size of one or more audiences.

***

### Comparative use of `estimate` VS `compute` endpoints

<table>
  <thead>
    <tr>
      <th>Feature</th>
      <th><code>compute</code> Audience Endpoint</th>
      <th><code>estimate</code> Audience Endpoint</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Purpose</strong></td>
      <td>Computes size for one or more previously saved audiences by ID.</td>
      <td>Provides a quick size estimation for a dynamically defined audience.</td>
    </tr>
    <tr>
      <td><strong>Input Definition</strong></td>
      <td>Requires a list of saved audience IDs.</td>
      <td>Audience is defined inline using logical structure (<code>algebra</code>).</td>
    </tr>
    <tr>
      <td><strong>Use Case</strong></td>
      <td>Used for analyzing reach of known, existing audience resources.</td>
      <td>Ideal for previewing potential audience reach during design.</td>
    </tr>
    <tr>
      <td><strong>Audience Composition</strong></td>
      <td>Audience composition is already stored in the system.</td>
      <td>Expressed directly in the request payload using <code>and</code>, <code>or</code>, <code>not</code>.</td>
    </tr>
    <tr>
      <td><strong>Response Format</strong></td>
      <td>Returns an array of audience sizes, one per provided ID.</td>
      <td>Returns a single audience size object.</td>
    </tr>
    <tr>
      <td><strong>Channel Specification</strong></td>
      <td>Required in the request payload.</td>
      <td>Required in the request payload.</td>
    </tr>
  </tbody>
</table>

## Compute Audience(s) sizes

This endpoint returns the size computation for one or multiple audiences (if available and if supported). If the size cannot be estimated, an error is returned.

<Badge color="blue" size="sm" shape="pill">POST</Badge>

```text
https://api.criteo.com/preview/retail-media/accounts/{accountId}/audiences/compute-sizes
```

***

## Estimate Audience size

<Warning>
  The `Estimate` endpoints are resource-intensive; therefore, bulk processing is not supported.
</Warning>

<Badge color="blue" size="sm" shape="pill">POST</Badge>

```text

```

## Errors and Warnings

#### `audience-size-not-available`

**Error**: Audience size not available  
**Message**: The audience size is currently not available.

#### `audience-size-too-small`

**Error**: Audience size is too small  
**Message**: Size is below the minimum: 20.
