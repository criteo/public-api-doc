- This endpoint and its underlying capabilities only apply Preferred Deals Line items.
- Store targeting allows you to restrict a Preferred Deals line items to serve only for users shopping in a limited list of stores

# Endpoints

- `GET /preferred-line-items/`{line-item-id}`/targeting/stores` Get All targeted Stores on a specific Line Item
- `PUT /preferred-line-items/`{line-item-id}`/targeting/stores` Initialize or Override Stores targeting on a specific Line Item
- `POST /preferred-line-items/`{line-item-id}`/targeting/stores/append` Add targeted Stores to a Specific Line Item
- `POST /preferred-line-items/`{line-item-id}`/targeting/stores/delete` Remove targeted Stores from a Specific Line Item

When using the feature, the PUT endpoint must be used to first initialize it, defining its `scope`. POST endpoints can only alter the list of stores.

# List all targeted Stores on a Preferred Deals Line Item

<Badge color="blue" size="sm" shape="pill">GET</Badge>

```text
https://api.criteo.com/preview/retail-media/preferred-line-items/`{line-item-id}`/targeting/stores
```

**Sample Request**

```bash
curl -X GET "https://api.criteo.com/preview/retail-media/preferred-line-items/2465695028166499188/targeting/stores" \
    -H "Authorization: Bearer myaccesstoken"
```

**Sample Response**

```json
{
    "data": {
        "type": "RetailMediaStoreTarget",
        "attributes": {
            "scope": "include",
            "storeIds": [
                "store3",
                "store4"
            ]
        }
    },
    "warnings": [],
    "errors": []
}
```

# Initiate targeted Stores list and scope on a Line item

- This endpoint initializes the feature on a Preferred Deals Line item.
- If targeted Stores were already defined, those will be overridden

<Badge color="orange" size="sm" shape="pill">PUT</Badge>

```text
https://api.criteo.com/preview/retail-media/preferred-line-items/`{line-item-id}`/targeting/stores
```

**Sample Response**

```bash
curl -X PUT "https://api.criteo.com/preview/retail-media/preferred-line-items/2465695028166499188/targeting/stores" \
    -H "Authorization: Bearer myaccesstoken" \
    -H "Content-Type: application/json" \
    -d '{
            "data": {
                "type": "RetailMediaStoreTarget",
                "attributes": {
                    "scope": "exclude",
                    "storeIds": [
                        "store1",
                        "store2"
                    ]
                }
            }
        }'
```

**Sample Request**

```json
{
    "data": {
        "type": "RetailMediaStoreTarget",
        "attributes": {
            "scope": "exclude",
            "storeIds": [
                "store1",
                "store2"
            ]
        }
    },
    "warnings": [],
    "errors": []
}
```
