## **Line Item Attributes**

<ParamField path="id" type="string">
  **Is Nullable?** No

  Line Item ID

  **Values:** int64

  **Default Values:** None

  **Write?** No
</ParamField>

<ParamField path="campaignId" type="string" required>
  **Is Nullable?** No

  Campaign ID

  **Values:** int64

  **Default Value:** None

  **Write?** No
</ParamField>

<ParamField path="name" type="string" required>
  **Is Nullable?** No

  Line item name; must be unique within a [campaign](/retail-media/docs/campaigns)

  **Values:** 255 char limit

  **Default Value:** None

  **Write?** Yes
</ParamField>

<ParamField path="targetRetailerId" type="string" required>
  **Is Nullable?** No

  ID of the [retailer](/retail-media/docs/retailers) the line item serves on

  **Values:** int64

  **Default Value:** None

  **Write?** No
</ParamField>

<ParamField path="startDate" type="date" required>
  **Is Nullable?** No

  Line item start date in the [account](/retail-media/docs/accounts) `timeZone`

  **Values:** YYYY-MM-DD

  **Default Value:** None

  **Write?** Yes
</ParamField>

<ParamField path="endDate" type="date">
  *Optional.*

  **Is Nullable?** Yes

  Line item end date in the [account](/retail-media/docs/accounts) `timeZone`; serves indefinitely if omitted or set to `null`

  **Values:** YYYY-MM-DD

  **Default Value:** `null`

  **Write?** Yes
</ParamField>

<ParamField path="budget" type="number">
  *Optional.*

  **Is Nullable?** Yes

  Line item lifetime spend cap; uncapped if omitted or set to `null`

  **Values:** at least 0

  **Default Value:** `null`

  **Write?** Yes
</ParamField>

<ParamField path="budgetSpent" type="number">
  **Is Nullable?** No

  Amount the line item has already spent

  **Values:** at least 0

  **Default Value:** `0.0`

  **Write?** No
</ParamField>

<ParamField path="budgetRemaining" type="number">
  **Is Nullable?** Yes

  Amount the line item has remaining until cap is hit; `null` if budget is uncapped

  **Values:** between 0 and budget

  **Default Value:** `null`

  **Write?** No
</ParamField>

<ParamField path="monthlyPacing" type="number">
  *Optional.*

  **Is Nullable?** Yes

  Amount the line item can spend per calendar month in the [account](/retail-media/docs/accounts) `timeZone`; resets each calendar month; uncapped if omitted or set to null

  **Values:** at least 0

  **Default Value:** `null`

  **Write?** Yes
</ParamField>

<ParamField path="dailyPacing" type="number">
  *Optional.*

  **Is Nullable?** Yes

  Amount the line item can spend per day in the [account](/retail-media/docs/accounts) timeZone; resets each day; overwritten by a calculation if `isAutoDailyPacing` is configured; uncapped if omitted or set to null

  **Values:** at least 0

  **Default Value:** `null`

  **Write?** Yes
</ParamField>

<ParamField path="isAutoDailyPacing" type="boolean" required>
  **Is Nullable?** No

  To activate, either line item `endDate` and budget, or `monthlyPace`, must be specified; overwrites `dailyPacing` with a calculation if not set prior

  **Values:** `true`, `false`

  **Default Value:** `false`

  **Write?** Yes
</ParamField>

<ParamField path="bidStrategy" type="enum">
  *Optional.*

  **Is Nullable?** No

  Bid algorithm optimizing for sales conversions, sales revenue, or clicks

  **Values:** `conversion`, `revenue`, `clicks`

  **Default Value:** `conversion`

  **Write?** Yes
</ParamField>

<ParamField path="targetBid" type="number" required>
  **Is Nullable?** No

  If optimizing for `conversion` or revenue, a target average amount to bid because each bid is modulated up or down by our optimization algorithm; bids stay constant if optimizing for `clicks`; must meet `minBid` for line item to serve; `minBid` depends on selected products and is retrieved through the [catalog](/retail-media/docs/catalogs); input excludes platform fees

  **Values:** at least the greatest value of minBid across all products on the line item

  **Default Value:** `0.3`

  **Write?** Yes
</ParamField>

<ParamField path="maxBid" type="number">
  *Optional.*

  **Is Nullable?** Yes

  If optimizing for `conversion` or `revenue`, the maximum amount allowed to bid for each bid; respected regardless of `targetBid`; must meet `minBid` for line item to serve; `minBid` depends on selected products and is retrieved through the [catalog](/retail-media/docs/catalogs); bidding is uncapped if omitted or set to `null`; does not apply if optimizing for `clicks`; input excludes platform fees

  **Values:** at least 0.01

  **Default Value:** `null`

  **Write?** Yes
</ParamField>

<ParamField path="status" type="enum">
  **Is Nullable?** No

  Line item status; can only be updated by a user to `active` or `paused`; all other values are applied automatically depending on flight dates, financials, or missing attributes required for line item to serve. To understand the conditions that will cause a status to change, check out our [status page](/retail-media/docs/status).

  **Values:** `active`, `paused`, `scheduled`, `ended`, `budgetHit`, `noFunds`, `draft`, `archived`

  **Default Value:** `None`

  **Write?** No
</ParamField>

<ParamField path="createdAt" type="timestamp">
  **Is Nullable?** No

  Timestamp in UTC of line item creation

  **Values:** ISO-8601

  **Default Value:** `None`

  **Write?** No
</ParamField>

<ParamField path="updatedAt" type="timestamp">
  **Is Nullable?** No

  Timestamp in UTC of last line item update

  **Values:** ISO-8601

  **Default Value:** `None`

  **Write?** No
</ParamField>

## **Create an Open Auction Line Item**

This endpoint creates a new open auction line item in the specified campaign.

<EndpointBadge method="post">
```http
https://api.criteo.com/preview/retail-media/campaigns/{campaignId}/auction-line-items
```
</EndpointBadge>

**Sample Request**

<CodeGroup>

```bash cURL
curl -X POST "https://api.criteo.com/preview/retail-media/campaigns/8343086999167541140/auction-line-items" \
    -H "Authorization: Bearer <MY_ACCESS_TOKEN>" \
    -H "Content-Type: application/json" \
    -d '{
            "data": {
                "type": "RetailMediaLineItem",
                "attributes": {
                    "name": "My New Line Item",
```

## **Update a Specific Open Auction Line Item**

This endpoint updates the specified open auction line item.  In this example, we enable auto daily pacing by setting a monthly pace simultaneously.  Note that with auto daily pacing enabled, daily pacing is automatically calculated and overwrites its previous value, if any.  Also, note the draft state of the line item because products to be promoted have not yet been added.

<EndpointBadge method="put">
```http
https://api.criteo.com/preview/retail-media/auction-line-items/{lineItemId}
```
</EndpointBadge>

**Sample Request**

<CodeGroup>

```bash cURL
curl -X PUT "https://api.criteo.com/preview/retail-media/auction-line-items/2465695028166499188" \
    -H "Authorization: Bearer <MY_ACCESS_TOKEN>" \
    -H "Content-Type: application/json" \
    -d '{
            "data": {
                "type": "RetailMediaLineItem",
                "id": "2465695028166499188",
                "attributes": {
                    "name": "My New Line Item",
                    "startDate": "2020-04-06",
                    "endDate": null,
                    "budget": null,
                    "monthlyPacing": 30.00,
                    "dailyPacing": null,
                    "isAutoDailyPacing": true,
                    "bidStrategy": "conversion",
                    "targetBid": 0.30,
                    "maxBid": null,
                    "status": "active"
                }
            }
        }'
```
