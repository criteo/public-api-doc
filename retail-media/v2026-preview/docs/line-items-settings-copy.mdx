<Note>
  **A FEW THINGS TO KNOW**

  1 -  A *line item* holds *promoted products* to advertise on any single retailer\
  2 -  Line items have basic settings such as start & end dates, optional budget settings & associated retailers on which ads are served\
  3 -  Budgets may additionally be controlled at the campaign level\
  4 -  Several [**reports**](/retail-media/v2025.07/docs/demand-side-analytics-get-started) are available to measure line item performance\
  5 -  Campaigns are limited to 10,000 non-archived line items\
  6 -  Line items are archived automatically 90 days after their end date
</Note>

## **Line Item Attributes**

Status actually appears to be a required parameter as not including it in LI update calls will result in a 400 error

<ParamField path="id" type="string" required>
  **Is Nullable?** No

  Line Item ID

  **Values:** int64

  **Write?** No
</ParamField>

<ParamField path="type" type="enum" required>
  **Is Nullable?** No

  Campaign type

  **Values:** `auction`, `preferred`

  **Write?** No
</ParamField>

<ParamField path="campaignId" type="string" required>
  **Is Nullable?** No

  Campaign ID

  **Values:** int64

  **Write?** No
</ParamField>

<ParamField path="name" type="string" required>
  **Is Nullable?** No

  Line item name; must be unique within a campaign

  **Values:** 255 char limit

  **Write?** Yes
</ParamField>

<ParamField path="targetRetailerId" type="string" required>
  **Is Nullable?** No

  ID of the [retailer](/retail-media/docs/retailers) the line item serves on

  **Values:** int64

  **Write?** No
</ParamField>

<ParamField path="startDate" type="date" required>
  **Is Nullable?** No

  Line item start date in the [account](/retail-media/docs/accounts-endpoints) `timeZone`

  **Values:** YYYY-MM-DD

  **Write?** Yes
</ParamField>

<ParamField path="endDate" type="date">
  *Optional.*

  **Is Nullable?** Yes

  Line item end date in the [account](/retail-media/docs/accounts-endpoints) `timeZone`; serves indefinitely if omitted or set to `null`

  **Values:** YYYY-MM-DD

  **Write?** Yes
</ParamField>

<ParamField path="budget" type="number">
  *Optional.*

  **Is Nullable?** No

  Line item lifetime spend cap; uncapped if omitted or set to `null`

  **Values:** at least 0

  **Write?** No
</ParamField>

<ParamField path="budgetSpent" type="number">
  **Is Nullable?** No

  The amount the line item has already spent

  **Values:** at least 0

  **Write?** No
</ParamField>

<ParamField path="budgetRemaining" type="number">
  **Is Nullable?** Yes

  Amount the line item has to remain until cap is hit; null if budget is uncapped

  **Values:** 0 and `budget`

  **Write?** No
</ParamField>

<ParamField path="status" type="enum">
  **Is Nullable?** No

  Line item status; can only be updated by a user to `active` or `paused`; all other values are applied automatically depending on flight dates, financials, or missing attributes required for line item to serve. To understand the conditions that will cause a status to change, check out our [status page](/retail-media/docs/status)

  **Values:** `active`, `paused`, `scheduled`, `ended`, `budgetHit`, `noFunds`, `draft`, `archived`

  **Write?** Yes
</ParamField>

<ParamField path="createdAt" type="timestamp">
  *Optional.*

  **Is Nullable?** No

  Timestamp in UTC of last line item update

  **Values:** ISO-8601

  **Write?** No
</ParamField>

 

## **Get All Line Items**

This endpoint lists all line items in the specified campaign.  Results are paginated.

<EndpointBadge method="get">
```http
https://api.criteo.com/preview/retail-media/accounts/{accountId}/line-items
```
</EndpointBadge>

**Sample Request** 

<CodeGroup>

```bash cURL
curl -X GET "https://api.criteo.com/preview/retail-media/accounts/123456/line-items" \
    -H "Authorization: Bearer <MY_ACCESS_TOKEN>"
```

```python Python

url = "https://api.criteo.com/preview/retail-media/accounts/4/line-items"

payload={}

  'Accept': 'application/json',
  'Authorization': 'Bearer <MY_ACCESS_TOKEN>'
```
