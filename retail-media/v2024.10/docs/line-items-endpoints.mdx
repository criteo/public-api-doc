2. [Line Items](/retail-media/v2024.10/docs/line-items)

<Info>
  **Getting Started**

  1. A line item holds promoted products to advertise on a single retailer.
  2. Line items include basic settings such as start and end dates, optional budget controls, and the associated retailer where ads are served.
  3. Budgets can also be managed at the campaign level.
  4. Several [reports](/retail-media/v2024.10/docs/demand-side-analytics-get-started) are available to track line item performance.
  5. Campaigns are limited to 10,000 non-archived line items.
  6. Line items are automatically archived 90 days after their end date.
</Info>

## **Endpoints**

| Method  | Endpoint                           | Description                                                 |
| ------- | ---------------------------------- | ----------------------------------------------------------- |
| **GET** | `/accounts/{accountId}/line-items` | Retrieve all line items associated with a specific account. |
| **GET** | `/line-items/{lineItemId}`         | Retrieve details of a specific line item by its ID.         |

## **Line Item Attributes**

Omitting the status field in line item update calls will result in a **400 error**. Ensure that you include the status when updating line items to avoid validation issues.

<ResponseField name="id" required>
**Data Type:** string

**Accepted Value:** int64

**Write?** No

**Nullable?** No

**Description:** Line Item ID
</ResponseField>

<ResponseField name="type" required>
**Data Type:** enum

**Accepted Value:** `auction`, `preferred`

**Write?** No

**Nullable?** No

**Description:** Campaign type
</ResponseField>

<ResponseField name="campaignId" required>
**Data Type:** string

**Accepted Value:** int64

**Write?** No

**Nullable?** No

**Description:** Campaign ID
</ResponseField>

<ResponseField name="name" required>
**Data Type:** string

**Accepted Value:** 255 char limit

**Write?** Yes

**Nullable?** No

**Description:** Line item name; must be unique within a campaign
</ResponseField>

<ResponseField name="targetRetailerId" required>
**Data Type:** string

**Accepted Value:** int64

**Write?** No

**Nullable?** No

**Description:** ID of the [retailer](/retail-media/docs/retailers) the line item serves on
</ResponseField>

<ResponseField name="startDate" required>
**Data Type:** date

**Accepted Value:** YYYY-MM-DD

**Write?** Yes

**Nullable?** No

**Description:** Line item start date in the [account](/retail-media/docs/accounts-endpoints) `timeZone`
</ResponseField>

<ResponseField name="endDate">
**Data Type:** date

**Accepted Value:** YYYY-MM-DD

**Write?** Yes

**Nullable?** Yes

**Description:** Line item end date in the [account](/retail-media/docs/accounts-endpoints) `timeZone`; serves indefinitely if omitted or set to `null`. A timestamp can be included as well if the line item is desired to end at a certain time of day
</ResponseField>

<ResponseField name="budget">
**Data Type:** number

**Accepted Value:** at least 0

**Write?** No

**Nullable?** Yes

**Description:** Line item lifetime spend cap; uncapped if omitted or set to `null`
</ResponseField>

<ResponseField name="budgetSpent">
**Data Type:** number

**Accepted Value:** at least 0

**Write?** No

**Nullable?** No

**Description:** The amount the line item has already spent
</ResponseField>

<ResponseField name="budgetRemaining">
**Data Type:** number

**Accepted Value:** 0 and `budget`

**Write?** No

**Nullable?** Yes

**Description:** Amount the line item has to remain until cap is hit; null if budget is uncapped
</ResponseField>

<ResponseField name="status">
**Data Type:** enum

**Accepted Value:** `active`, `paused`, `scheduled`, `ended`, `budgetHit`, `noFunds`, `draft`, `archived`

**Write?** Yes

**Nullable?** No

**Description:** Line item status; can only be updated by a user to `active` or `paused`; all other values are applied automatically depending on flight dates, financials, or missing attributes required for line item to serve. To understand the conditions that will cause a status to change, check out our [status page](/retail-media/docs/campaign-lineitem-status)
</ResponseField>

<ResponseField name="createdAt">
**Data Type:** timestamp

**Accepted Value:** ISO-8601

**Write?** No

**Nullable?** No

**Description:** Timestamp in UTC of last line item update
</ResponseField>

## **Get All Line Items**

This endpoint lists all line items in the specified campaign. Results are paginated.

<EndpointBadge method="get">
```http
https://api.criteo.com/{version}/retail-media/accounts/{accountId}/line-items
```

## **Responses**

| Response | Description                                                                                                                                                   |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 🟢`200`  | Call completed successfully. The specified line item details are returned.                                                                                    |
| 🔴`403`  | API user is not authorized to make requests for the account ID. To request authorization, follow the[authorization request](/retail-media/v2024.10/docs/authorization-requests)steps. |
| 🔴`404`  | Line item ID not found. Ensure the`lineItemId`is correct and exists.                                                                                          |
