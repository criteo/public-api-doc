## **Endpoints**

| Verb     | Endpoint                                     | Description                                              |
| -------- | -------------------------------------------- | -------------------------------------------------------- |
| **GET**  | `/campaigns/{campaignId}/auction-line-items` | Get all Open Auction Line items from a specific Campaign |
| **POST** | `/campaigns/{campaignId}/auction-line-items` | Creates an Open Auction Line item                        |
| **GET**  | `/auction-line-items/{lineItemId}`           | Get a specific Open Auction Line item                    |
| **PUT**  | `/auction-line-items/{lineItemId}`           | Updates a specific Open Auction Line item                |

<Info>
  ### Info

  * Create operations using the `POST` method expect every **Required (R)** field; omitting **Optional (O)** fields will set those fields to **Default** values
  * Update operations using the `PUT` method expect every **Write (W)** field; omitting these fields is equivalent to setting them to `null`, if possible
</Info>

 

 

## **Line Item Attributes**

id

***

* **Data Type:** string

  **Description:** Line Item ID

  **Accepted Value:** int64

  **Default Values:** None

  **Write?** No

  **Nullable?** No

campaignId *Required*

***

* **Data Type:** string

  **Description:** Campaign ID

  **Accepted Value:** int64

  **Default Value:** None

  **Write?** No

  **Nullable?** No

name *Required*

***

* **Data Type:** string

  **Description:** Line item name; must be unique within a [campaign](/retail-media/docs/campaigns-endpoints)

  **Accepted Value:** 255 char limit

  **Default Value:** None

  **Write?** Yes

  **Nullable?** No

targetRetailerId *Required*

***

* **Data Type:** string

  **Description:** ID of the [retailer](/retail-media/docs/retailers) the line item serves on

  **Accepted Value:** int64

  **Default Value:** None

  **Write?** No

  **Nullable?** No

startDate *Required*

***

* **Data Type:** date

  **Description:** Line item start date in the [account](/retail-media/docs/accounts-endpoints) `timeZone`

  **Accepted Value:** YYYY-MM-DD

  **Default Value:** None

  **Write?** Yes

  **Nullable?** No

endDate *Optional*

***

* **Data Type:** date

  **Description:** Line item end date in the [account](/retail-media/docs/accounts-endpoints) `timeZone`; serves indefinitely if omitted or set to `null`

  **Accepted Value:** YYYY-MM-DD

  **Default Value:** `null`

  **Write?** Yes

  **Nullable?** Yes

budget *Optional*

***

* **Data Type:** number

  **Description:** Line item lifetime spend cap; uncapped if omitted or set to `null`

  **Accepted Value:** at least 0

  **Default Value:** `null`

  **Write?** Yes

  **Nullable?** Yes

budgetSpent

***

* **Data Type:** number

  **Description:** Amount the line item has already spent

  **Accepted Value:** at least 0

  **Default Value:** `0.0`

  **Write?** No

  **Nullable?** No

budgetRemaining

***

* **Data Type:** number

  **Description:** Amount the line item has remaining until cap is hit; `null` if budget is uncapped

  **Accepted Value:** between 0 and budget

  **Default Value:** `null`

  **Write?** No

  **Nullable?** Yes

monthlyPacing *Optional*

***

* **Data Type:** number

  **Description:** Amount the line item can spend per calendar month in the [account](/retail-media/docs/accounts-endpoints) `timeZone`; resets each calendar month; uncapped if omitted or set to null

  **Accepted Value:** at least 0

  **Default Value:** `null`

  **Write?** Yes

  **Nullable?** Yes

dailyPacing *Optional*

***

* **Data Type:** number

  **Description:** Amount the line item can spend per day in the [account](/retail-media/docs/accounts-endpoints) timeZone; resets each day; overwritten by a calculation if `isAutoDailyPacing` is configured; uncapped if omitted or set to null

  **Accepted Value:** at least 0

  **Default Value:** `null`

  **Write?** Yes

  **Nullable?** Yes

isAutoDailyPacing *Required*

***

* **Data Type:** boolean

  **Description:** To activate, either line item `endDate` and budget, or `monthlyPace`, must be specified; overwrites `dailyPacing` with a calculation if not set prior

  **Accepted Value:** `true`, `false`

  **Default Value:** `false`

  **Write?** Yes

  **Nullable?** No

bidStrategy *Optional*

***

* **Data Type:** enum

  **Description:** Bid algorithm optimizing for sales conversions, sales revenue, or clicks

  **Accepted Value:** `conversion`, `revenue`, `clicks`

  **Default Value:** `conversion`

  **Write?** Yes

  **Nullable?** No

targetBid *Required*

***

* **Data Type:** number

  **Description:** If optimizing for `conversion` or revenue, a target average amount to bid because each bid is modulated up or down by our optimization algorithm; bids stay constant if optimizing for `clicks`; must meet `minBid` for line item to serve; `minBid` depends on selected products and is retrieved through the [catalog](/retail-media/docs/catalogs); input excludes platform fees

  **Accepted Value:** at least the greatest value of minBid across all products on the line item

  **Default Value:** `0.3`

  **Write?** Yes

  **Nullable?** No

maxBid *Optional*

***

* **Data Type:** number

  **Description:** If optimizing for `conversion` or `revenue`, the maximum amount allowed to bid for each bid; respected regardless of `targetBid`; must meet `minBid` for line item to serve; `minBid` depends on selected products and is retrieved through the [catalog](/retail-media/docs/catalogs); bidding is uncapped if omitted or set to `null`; does not apply if optimizing for `clicks`; input excludes platform fees

  **Accepted Value:** at least 0.01

  **Default Value:** `null`

  **Write?** Yes

  **Nullable?** Yes

status

***

* **Data Type:** enum

  **Description:** Line item status; can only be updated by a user to `active` or `paused`; all other values are applied automatically depending on flight dates, financials, or missing attributes required for line item to serve. To understand the conditions that will cause a status to change, check out our [status page](/retail-media/docs/campaign-lineitem-status).

  **Accepted Value:** `active`, `paused`, `scheduled`, `ended`, `budgetHit`, `noFunds`, `draft`, `archived`

  **Default Value:** `None`

  **Write?** No

  **Nullable?** No

createdAt

***

* **Data Type:** timestamp

  **Description:** Timestamp in UTC of line item creation

  **Accepted Value:** ISO-8601

  **Default Value:** `None`

  **Write?** No

  **Nullable?** No

updatedAt

***

* **Data Type:** timestamp

  **Description:** Timestamp in UTC of last line item update

  **Accepted Value:** ISO-8601

  **Default Value:** `None`

  **Write?** No

  **Nullable?** No

## **Update a Specific Onsite Sponsored Products Line Item**

This endpoint updates the specified Onsite Sponsored Products line item. In this example, we enable auto daily pacing by setting a monthly pace simultaneously. Note that with auto daily pacing enabled, daily pacing is automatically calculated and overwrites its previous value, if any. Also, note the draft state of the line item because products to be promoted have not yet been added.

![](/images/retail-media/v2024.10/docs/1ee76cc-put.png)

```

```
