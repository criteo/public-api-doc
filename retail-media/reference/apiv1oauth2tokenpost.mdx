---
title: "/oauth2/token"
api: POST https://api.criteo.com/oauth2/token
description: "Creates a token when the supplied client credentials are valid"
contentType: "application/json"
---

## Form data

The clientId and clientSecret used to authenticate

<ParamField path="client_id" type="string">
  Set up your API credentials through our Developer Portal
</ParamField>
<ParamField path="client_secret" type="string">
  Set up your API credentials through our Developer Portal
</ParamField>
<ParamField path="grant_type">
  Must be `client_credentials` or `authorization_code`
</ParamField>

## Responses

### 200 Success

<ResponseField name="Response Body" type="object">
    <ResponseField name="access_token" type="string">
        The access token issued by the authorization server.
    </ResponseField>
    <ResponseField name="token_type" type="string">
        The access token issued by the authorization server.
    </ResponseField>
    <ResponseField name="expires_in" type="int32">
        The lifetime in seconds of the access token.For example, the value "3600" denotes that the access token will expire in one hour from the time the response was generated. If omitted, the authorization server SHOULD provide the expiration time via other means or document the default value.     
        </ResponseField>
</ResponseField>

### 400 Bad request

<ResponseField name="Response Body" type="object">
    <ResponseField name="error" type="string" required>
        The access token issued by the authorization server.
    </ResponseField>
    <ResponseField name="error_description" type="string">
        The access token issued by the authorization server.
    </ResponseField>
</ResponseField>

```bash cURL
     --header 'accept: application/json' \
     --header 'content-type: application/x-www-form-urlencoded' \
     --data grant_type=client_credentials
```
</RequestExample>
