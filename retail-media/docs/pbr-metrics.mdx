## Introduction

In this page, you will find the definition and details of all metrics currently supported in the [Partner Billing Report](/retail-media/docs/partner-billing-report).

***

## Metrics

<table>
  <thead>
    <tr>
      <th>Dimensions</th>
      <th>Data Type</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>AccountName</code></td>
      <td>string</td>
      <td>Account name</td>
    </tr>
    <tr>
      <td><code>AccountId</code></td>
      <td>string</td>
      <td>Account ID, generated by Criteo</td>
    </tr>
    <tr>
      <td><code>IsUnbillable</code></td>
      <td>string</td>
      <td>Flag if report item is billable or not based on the non-billable settings</td>
    </tr>
    <tr>
      <td><code>AccountCategory</code></td>
      <td>string</td>
      <td>The category the account falls under, values can be:<br /><ul><li><code>Supply - Legacy</code></li><li><code>Supply - Private Market</code></li><li><code>Network Demand</code></li><li><code>Private Market Demand - Brand</code></li><li><code>Private Market Demand - Seller</code></li></ul></td>
    </tr>
    <tr>
      <td><code>RetailerName</code></td>
      <td>string</td>
      <td>Retailer name tied to the account</td>
    </tr>
    <tr>
      <td><code>AccountService</code></td>
      <td>string</td>
      <td>Managed service or self-service<br />⚠️ <strong>Note</strong>: if the <code>AccountCategory</code> is Network Demand, the value will be <code>null</code>.</td>
    </tr>
    <tr>
      <td><code>BudgetModel</code></td>
      <td>string</td>
      <td>The value should reflect either “Retailer” or “Criteo” is doing the billing</td>
    </tr>
    <tr>
      <td><code>AccountCurrencyCode</code></td>
      <td>string</td>
      <td>Account currency, in 3-chars code (in ISO-4217 )</td>
    </tr>
    <tr>
      <td><code>Origin</code></td>
      <td>string</td>
      <td>Indicates the source of account creation, distinguishing whether the account was created by a third-party partner working with Criteo; if so, the partner’s name is specified, otherwise, <code>null</code> is expected</td>
    </tr>
    <tr>
      <td><code>OriginId</code></td>
      <td>integer</td>
      <td><code>Origin</code> ID, generated by Criteo</td>
    </tr>
    <tr>
      <td><code>BalanceId</code></td>
      <td>string</td>
      <td>Balance ID associated with the account, generated by Criteo</td>
    </tr>
    <tr>
      <td><code>BalanceName</code></td>
      <td>string</td>
      <td>Balance name<br />⚠️ <strong>Note</strong>: if the <code>AccountCategory</code> is Network Demand, the value will be <code>null</code>.</td>
    </tr>
    <tr>
      <td><code>PurchaseOrder</code></td>
      <td>string</td>
      <td>Purchase order associated to the Balance<br />⚠️ <strong>Note</strong>: if the <code>AccountCategory</code> is Network Demand, the value will be <code>null</code>.</td>
    </tr>
    <tr>
      <td><code>BalanceStartDate</code></td>
      <td>timestamp</td>
      <td>Start date of the Balance, in format <code>yyyy-mm-ddThh:mm:ss.sTZD</code><br />⚠️ <strong>Note</strong>: if the <code>AccountCategory</code> is Network Demand, the value will be <code>null</code>.</td>
    </tr>
    <tr>
      <td><code>PrivateMarketPaymentOption</code></td>
      <td></td>
      <td>The payment option of the Private Market demand account at the balance level<br />⚠️ <strong>Note</strong>: if the <code>AccountCategory</code> is Network Demand, the value will be <code>null</code>.</td>
    </tr>
    <tr>
      <td><code>CampaignId</code></td>
      <td>string</td>
      <td>Campaign ID, generated by Criteo</td>
    </tr>
    <tr>
      <td><code>CampaignName</code></td>
      <td>string</td>
      <td>Campaign name</td>
    </tr>
    <tr>
      <td><code>CampaignType</code></td>
      <td>string</td>
      <td>Campaign type (⚠️ Campaigns eg: <code>SponsoredProducts</code> )</td>
    </tr>
    <tr>
      <td><code>BuyType</code></td>
      <td>string</td>
      <td>Campaign buying type (e.g. Auction, Preferred Deals, Sponsorship)</td>
    </tr>
    <tr>
      <td><code>LineItemId</code></td>
      <td>string</td>
      <td>Line item ID, generated by Criteo</td>
    </tr>
    <tr>
      <td><code>LineItemName</code></td>
      <td>string</td>
      <td>Line item name</td>
    </tr>
    <tr>
      <td><code>WorkingMediaSpend</code></td>
      <td>decimal</td>
      <td>Working media spend</td>
    </tr>
    <tr>
      <td><code>CreditSpend</code></td>
      <td>decimal</td>
      <td>Credit spend is the amount of credit applied in a given month to restore a working media budget that was originally impacted by bots, technical errors, or human errors.</td>
    </tr>
    <tr>
      <td><code>ValueAddSpend</code></td>
      <td>decimal</td>
      <td>Value add spend is the amount of free budget applied in a given month, typically granted to brands or agencies as an incentive to spend above a specific threshold, invest in new retailers, expand into new product categories, or test different formats and audiences.</td>
    </tr>
    <tr>
      <td><code>Clicks</code></td>
      <td>integer</td>
      <td>Amount of billable clicks recorded</td>
    </tr>
    <tr>
      <td><code>Impressions</code></td>
      <td>integer</td>
      <td>Amount of billable impressions recorded</td>
    </tr>
    <tr>
      <td><code>SupplyFeeAmount</code></td>
      <td>decimal</td>
      <td>Retailer's SSP fee amount</td>
    </tr>
    <tr>
      <td><code>RetailerManagedFeeRate</code></td>
      <td>decimal</td>
      <td>Managed service fee set at the supply account, which applies to the demand side of the supply account</td>
    </tr>
    <tr>
      <td><code>RetailerManagedFeeAmount</code></td>
      <td>decimal</td>
      <td>Retailer's managed service fee amount</td>
    </tr>
    <tr>
      <td><code>RetailerAudienceDataFeeAmount</code></td>
      <td>decimal</td>
      <td>Retailer's offsite audience revenue before Criteo's take rate</td>
    </tr>
    <tr>
      <td><code>CriteoRetailerDataFeeAmount</code></td>
      <td>decimal</td>
      <td>Criteo's take rate from retailer offsite audience revenue</td>
    </tr>
    <tr>
      <td><code>UtcOffset</code></td>
      <td>string</td>
      <td>Timezone offset of the account</td>
    </tr>
    <tr>
      <td><code>SalesEnablementServiceFeeAmount</code></td>
      <td>decimal</td>
      <td>Retailer's sales enablement professional service fee amount</td>
    </tr>
    <tr>
      <td><code>SelfServiceEnablementServiceFeeAmount</code></td>
      <td>decimal</td>
      <td>Retailer's self-service enablement professional service fee amount</td>
    </tr>
  </tbody>
</table>
