<Note>
  ### Getting Started

  * Preferred Deals Line Items have an associated Creative and Creative Template that defines the requirements for **SkuCollections**. 

  * SkuCollections, also known as **Product Buttons**, is a many-to-one relationship with the LineItem. Each SKU Collection has some attributes including one or more SKUs.

  * **NOTE:** In this documentation, we will refer to these endpoints as **Product SKU Collections**
</Note>

## **Product SKU Collections Parameters**

<ParamField path="line-item-id" type="string">
  **Value:** int64

  **Is Nullable?** No

  Long external id of the associated line item
</ParamField>

<ParamField path="product-button-id" type="integer">
  **Value:** int32

  **Is Nullable?** No

  Sequential id of the product button
</ParamField>

## **Product SKU Collections Attributes**

<ParamField path="id" type="string" required>
  *Required for PUT/POST calls.*

  **Value:** -

  **Is Nullable?** computed

  Sequence id of the Product Button
</ParamField>

<ParamField path="type" type="string">
  **Value:** -

  **Is Nullable?** computed

  Type of entity
</ParamField>

<ParamField path="name" type="string" required>
  **Value:** -

  **Is Nullable?** No

  Product Button Display Name
</ParamField>

<ParamField path="backgroundImageUrl" type="string" required>
  **Value:** -

  **Is Nullable?** No

  URL to the SKU background image
</ParamField>

<ParamField path="rank" type="integer" required>
  **Value:** int32

  **Is Nullable?** No

  Rank of the Product Button
</ParamField>

<ParamField path="isMandatory" type="boolean" required>
  **Value:** `true`, `false`

  **Is Nullable?** No

  If the Product Button Is Mandatory for the ad to serve
</ParamField>

<ParamField path="skus" type="string" required>
  **Value:** -

  **Is Nullable?** No

  List of SKUs on the product button
</ParamField>

## **Create Product SKU Collection to Line Item**

Create a new product SKU collection on a line item

<Info>
  **Initializing Product SKU on Line Item**

  Before creating a new SKU collection, make sure that the SKU has been added to the line item using the [**Add Products to a Specific Line Item**](/retail-media/docs/promoted-products#add-products-to-a-specific-line-item-or-update-a-bid-override) endpoint
</Info>

<Badge color="red" size="sm" shape="pill">POST</Badge>

## **Update Line Item Product SKU Collection**

Updates the product SKU collection associated with a line item. 

<Info>
  **When Adding New SKUs to an Existing Collection**

  The **PUT** method updates all data in the resources. When updating an existing collection to add new SKUs, make sure to also include the list of the existing SKUs if you wish to avoid overriding previously added SKUs to the collection.
</Info>

<Badge color="orange" size="sm" shape="pill">PUT</Badge>

```http
https://api.criteo.com/{version}/retail-media/line-items/{line-Item-Id}/product-buttons/{product-button-id}
```
