## Introduction

- To get started with our APIs, use the endpoint below to generate an Access Token with your API credentials.
- The <Tooltip tip="access token - is a credential issued by an authentication server that grants a client application permission to access specific resources on behalf of a user or system. It is a short-lived, encoded string containing information such as permissions (scopes) and expiration time, used in API requests to validate the client's identity and authorization level.">access token</Tooltip> is a Bearer token to be included in the Authorization Header of all API requests.
- Multiple tokens may be generated, each valid for 15 minutes or 900 seconds.

## Endpoint

```bash
POST https://api.criteo.com/oauth2/token
```

<Note>
**Note**

Generate another access token when it expires, signaled by a `401 Unauthorized` HTTP status code
</Note>

<ParamField body="client_id" type="string">
  Set up your API credentials through our Developer Portal
</ParamField>
<ParamField body="client_secret" type="string">
  Set up your API credentials through our Developer Portal
</ParamField>
<ParamField body="grant_type" type="string">
  Must be `client_credentials` or `authorization_code`
</ParamField>


For more info, check [OAuth App Implementation](https://developers.criteo.com/retail-media/docs/oauth-app-implementation)

## Generate an Access Token
    - This endpoint generates a new access token using your API credentials

```bash
POST https://api.criteo.com/oauth2/token
```

<ResponseExample>
```json JSON
{
    "access_token": "<TOKEN STRING>",
    "token_type": "Bearer",
    "expires_in": 900
}
```
</ResponseExample>
