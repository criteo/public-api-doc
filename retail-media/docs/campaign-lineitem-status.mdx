## Statuses

<table>
  <thead>
    <tr>
      <th>Level</th>
      <th>Available Status</th>
      <th>Read Only Status</th>
      <th>Is Mutable?</th>
      <th>Conditions</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Campaign</strong></td>
      <td><code>active</code><br /><code>inactive</code></td>
      <td><code>active</code><br /><code>inactive</code></td>
      <td>No</td>
      <td>Campaign statuses will automatically change to active if at least one line item is <strong><code>active</code></strong>. All other line-item statuses will result in the campaign to be <strong><code>inactive</code></strong></td>
    </tr>
    <tr>
      <td><strong>Line Items</strong></td>
      <td><code>active</code><br /><code>paused</code><br /><code>scheduled</code><br /><code>ended</code><br /><code>budgetHit</code><br /><code>noFunds</code><br /><code>draft</code><br /><code>archived</code></td>
      <td><code>scheduled</code><br /><code>ended</code><br /><code>budgetHit</code><br /><code>noFunds</code><br /><code>draft</code><br /><code>archived</code></td>
      <td>Yes, line items statuses can be updated through the API to <strong><code>active</code></strong> or <strong><code>paused</code></strong></td>
      <td>Line item will deliver if the status is <code>active</code>, while all other values are applied automatically depending on flight dates, financials, or missing attributes required for line item to serve.<br />The following line-item status will result in a campaign status to be inactive: <strong><code>paused</code>,<code>scheduled</code>, <code>ended</code>, <code>budgetHit</code>, <code>noFunds</code>, <code>draft</code>, <code>archived</code></strong></td>
    </tr>
    <tr>
      <td><strong>Products</strong></td>
      <td><code>active</code><br /><code>inactive</code></td>
      <td>Not entirely - <strong><code>active</code></strong> and <strong><code>paused</code></strong> statuses are read only with most promoted products endpoints. However, they can be directly updated using the pause and unpause endpoints.</td>
      <td>Yes, with the <code>pause</code> and <code>unpause</code> endpoints<br />/line-items/&#92;&#123;LineItemId&#125;/products/pause directly updates promoted product status to <strong><code>paused</code></strong><br />/line-items/&#92;&#123;LineItemId&#125;/products/unpause directly updates promoted product to <strong><code>active</code></strong></td>
      <td>Pausing products SKU’s on a line item will not change the status of a campaign or line item</td>
    </tr>
  </tbody>
</table>
