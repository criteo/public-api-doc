---
title: Sync Attributed Transactions Report
description: POST /reports/sync/attributed-transactions — synchronous transaction-level attribution report for Retail Media DSP partners.
---

This endpoint returns a transaction-level log of every purchase event attributed to your Retail Media DSP campaigns — synchronously, with no polling required.

Each row in the response represents a single attributed transaction and captures the product advertised, the product purchased, the attribution rule applied, and the time elapsed between ad delivery and purchase.

<Info>
  **This is not a performance report.** Aggregated metrics — impressions, clicks, spend, ROAS — are not available here. Use [`POST /reports/sync/campaigns`](/retail-media/docs/sync-campaigns-report) or [`POST /reports/sync/line-items`](/retail-media/docs/sync-line-items-report) for those.
</Info>

**Primary audience:** Data scientists and analysts auditing attribution methodology, understanding cross-sell and halo effects, or analyzing purchase behavior at the transaction level.

***

## Request

```http
POST /2026-07/retail-media/reports/sync/attributed-transactions
```

### Required fields

| Field | Type | Description |
|-------|------|-------------|
| `accountId` | string | Account ID to report on. |
| `startDate` | string | Start of the reporting period. ISO 8601 date (`YYYY-MM-DD`). Based on ad delivery date (`advDate`). |
| `endDate` | string | End of the reporting period. ISO 8601 date (`YYYY-MM-DD`). |

### Optional fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `campaignIds[]` | array of strings | — | Filter to specific campaign IDs. |
| `lineItemIds[]` | array of strings | — | Filter to specific line item IDs. |
| `campaignType` | string | `all` | Filter by campaign type: `sponsoredProducts`, `onSiteDisplays`, or `all`. |
| `mediaType` | string | `all` | Filter by media type: `display`, `video`, `unknown`, or `all`. |
| `salesChannel` | string | `all` | Filter by sales channel: `online`, `offline`, or `all`. |
| `clickAttributionWindow` | string | `none` | Post-click attribution window: `none`, `7D`, `14D`, or `30D`. |
| `viewAttributionWindow` | string | `none` | Post-view attribution window: `none`, `1D`, `7D`, `14D`, or `30D`. |
| `dimensions[]` | array of strings | — | Fields to include in the output. See [Dimensions](#dimensions) below. |
| `metrics[]` | array of strings | — | Metrics to include: `attributedSales`, `attributedUnits`. |
| `timezone` | string | `UTC` | Timezone for date fields. IANA format (e.g. `America/New_York`). |

### Example request

```http
POST /2026-07/retail-media/reports/sync/attributed-transactions
Authorization: Bearer {token}
Content-Type: application/json

{
  "data": {
    "type": "RetailMediaReportRequest",
    "attributes": {
      "accountId": "123456",
      "startDate": "2026-07-01",
      "endDate": "2026-07-07",
      "clickAttributionWindow": "14D",
      "viewAttributionWindow": "1D",
      "dimensions": [
        "advDate", "campaignId", "campaignName",
        "advProductId", "advProductName",
        "purchasedProductId", "purchasedProductName",
        "advToPurchasedProductRelationship", "salesChannel"
      ],
      "metrics": ["attributedSales", "attributedUnits"]
    }
  }
}
```

## Metrics

| Metric | Description |
|--------|-------------|
| `attributedSales` | Revenue attributed to this transaction in the account's reporting currency. |
| `attributedUnits` | Units sold in this attributed transaction. |

## Dimensions

### Ad delivery fields

| Dimension | Description |
|-----------|-------------|
| `advDate` | Date the ad was delivered (`YYYY-MM-DD`). |
| `advHour` | Hour the ad was delivered (0–23). |
| `advEngagement` | Type of engagement that triggered attribution: `imp` (impression) or `click`. |

### Attribution fields

| Dimension | Description |
|-----------|-------------|
| `attributionWindow` | Lookback window applied to this transaction (e.g. `C14V01`). |
| `daysDifference` | Days between ad delivery and the attributed purchase. |
| `advToPurchasedProductRelationship` | Attribution rule connecting the ad to the purchase: `same sku`, `same parent sku`, `same taxonomy`, `same brand`, or `same seller`. |

### Advertised product fields

| Dimension | Description |
|-----------|-------------|
| `advProductId` | ID of the product that was advertised. References the Catalogs product ID. |
| `advProductName` | Name of the advertised product. |
| `advProductCategory` | Category of the advertised product. |
| `advProductGtin` | GTIN/EAN/UPC of the advertised product, if available. |
| `advProductMpn` | Manufacturer Part Number of the advertised product, if available. |

### Purchased product fields

| Dimension | Description |
|-----------|-------------|
| `purchasedDate` | Date the attributed purchase occurred (`YYYY-MM-DD`). |
| `purchasedHour` | Hour the attributed purchase occurred (0–23). |
| `purchasedProductId` | ID of the product that was purchased. |
| `purchasedProductName` | Name of the purchased product. |
| `purchasedProductCategory` | Category of the purchased product. |
| `purchasedProductGtin` | GTIN/EAN/UPC of the purchased product, if available. |
| `purchasedProductMpn` | Manufacturer Part Number of the purchased product, if available. |

### Context fields

| Dimension | Description |
|-----------|-------------|
| `campaignId` | Campaign ID. |
| `campaignName` | Campaign name. |
| `lineItemId` | Line item ID. |
| `lineItemName` | Line item name. |
| `retailerName` | Retailer where the line item served. |
| `pageTypeName` | Retailer page type where the ad rendered. |
| `keyword` | Search keyword on the page where the ad rendered, if applicable. |
| `salesChannel` | Purchase channel: `online` or `offline`. |
| `activitySellerId` | ID of the seller responsible for the advertising event. |
| `activitySellerName` | Name of the seller responsible for the advertising event. |
| `saleSellerId` | ID of the seller responsible for the sale. |
| `saleSellerName` | Name of the seller responsible for the sale. |

## Understanding advertised vs. purchased product fields

The advertised product (`advProduct*`) and purchased product (`purchasedProduct*`) are often the same — but not always. When the attribution rule is `same brand`, `same taxonomy`, or `same seller`, the ad may have driven a purchase of a related product rather than the exact item advertised.

**Example:** An ad for a camping jacket is served. The shopper purchases a different jacket from the same brand. The row will show:
- `advProductId`: the advertised jacket's ID
- `purchasedProductId`: the purchased jacket's ID
- `advToPurchasedProductRelationship`: `same brand`

This bilateral product structure makes this endpoint useful for cross-sell and halo analysis — you can see exactly which advertised products drove purchases of which other products, and under which attribution rule.
