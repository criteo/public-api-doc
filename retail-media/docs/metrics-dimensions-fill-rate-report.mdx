## Introduction

In this page, you will find all the metrics and dimensions that are currently supported in the reports for Supply Side accounts:

* [Revenue Report](/retail-media/docs/revenue-report-ssp): provides attributed performance data from campaigns.
* [Fill Rate Report](/retail-media/docs/fill-rate-report): provides summary of placement opportunities and impression filled and covered in retailer(s) inventory.

<Info>
  Please note that some of the metrics and dimensions listed here are currently supported only through the API.
</Info>

***

# Fill Rate Report

      <td><code>retailerName</code></td>
      <td>The retailer name that is associated to the report activity.</td>
    </tr>
    <tr>
      <td><code>environment</code></td>
      <td>The environment where the ad was rendered on. Options will include web, mobile and app</td>

    <tr>
      <td><code>pageTypeName</code></td>
      <td>The page type where the ad was rendered</td>
    </tr>
    <tr>

      <td>The <code>id</code> of the placement where the ad creative was displayed.</td>
    </tr>
    <tr>
      <td><code>placementName</code></td>
      <td>The name of the placement where the ad creative was displayed.</td>
    </tr>
    <tr>

      <td><code>retailerCategoryId</code></td>

    </tr>

    </tr>

## Metrics

<table>
  <thead>
    <tr>
      <th>Dimensions</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>availablePlacements</code></td>
      <td>The number of available placements that were available to fill, including non-deliverable and deliverable placements.</td>
    </tr>
    <tr>
      <td><code>clicks</code></td>
      <td>The total count of all click events<br /><strong>Formula</strong>: <code>placementClicks</code> + <code>productClicks</code></td>
    </tr>
    <tr>
      <td><code>coverageRate</code></td>
      <td>The number of covered placements divided by the number of deliverable placements</td>
    </tr>
    <tr>
      <td><code>coveredPlacements</code></td>
      <td>A placement with candidates where Criteo has provided SKUs or line-items in the ad response</td>
    </tr>
    <tr>
      <td><code>cpc</code></td>
      <td>Average cost-per-click<br /><strong>Formula</strong>: <code>openAuctionRevenue</code> / <code>clicks</code> or <code>preferredDealsRevenue</code> / <code>clicks</code></td>
    </tr>
    <tr>
      <td><code>cpm</code></td>
      <td>Average cost per &#95;mille&#95;, calculated by:<br /><strong>Formula</strong>: <code>preferredDealsRevenue</code> / <code>impressions</code> &#42; 1000</td>
    </tr>
    <tr>
      <td><code>deliverablePlacements</code></td>
      <td>Available placement that the retailer allowed Criteo to run an auction for</td>
    </tr>
    <tr>
      <td><code>fillRate</code></td>
      <td>The number of times a deliverable placement on a page was populated with an ad (even if the impression was not viewable). Calculated by dividing the placement impressions by deliverable placements.<br /><strong>Formula</strong>: <code>placementImpressions</code> /<code>deliverablePlacements</code></td>
    </tr>
    <tr>
      <td><code>impressions</code></td>
      <td>An impression represents each time an ad renders on a page, regardless of clicks or views.<br />An impression for a Sponsored Product ad is when the sponsored SKU renders on the page (the product being the ad).<br />An impression for a Commerce Display ad is when the entire banner (creative + product) renders on the page.<br />An impression for a display banner ad is when the creative banner renders on the page.</td>
    </tr>
    <tr>
      <td><code>netRevenue</code></td>
      <td>The total revenue from brand spend for the retailer</td>
    </tr>
    <tr>
      <td><code>nonDeliverablePlacements</code></td>
      <td>A non-deliverable placement is a placement against which Criteo is not allowed by the retailer to run an auction.</td>
    </tr>
    <tr>
      <td><code>pageViews</code></td>
      <td>The count of unique ad calls on a retailer network. Use this metric when referencing to dimensions that relates to itself to receive a more comprehensive data. Using dimensions such as <code>pageTypeName</code>, <code>placementId</code>or <code>placementName</code>will provide you data to understand the performance of this metric.</td>
    </tr>
    <tr>
      <td><code>placementClicks</code></td>
      <td>Total count of the click events that occur when a user clicks a placement</td>
    </tr>
    <tr>
      <td><code>placementImpressions</code></td>
      <td>The number of placement impressions.</td>
    </tr>
    <tr>
      <td><code>placementImpressionsCTR</code></td>
      <td>The percentage of shoppers who clicked an ad placement rendered on a page<br /><strong>Formula</strong>: (<code>placementClicks</code> + <code>productClicks</code>) / <code>placementImpressions</code></td>
    </tr>
    <tr>
      <td><code>placementImpressionsRevenue</code></td>
      <td>The total revenue generated by placement impressions of CPM ads</td>
    </tr>
    <tr>
      <td><code>placementsWithCandidates</code></td>
      <td>A placement with candidates is a placement for which Criteo is allowed by the retailer to run an auction and has sufficient information and demand to run it.</td>
    </tr>
    <tr>
      <td><code>productClicks</code></td>
      <td>Total count of the click events that occur when a user clicks a product in a placement</td>
    </tr>
    <tr>
      <td><code>productClicksRevenue</code></td>
      <td>The total revenue generated by product clicks of CPC ads</td>
    </tr>
    <tr>
      <td><code>productImpressions</code></td>
      <td>The number of filled product impressions</td>
    </tr>
    <tr>
      <td><code>productImpressionsCTR</code></td>
      <td>The number of clicks divided by the number of product impressions<br /><strong>Formula</strong>: <code>productClicks</code>/<code>productImpressions</code></td>
    </tr>
    <tr>
      <td><code>revenue</code></td>
      <td>The total revenue generated on the set date. The <code>revenue</code> metric takes the total revenue generated across preferred deals or auction campaigns for that period and is computed the same way as the <code>openAuctionRevenue</code> and <code>preferredDealsRevenue</code></td>
    </tr>
    <tr>
      <td><code>unfilledPlacements</code></td>
      <td>The number of unfilled placements</td>
    </tr>
    <tr>
      <td><code>workingMedia</code></td>
      <td>The total spend paid by the brand</td>
    </tr>
  </tbody>
</table>

### Fill Rate Unfilled Reasons

<table>
  <thead>
    <tr>
      <th>Reason</th>
      <th>Definition</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Uncovered</strong></td>
      <td></td>
    </tr>
    <tr>
      <td><code>totalUnfilledPlacements</code></td>
      <td>The number of available ad placements that did not serve any impressions.</td>
    </tr>
    <tr>
      <td><code>unfilledNotEnoughDemand</code></td>
      <td>There is not enough line items eligible to the placement for run of site placements.</td>
    </tr>
    <tr>
      <td><code>uncoveredBrokenPlacement</code></td>
      <td>Ads returned to the retailer, but on placements that the retailer rarely paints. If a placement has a painted rate of less than 1% over the last 7 days, it can be considered as broken.</td>
    </tr>
    <tr>
      <td><code>uncoveredFilteredOutDemand</code></td>
      <td>Filtering reasons are the set of reasons for which line items can and will be excluded from pre-auctions, which decreases the number of ad opportunities overall. Filtering reasons may be of technical, supply, or demand nature (out of stock, dayparting,<br />audience targeting, capping…)</td>
    </tr>
    <tr>
      <td><code>uncoveredNoDemandBrandedKeywordConquestingDisabled</code></td>
      <td>Available opportunities without candidate campaigns, that happen on branded keywords on retailers that disabled conquesting (on that particular placement). No demand on branded keyword conquesting disabled</td>
    </tr>
    <tr>
      <td><code>uncoveredNoDemandBrandedKeywordConquestingEnabled</code></td>
      <td>Available opportunities without candidate campaigns, that happen on branded keywords on retailers that enabled conquesting (on that particular placement)</td>
    </tr>
    <tr>
      <td><code>uncoveredNoDemandUnbrandedInventory</code></td>
      <td>Ad placements available on general (non-branded) inventory, but no campaigns are set up to use them</td>
    </tr>
    <tr>
      <td><code>uncoveredNotPainted</code></td>
      <td>Ads returned to the retailer but not displayed (on placements that aren't broken).</td>
    </tr>
    <tr>
      <td><code>uncoveredSearchTermWithoutCategory</code></td>
      <td>Ad placements on keywords that the keyword model cannot<br />categorize.</td>
    </tr>
    <tr>
      <td><strong>Undeliverable</strong></td>
      <td></td>
    </tr>
    <tr>
      <td><code>nondeliverableBlockedOptOut</code></td>
      <td>Ad placements for opt-out users that are not delivering because you have not enabled ads for these users</td>
    </tr>
    <tr>
      <td><code>nonDeliverableBlockedPageCategory</code></td>
      <td>Ad placements on pages where you have blocked Criteo from running auctions (e.g., some categories on which you do not want any ads to be served)</td>
    </tr>
    <tr>
      <td><code>nonDeliverableInsufficientOrganicResults</code></td>
      <td>Ad placements where the page does not have enough organic results to display ads. For example, if an ad is meant to appear on line 5 but fewer than 5 lines of products exist.</td>
    </tr>
    <tr>
      <td><code>nonDeliverableTestPlacement</code></td>
      <td>These are ad placements that are still in testing and not fully enabled.</td>
    </tr>
    <tr>
      <td><code>nonDeliverableUnmappedCategories</code></td>
      <td>Ad placements on category and merchandising pages that aren't deliverable because their category keys are missing from the retailer’s product feed.</td>
    </tr>
    <tr>
      <td><code>nonDeliverablePagesWithUnknownProducts</code></td>
      <td>Product pages where items can't be identified in the retailer's product feed.</td>
    </tr>
