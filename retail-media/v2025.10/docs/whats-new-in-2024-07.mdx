<Note>
  **Version Schedule**

  Version 2024.07 of the Criteo Retail Media API is live as of **July 23rd, 2024**. This version will be supported until **July 31st, 2025**.

  Following this new version release, version 2023.07 will lose support starting **July 31st, 2024**
</Note>

This version introduces new capabilities to Criteo Retail Media API and changes to existing endpoints. This page aims at listing all changes that happened endpoint per endpoint. Only endpoints with changes are listed here.

# New Base URL

With the introduction of this new version, the base URL will now change from the

**Old base URL**

```http
https://api.criteo.com/2024-04/retail-media/{endpoint}
```

to

**New base URL**

```http
https://api.criteo.com/2024-07/retail-media/{endpoint}
```

# What's New

In version 2024.07, we've introduced a new endpoint. This endpoint allows you to get a full list of all changes or updates made to a given balance/insertion order once the ID is supplied.

### Balance History

We have released a new endpoint to our balances API. The endpoint allows users to view the history of all modifications made to a balance. More information now available in the [Balances](/retail-media/docs/balances) guide.

### New Reporting Dimension

In the demand side reporting, a new dimension has been introduced: `searchTerm` indicating the precise keyword or phrase a shopper used while searching on a retailer's website. For additional details on this dimension, please refer to the [Metrics and Dimensions](/retail-media/docs/metrics-and-dimensions) page.

_Note_ The new dimension can be used alongside the existing `keyword` dimension which is the normalized keyword submitted by the advertisers. Criteo also refers to `keyword` as the **stemmed search term** which is how the dimension is referred to in our user interface.
