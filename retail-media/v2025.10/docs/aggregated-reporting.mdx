<Note>
  ### Overview

  * The reporting accounts endpoint offers an `accountId []` array, accommodating up to 10 private market demand accounts. Depending on the chosen aggregation level, the report will compile a comprehensive list of all line items or campaigns associated with each specified private market account ID
  * By utilizing the `aggregationLevel` attribute, you can precisely determine whether the report should aggregate data at the campaign or line-item level
  * Furthermore, the endpoint grants flexibility with a selectable interval of 31 days between the designated start and end dates
</Note>

<ParamField path="clickAttributionWindow" type="string" required>
  The post-click attribution window, defined as the maximum number of days considered between a click and a conversion for attribution; conversions are attributed to the date of conversion, not the date of click; defaults to campaign settings if omitted; must be specified if `viewAttributionWindow` is one of the accepted values.

  **Values:** `7D`, `14D`,`30D`
</ParamField>

<ParamField path="viewAttributionWindow" type="string" required>
  The post-view attribution window, defined as the maximum number of days considered between an impression and a conversion for attribution; conversions are attributed to the date of conversion, not the date of impression; defaults to campaign settings if omitted; must be less than or equal to `clickAttributionWindow`; must be specified if `clickAttributionWindow` is one of the accepted values.

  **Values:** none, `1D`,`7D`, `14D`, `30D`
</ParamField>

<ParamField path="salesChannel" type="string">
  Filter on specific sales channel: offline or online

  **Values:** `offline`, `online`
</ParamField>

<ParamField path="format" type="string">
  *Optional.*

  Format of the report data returned

  **Values:** json-compact, json-newline, json (default), csv
</ParamField>

## **Create a Report Request**

<Badge color="red" size="sm" shape="pill">POST</Badge>

```text
https://api.criteo.com/{version}/retail-media/reports/async/accounts
```

**Sample Request** 

```bash
curl -X 'POST' \
  'https://api.criteo.com/{version}/retail-media/reports/accounts' \
  -H 'accept: text/plain' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer &lt;TOKEN&gt;' \
  -d '{
  "data": {
    "type": "Reporting",
    "attributes": {
      "accountIds": ["449584817843912704","451708028784234496","452114665261600768", 
      "454939415980695552","455008664277487616","456032104520957952","456794723642474496", 
      "456794968072671232","456795173615796224","456795380471586816"
      ],
      "aggregationLevel": "campaign",
      "campaignType": "sponsoredProducts",
      "salesChannel": "offline",
      "format": "json-compact",
      "clickAttributionWindow": "30D",
      "viewAttributionWindow": "7D",
      "dimensions": [
        "date", "campaignId", "accountId", "accountName","lineItemId","lineItemName"
      ],
      "metrics": [
        "impressions"
      ],
      "startDate": "2024-01-01",
      "endDate": "2024-02-01",
      "timezone": "UTC"
    }
  }
}'
```

## **Get Status of a Specific Report**

This endpoint retrieves the status of a specific report.  Status can be `pending`, `success`, `failure`, or `expired`

<Badge color="blue" size="sm" shape="pill">GET</Badge>

```text
https://api.criteo.com/{version}/retail-media/reports/{reportId}/status
```

**Sample Request**

<CodeGroup>

```bash cURL
curl -X GET "https://api.criteo.com/{version}/retail-media/reports/2e733b8c-9983-4237-aab9-17a42f4267cb/status" \
    -H "Authorization: Bearer <MY_ACCESS_TOKEN>"
```

