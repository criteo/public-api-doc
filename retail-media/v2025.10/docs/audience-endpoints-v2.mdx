<Note>
  ### Getting Started

  An _audience_ represents an aggregation of all users who interacted with specific brands or categories over a defined period. Audiences are used only with preferred deals line items with the audience type options of delivering audiences based on a **customerList** or **userBehavior**. Additionally, in Audience V2, there is now an option to specify the user segments to _"include"_ or _"exclude"_ in your user segment targeting.
</Note>

## **Audience Attributes**

#### `id` Required

- **Data Type:** string
- **Values:** int64

**Description:** Audience ID

#### `type` Required

- **Data Type:** enum
- **Values:** `customerList`, `userBehaviour`

**Description:** Type of the audience

#### `name` Required

- **Data Type:** string
- **Values:** -

**Description:** Audience name

#### `userAction` Required

- **Data Type:** enum
- **Values:** `viewed`, `buy`, `addToCart`

**Description:** User behavior that is targeted

#### `lookbackWindow` Required

- **Data Type:** enum
- **Values:** `P7D`, `P14D`, `P30D`, `P45D`, `P60D`, `P90D`, `P120D`, `P150D`, `P180D`

**Description:** Length of the loopback window, in days

#### `brandIds` Optional

- **Data Type:** int array
- **Values:** list of brand ids

**Description:** Brand ids to target
- **Note:** Either categoryId or brandIds must be specified in the call when creating new audiences

#### `categoryIds` Optional

- **Data Type:** int array
- **Values:** list of category ids

**Description:** Category ids to target.
- **Note:** Either categoryId or brandIds must be specified in the call when creating new audiences

#### `retailerId` Required

- **Data Type:** string
- **Values:** -

**Description:** Id of the retailer associated with the audience

#### `userBehaviorDetails` Required

- **Data Type:** object
- **Values:** UserBehaviorDetails

**Description:** Details related to the user behavior audience.

#### `inclusiveSegment` Required

- **Data Type:** object
- **Values:** UserBehaviorAudienceSegment

**Description:** Details related to inclusive user behavior audience

#### `exclusiveSegment` Optional

- **Data Type:** object
- **Values:** UserBehaviorAudienceSegment

**Description:** Details related to exclusive user behavior audience. Can only be specified if inclusiveSegment is specified

#### `minPrice` Optional

- **Data Type:** double
- **Values:** -

**Description:** The min price of targeted skus.

#### `minPrice` Optional

- **Data Type:** double
- **Values:** -

**Description:** The max price of targeted skus.

#### `customerListDetails` Optional

- **Data Type:** object
- **Values:** customerListDetails

**Description:** Details related to the customer list audience

#### `size` Required

- **Data Type:** string
- **Values:** int64

**Description:** Number of users in the customer list audience.

## **Get All Audiences by Account Id**

<Badge color="blue" size="sm" shape="pill">GET</Badge>

```text
https://api.criteo.com/{version}/retail-media/v2/accounts/{accountId}/audiences
```

**Sample Request**

<CodeGroup>

```bash cURL
curl -L -X GET 'https://api.criteo.com/{version}/retail-media/v2/accounts/5/audiences?pageSize=3' \
-H 'Accept: application/json' \
-H 'Authorization: Bearer <MY_ACCESS_TOKEN>'
```

```python Python
import http.client

conn = http.client.HTTPSConnection("api.criteo.com")
payload = ''
headers = {
  'Accept': 'application/json',
  'Authorization': 'Bearer <MY_ACCESS_TOKEN>'
}
conn.request("GET", "/{version}/retail-media/v2/accounts/5/audiences?pageSize=3", payload, headers)
res = conn.getresponse()
data = res.read()
print(data.decode("utf-8"))
```

