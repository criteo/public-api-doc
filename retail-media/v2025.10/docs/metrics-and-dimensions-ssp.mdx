In this page you will find all the metrics and dimensions that are currently support with the [Revenue Report](/retail-media/v2025.10/docs/revenue-report-ssp). Please note that many of the metrics and dimensions listed here are currently supported only through the API.

### Dimensions

  <thead>

  </thead>

      <td><code>date</code></td>
      <td>The date of when reported activity took place occurred</td>
    </tr>
    <tr>
      <td><code>hour</code></td>
      <td>The hour of the day when rendered ad received an event</td>
    </tr>
    <tr>
      <td><code>parentAccount</code></td>
      <td>The associated account that is connected to the supply or demand account</td>
    </tr>
    <tr>
      <td><code>accountId</code></td>
      <td>The computed and unique identifier of the supply or demand account</td>
    </tr>
    <tr>
      <td><code>accountName</code></td>
      <td>The supply or demand account name</td>
    </tr>
    <tr>
      <td><code>accountTypeName</code></td>
      <td>The account type of the reported activity. Options will include:<br /><ul><li><code>Demand</code> - for revenue generated from an indirect sold or private market demand account.</li><li><code>Supply</code> - revenue was generated from direct sold campaigns</li></ul></td>
    </tr>
    <tr>
      <td><code>advertiserType</code></td>
      <td>The advertiser account type where the ad activity originated from. Options includes:<br /><ul><li><code>retailer</code> - ad activity sold directly in the supply account</li><li><code>brand</code> - ad activity sold to a demand brand account in the Criteo network or through private market</li><li><code>seller</code> - ad activity sold to a demand seller account in the Criteo network or through private market</li></ul></td>
    </tr>
    <tr>
      <td><code>campaignId</code></td>
      <td>The computed unique campaign identifier of the demand or supply account that generated the activity</td>
    </tr>
    <tr>
      <td><code>campaignName</code></td>
      <td>The campaign name of the demand or supply account that generated the activity</td>
    </tr>
    <tr>
      <td><code>campaignTypeName</code></td>
      <td>The campaign type available. Available options are <code>Open Auction</code> or <code>Preferred Deals</code></td>
    </tr>
    <tr>
      <td><code>campaignStartDate</code></td>
      <td>The start date set for the campaign</td>
    </tr>
    <tr>
      <td><code>campaignEndDate</code></td>
      <td>The end date set for campaign</td>
    </tr>
    <tr>
      <td><code>lineItemId</code></td>
      <td>The computed unique identifier of a line-item</td>
    </tr>
    <tr>
      <td><code>lineItemName</code></td>
      <td>The line-item name provided by the advertiser</td>
    </tr>
    <tr>
      <td><code>lineItemStartDate</code></td>
      <td>The start date set for the line-item</td>
    </tr>
    <tr>
      <td><code>lineItemEndDate</code></td>
      <td>The end date set for the line-item</td>
    </tr>
    <tr>
      <td><code>lineItemStatus</code></td>
      <td>The status of the line-item associated with the selected dimension, such as the date or hour indicating of when the event took place. Check out <a href="/retail-media/v2025.10/docs/campaign-lineitem-status">Campaign &amp; Line Item Status</a> for more details on available values and their behaviors.</td>
    </tr>
    <tr>
      <td><code>retailerId</code></td>
      <td>The retailer ID that is associated with the report activities . This will be the same <code>id</code> provided in the API call</td>
    </tr>
    <tr>
      <td><code>retailerName</code></td>
      <td>The retailer name that is associated to the report activity.</td>
    </tr>
    <tr>
      <td><code>brandId</code></td>
      <td>The brand ID provided by the retailer brand ID associated with the advertised SKU</td>
    </tr>
    <tr>
      <td><code>brandName</code></td>
      <td>The retailer brand name associated with the advertised SKU</td>
    </tr>
    <tr>
      <td><code>placementId</code></td>
      <td>The id of the placement where the ad creative was displayed.</td>
    </tr>
    <tr>
      <td><code>placementName</code></td>
      <td>The name of the placement where the ad creative was displayed.</td>
    </tr>
    <tr>
      <td><code>pageTypeName</code></td>
      <td>The page type where the ad was rendered</td>
    </tr>
    <tr>
      <td><code>environment</code></td>
      <td>The environment where the ad was rendered on. Options will include web, mobile and app</td>
    </tr>
    <tr>
      <td><code>soldBy</code></td>
      <td>The source of where activities originated from. Options will include <code>Direct Sold</code>, <code>Indirect Sold</code> or <code>Private Market</code></td>
    </tr>
    <tr>
      <td><code>buyType</code></td>
      <td>The campaign buying strategy. Options will include <code>auction</code>, <code>preferredDeals</code>, or <code>sponsorship</code></td>
    </tr>
    <tr>
      <td><code>salesChannel</code></td>
      <td>Sales channel the attributed purchase was made through.<br />Available options are:<br /><code>online</code> - for sales attributed to an ad that was served and purchased online or in-app<br /><code>offline</code> - for sales to an ad that was served online but purchase in store</td>
    </tr>
    <tr>
      <td><code>attributionSettings</code></td>
      <td>The lookback click and view attribution window of the requested report. The attribution window is based on the parameters selected in the report API request. The <code>attributedSettings</code>field will be based on the lookback windows provided in the API call.<br /><strong>Post-Click (C) lookback window</strong><br />none, 7, 14, 30 days<br /><strong>Post-view (V) lookback window</strong><br />none, 1, 7, 14, 30 days</td>
    </tr>
    <tr>
      <td><code>activityType</code></td>
      <td>The type of ad engagement the sale was attributed to. Available options will include <code>imp</code> for impressions or <code>click</code> for clicks</td>
    </tr>
    <tr>
      <td><code>keyword</code></td>
      <td>Keyword or phrase used to land on the search page where an ad creative rendered.</td>
    </tr>
    <tr>
      <td><code>skuRelation</code></td>
      <td>Attribution rule used to match the impression or click to a sale. Available options are: <code>Same SKU</code>, <code>Same Parent SKU</code>, <code>Same Category</code>, <code>Same Brand</code> or <code>Same Seller</code></td>
    </tr>
    <tr>
      <td><code>advProductId</code></td>
      <td>Advertised product ID; this references the same product ID of the retailer Catalogs</td>
    </tr>
    <tr>
      <td><code>advProductName</code></td>
      <td>Advertised product name</td>
    </tr>
    <tr>
      <td><code>advProductGtin</code></td>
      <td>Advertised product GTIN</td>
    </tr>
    <tr>
      <td><code>advProductMpn</code></td>
      <td>Advertised product MPN</td>
    </tr>
    <tr>
      <td><code>pageCategory</code></td>
      <td>The retailer defined category of the page where ads are rendered</td>
    </tr>
    <tr>
      <td><code>retailerCategoryId</code></td>
      <td>The retailer category provides the type of product the SKU belongs to. This fields provides the id of that product type</td>
    </tr>
    <tr>
      <td><code>retailerCategoryName</code></td>
      <td>The name of that product type</td>
    </tr>
    <tr>
      <td><code>taxonomyBreadcrumb</code></td>
      <td>The category breadcrumb of the product using retailer product taxonomies (L1, L2 L3 etc.). This is for analysis where SKUs are present in the displays</td>
    </tr>
    <tr>
      <td><code>taxonomy1Id </code><br /><code>taxonomy2Id</code><br /><code>taxonomy3Id</code><br /><code>taxonomy4Id</code><br /><code>taxonomy5Id</code><br /><code>taxonomy6Id</code><br /><code>taxonomy7Id</code></td>
      <td>The ID SKU's taxonomy. The revenue report provides the option to select level 1 through level 7.<br />ℹ️ <strong>Note</strong>: SKUs will normally have at least the primary sku taxonomy (L1). All following levels are considered additional taxonomies. If a SKU does not have an additional taxonomy level, the report will default to <code>unknown</code></td>
    </tr>
    <tr>
      <td><code>taxonomy1Name</code><br /><code>taxonomy2Name</code><br /><code>taxonomy3Name</code><br /><code>taxonomy4Name</code><br /><code>taxonomy5Name</code><br /><code>taxonomy6Name</code><br /><code>taxonomy7Name</code></td>
      <td>The name SKU's taxonomy. If a SKU does not have an additional taxonomy level, the report will default to <code>unknown</code></td>
    </tr>
  </tbody>
</table>

### Metrics

<table>
  <thead>
    <tr>
      <th>Dimensions</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>numberOfCampaigns</code></td>
      <td>the total count of campaigns associated with the selected report dimension.</td>
    </tr>
    <tr>
      <td><code>numberOfLineItems</code></td>
      <td>The total count of line-items associated with the selected report dimension.</td>
    </tr>
    <tr>
      <td><code>numberOfSkus</code></td>
      <td>The total count of skus associated with the selected report dimension.</td>
    </tr>
    <tr>
      <td><code>skuUnitPrice</code></td>
      <td>The total price of each SKU unit.<br /><ul><li>&#95;Note&#95;&#42;: It is recommended to utilize metrics such as <code>advProductId</code>, <code>advProductGtin</code>, and <code>advProductMpn</code> to access individual SKU prices. However, when considering other dimensions at the line-item, campaign, or account level, the sum of all SKU prices will be calculated.</li></ul></td>
    </tr>
    <tr>
      <td><code>pageViews</code></td>
      <td>The count of unique ad calls on a retailer network. Use this metric when referencing to dimensions that relates to itself to receive a more comprehensive data. Using dimensions such as <code>pageTypeName</code>, <code>placementId</code>or <code>placementName</code>will provide you data to understand the performance of this metric.</td>
    </tr>
    <tr>
      <td><code>impressions</code></td>
      <td>An impression represents each time an ad renders on a page, regardless of clicks or views.<br />An impression for a Sponsored Product ad is when the sponsored SKU renders on the page (the product being the ad).<br />An impression for a Commerce Display ad is when the entire banner (creative + product) renders on the page.<br />An impression for a Display banner ad is when the creative banner renders on the page.</td>
    </tr>
    <tr>
      <td><code>productClicks</code></td>
      <td>Total count of the click events that occur when a user clicks a product in a placement</td>
    </tr>
    <tr>
      <td><code>placementClicks</code></td>
      <td>Total count of the click events that occur when a user clicks a placement</td>
    </tr>
    <tr>
      <td><code>clicks</code></td>
      <td>The total count of all click events<br /><ul><li>&#95;Formula&#95;&#42;: <code>placementClicks</code> + <code>productClicks</code></li></ul></td>
    </tr>
    <tr>
      <td><code>sales</code></td>
      <td>The total attributed revenue from product sales</td>
    </tr>
    <tr>
      <td><code>units</code></td>
      <td>The total attributed product units sold</td>
    </tr>
    <tr>
      <td><code>assitedSales</code></td>
      <td>Sales revenue attributed to a click or impression of all the ads that appeared on a page within the chosen attribution window (specified at the campaign level) but were not determined to be the <code>attributedSales </code>(the last ad before the customer purchased).</td>
    </tr>
    <tr>
      <td><code>assistedUnits</code></td>
      <td>The number of events that occurred that helped generated a sale. Assisted units are excluded from <code>attributedUnits</code>. This means its not counted in the attributed units calculation</td>
    </tr>
    <tr>
      <td><code>openAuctionRevenue</code></td>
      <td>The total amount of <strong>open auction revenue</strong> generated on the set date</td>
    </tr>
    <tr>
      <td><code>preferredDealsRevenue</code></td>
      <td>The total amount of <strong>preferred deals revenue</strong> generated on the set date</td>
    </tr>
    <tr>
      <td><code>revenue</code></td>
      <td>The total revenue generated on the set date. The <code>revenue</code> metric take the total revenue generated across preferred deals or auction campaigns for that period and is computed the same way as the <code>openAuctionRevenue</code> and <code>preferredDealsRevenue</code>.</td>
    </tr>
    <tr>
      <td><code>transactions</code></td>
      <td>The total amount of attributed orders/transactions</td>
    </tr>
    <tr>
      <td><code>ctr</code></td>
      <td>The percentage of shoppers who clicked an ad rendered on a page.<br />Product and Product Category Report Types<br /><ul><li>&#95;Formula&#95;&#42;: <code>clicks</code> / <code>productImpressions</code></li></ul><br />All other report types<br /><ul><li>&#95;Formula&#95;&#42;: <code>clicks</code> / <code>placementImpressions</code></li><li>&#95;Note&#95;&#42;: there are two types of impressions that can be measured to know the success of your ads: <code>placementImpressions</code> and <code>productImpressions</code>. Preferred deals have placement impressions, while open auction uses product impressions. For more info: <a href="https://help.retailmedia.criteo.com/kb/guide/en/product-placement-impressions-Yg9vyLMSMS/Steps/2492962">Product &amp; Placement Impression</a> Help Center article</li></ul></td>
    </tr>
    <tr>
      <td><code>cr</code></td>
      <td>The number of conversion generated by the visitors to the retailer site.<br /><ul><li>&#95;Formula&#95;&#42;: number of conversions / <code>clicks</code> &#42; 100</li></ul></td>
    </tr>
    <tr>
      <td><code>cpc</code></td>
      <td>Average cost-per-click calculated by dividing openAuctionRevenue / clicks or preferredDealsRevenue / clicks</td>
    </tr>
    <tr>
      <td><code>cpm</code></td>
      <td>Average cost per mille calculated by diving <code>preferredDealsRevenue</code> / <code>impressions</code> &#42; 1000</td>
    </tr>
    <tr>
      <td><code>roas</code></td>
      <td>Return-on-ad-spend (ROAS), calculated by dividing <code>sales</code> / <code>openAuctionRevenue</code> or <code>sales</code> / <code>preferredDealsRevenue</code></td>
    </tr>
    <tr>
      <td><code>workingMedia</code></td>
      <td>The total spend paid by the brand</td>
    </tr>
    <tr>
      <td><code>netRevenue</code></td>
      <td>The total revenue from brand spend for the retailer</td>
    </tr>
    <tr>
      <td><code>uniqueVisitors</code></td>
      <td>The number of distinct shoppers exposed to an ad within the reporting period. Each shopper is counted once using the retailer cookie ID.<br />Criteo uses retailer cookie IDs to identify users across sessions on the same device. Each ID can last up to a year, provided the user does not clear their cookies.</td>
    </tr>
    <tr>
      <td><code>frequency</code></td>
      <td>An average representing how often an impression has been shown to the same user</td>
    </tr>
  </tbody>
</table>
