<Info>
  Version **`2026.07`** of the Criteo Retail Media API is live as of **July 21st, 2026** and will be supported until **[date TBD]**. A new Postman collection for `2026.07` is also available in the [Criteo Postman workspace](https://www.postman.com/realcriteo/workspace/criteo/overview).

***

This version introduces new capabilities to the Criteo Retail Media API and changes to existing endpoints. **Only endpoints with changes are listed here.**

With the introduction of this new version, the base URL changes to:

```text URL
https://api.criteo.com/2026-07/retail-media/{endpoint}
```

***

# New in Stable 2026.07

## Retailer Search

The Retailer Search endpoint has been updated with the following changes:

<Warning>
  **Breaking change:** The `isAvailable` field has been removed from Retailer Search responses. Use `budgetModelAvailabilities` to determine targeting eligibility by budget model.
</Warning>

- `isAvailable` has been removed from responses.
- `budgetModelAvailabilities` now returns an array of objects indicating availability per budget model, with their valid `pageType` and environment combinations.
- `aiAssistant` has been added as an accepted value for the `pageTypes` field.

<Info>
  See the full documentation for the [Retailer Search endpoint](/retail-media/docs/retailer-search).
</Info>

***

## SSP Revenue Report

The SSP Revenue Report now includes three additional fields:

- `authorizedBuyer` — added as a new value in the `soldBy` enum
- `budgetModels` — new array field indicating applicable budget models (`CriteoBudget`, `RetailerBudget`)
- `activationPlatforms` — new array field indicating applicable activation platforms (`CommerceMax`, `PrivateMarket`)

<Info>
  See the full documentation for the [Revenue Report (SSP)](/retail-media/docs/revenue-report-ssp).
</Info>

***

## Real-Time Performance API — Metric Renaming

<Warning>
  The following metrics have been renamed. The old names are deprecated as of this version:

  | Old name | New name |
  |---|---|
  | `billableImpressions` | `impressions` |
  | `billableClicks` | `clicks` |

  Update your integrations to use the new metric names.
</Warning>

# New in Experimental

## Balance Management — PATCH Endpoint

A new PATCH endpoint is now available in the Experimental version, enabling partial updates to existing balances. This release also includes the following field-level changes across all balance endpoints:

- `retailerPoNumber` replaces `poNumber` in all balance request and response payloads.
- `endDate` now accepts a Nillable wrapper object to support three-state behavior:
  - `{ "value": "YYYY-MM-DD" }` — sets the end date
  - `{ "value": null }` — actively clears the end date
  - Omitting the field — leaves the current value unchanged

<Warning>
  Sending `{ "value": null }` for `endDate` is a destructive operation that actively clears the date. It is distinct from omitting the field.
</Warning>

<Info>
  See the full documentation for [Balance Management](/retail-media/docs/balance-management).
</Info>
