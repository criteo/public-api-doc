**Release date: 21 July 2026**

This release promotes several features from preview to stable, introduces experimental endpoints, and expands existing APIs with new fields and capabilities.

---

## Stable

### CMax Retailer Budgets Expansion

<Note>This feature was already available in 2026.01 and carries forward to 2026.07 stable.</Note>

Expands the Retailer Budgets API to support the full CMax budget model. Changes across six endpoints:

- **Balances** ÔÇö retailer budget visibility, updated field names, new request/response samples
- **Campaigns** ÔÇö retailer ID validation, updated field requirements
- **Onsite Sponsored Products Line Items** ÔÇö `targetRetailerId` enforcement, bid strategy updates
- **Retailer Search** ÔÇö `budgetModelAvailabilities` field added, `isAvailable` removed
- **Retailer Budgets** ÔÇö feature overview updated
- **Account-Level Report** ÔÇö `budgetModels` and `activationPlatforms` filter fields added

**Affected pages:** [Balances](https://developers.criteo.com/retail-media/docs/balances-endpoints) ┬À [Campaigns](https://developers.criteo.com/retail-media/docs/campaigns-endpoints) ┬À [Line Items](https://developers.criteo.com/retail-media/docs/onsite-sponsored-products-line-items) ┬À [Retailer Search](https://developers.criteo.com/retail-media/docs/retailer-search) ┬À [Retailer Budgets](https://developers.criteo.com/retail-media/docs/retailer-budgets) ┬À [Account-Level Report](https://developers.criteo.com/retail-media/docs/account-level-report)

The Supply-Side Revenue Report now supports three additional filter fields:

| Field | Description |
|-------|-------------|
| `authorizedBuyer` | Filter revenue by authorized buyer. |
| `budgetModels` | Filter by budget model type. |

**Affected page:** [Revenue Report (SSP)](https://developers.criteo.com/retail-media/docs/revenue-report-ssp)

## Experimental

### PATCH Endpoint for Balance Management

A new `PATCH` endpoint for balance management is available in the experimental channel. This release includes:

- `retailerPoNumber` replaces `poNumber` across all request and response samples
- `endDate` updated to a Nillable wrapper object (`{ "value": "YYYY-MM-DD" | null }`) supporting three-state behavior:
  - Omitting the field leaves the existing end date unchanged
  - `{ "value": "YYYY-MM-DD" }` sets a new end date
  - `{ "value": null }` explicitly clears the end date

<Warning>
Sending `{ "value": null }` actively clears the end date. This is distinct from omitting the field entirely.
</Warning>

**Affected page:** [Balance Management](/retail-media/v2026-preview/docs/balance-management) *(experimental link ÔÇö stable URL coming soon)*

## Preview

### Reporting API Redesign

<Note>
This redesign is available in the **2026 preview** channel only and is not yet part of the 2026.07 stable or experimental release. The legacy endpoints (`/reports/campaigns`, `/reports/line-items`, `/reports/accounts`) remain available in all channels while the migration is in progress.
</Note>

Three dedicated report endpoints replace the previous multi-purpose pattern:

| New endpoint | Replaces |
|---|---|
| `POST /reports/performance` | `/reports/campaigns`, `/reports/line-items`, `/reports/accounts` with `reportType` |
| `POST /reports/missed-opportunities` | `reportType: capout` ÔÇö metrics renamed `capoutMissed*` ÔåÆ `missed*` |
| `POST /reports/attributed-transactions` | `reportType: attributedTransactions` |

**Affected pages:** [Performance Report](/retail-media/v2026-preview/docs/performance-report) ┬À [Missed Opportunities](/retail-media/v2026-preview/docs/missed-opportunities-report) ┬À [Attributed Transactions](/retail-media/v2026-preview/docs/attributed-transactions-report)
