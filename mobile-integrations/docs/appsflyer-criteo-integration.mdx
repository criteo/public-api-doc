## Overview

This page will guide you in setting up your Criteo App campaign(s) with AppsFlyer, either App Install or App Retargeting campaigns.

---

## Dashboard Configuration

### App Settings

Under **Settings > App Settings**, please follow these steps:

1. Enable **View-through attribution via probabilistic modeling**
2. Enable **Re-engagement attribution**
3. Leave default option **None** for Minimum time between re-engagement conversions
4. Hit **Save settings**

<Warning>
  If Enabling retargeting attribution is left disabled, Criteo won't have any in-app conversion/sales attributed to our Retargeting campaigns.
</Warning>

### Partner Configuration

To proceed with Criteo-specific configurations:

1. Under **Collaborate > Partner Marketplace**, search for Criteo

   <img src="/images/mobile-integrations/appsflyer/marketplace.png" alt="AppsFlyer Partner Marketplace" />

2. If not enabled, choose to **Set up integration**

3. Under the **Integration** tab, enable **Activate partner**

   <img src="/images/mobile-integrations/appsflyer/activate_partner.png" alt="Activate partner" />

4. Enable **Install view-through attribution**

   <img src="/images/mobile-integrations/appsflyer/vt-attributions2.png" alt="View-through attribution settings" />

   - Recommended: leave Re-engagement View-Through attribution **disabled** (to avoid discrepancies cross-platforms)

5. Select **All media sources, including organic** for Install.

6. To configure the Criteo postbacks, set **In-App Events Postbacks** to ON:

   <img src="/images/mobile-integrations/appsflyer/default-postbacks.png" alt="Default postbacks configuration" />

   - Set **In-App Events Postback Window** to **Lifetime**, so that Criteo can receive postbacks from all active app users
   - Perform the mapping of all AppsFlyer events to **CriteoData**

   <img src="/images/mobile-integrations/appsflyer/events-criteodata.png" alt="Event mapping to CriteoData" />

   - As **Sending Option**, choose **All media sources, including organic**
   - As **Send revenue**, choose **Values & revenue** to forward all event parameters available
   - Save it.

<Note>
  See the section [Recommended Events per Vertical](#recommended-events-per-vertical) below for the list of recommended events per vertical.
</Note>

### Criteo Permissions

Under the **Permissions** tab, it is highly recommended to:

<img src="/images/mobile-integrations/appsflyer/4-permission-network-updated.png" alt="Permission settings" />

1. Enable **Ad Network Permissions** for Criteo to see your app in the dashboard
2. Grant, at least, the following permissions to the Criteo team:
   - Configure integration
   - Configure In-app event postbacks
   - Access retention report
   - Access aggregate conversion data
   - Access aggregate in-app events data
3. Please don't forget to **Save**.

<Note>
  Allowing Criteo to configure the integration and in-app event postbacks can increase the efficiency in the partner setup and allow quicker troubleshooting.
</Note>

## Tracking in AppsFlyer

### Tracking Generation

Under the **Attribution Link** tab, generate the Click Tracking Link and send it to your Criteo contact.

<img src="/images/mobile-integrations/appsflyer/attribution-link1.png" alt="Attribution link tab" />

Some typical settings:

1. Add **campaign name**

2. Enable **Retargeting settings** (if applicable)

   <img src="/images/mobile-integrations/appsflyer/attribution-link2.png" alt="Retargeting settings" />

   - Provide URI scheme deeplink
   - Leave **Ignore active users for retargeting** OFF (recommended)
   - Define **Re-engagement window** (recommended 30 days)

3. Define **Click-through attribution** (recommended 7 days)
4. Define **View-through attribution** (recommended 24 hours)

   <img src="/images/mobile-integrations/appsflyer/attribution-link3.png" alt="Attribution windows" />

5. Hit **Save Attribution link** and share the generated links with your Criteo contact.

## Additional Features

### Cost Data

Criteo has the ability to share Cost Data from our campaigns with AppsFlyer.

This feature is currently available only for accounts with **AppsFlyer ROI360**. The activation is straightforward:

1. Select the **Cost** tab under our integration module
2. If available, enable the checkbox **Get cost data**

   <img src="/images/mobile-integrations/appsflyer/cost-data.png" alt="Cost data configuration" />

3. Hit **Save Cost** to confirm the change

The data is synchronized **2x per day** and populates the cost of all your app campaigns in AppsFlyer dashboards, enabling cost-dependent metrics like Avg eCPI, ROI, etc.

For more info, see [AppsFlyer ROI360 cost aggregation overview](https://support.appsflyer.com/hc/en-us/articles/207040526-ROI360-cost-aggregation-overview).

### Audiences

Criteo is connected with AppsFlyer Audiences, which allows our platform to receive and work with your audiences seamlessly.

If this feature is available in your AppsFlyer plan and you already have Audiences built, follow the steps below:

1. Under **Engage > Audiences > Connections**, click **+ New Connection**

   <img src="/images/mobile-integrations/appsflyer/audience-new-connection.webp" alt="New audience connection" />

2. Search for **Criteo** in Partner name
3. Define an arbitrary **Connection name** of your choice
4. Click **Log in with Criteo** — you will be redirected to a Criteo page to review permissions:

   <img src="/images/mobile-integrations/appsflyer/audience-consent-dialog.webp" alt="Audience consent dialog" />

   - **Audience**: Manage access, to create/update audiences in your Criteo account
   - **Analytics**: Read access, to read campaign statistics
   - **Campaign**: Read access, to read campaign status
   - Under **Portfolio access**, select all entities to allow AppsFlyer to access all your advertiser accounts

5. Approve it, then select all **User Identifiers** available: IDFA, GAID, Email, etc.
6. **Save** to complete the connection.

For more info, see [AppsFlyer Audiences](https://support.appsflyer.com/hc/en-us/sections/6551336428945-Audiences).

### Agency-Managed Setups

For agency teams managing app campaigns on behalf of marketer clients, it is important to follow these instructions:

1. The integration module must be configured by an **Agency user**, not users connected to the app-owner team — even if the module is already pre-configured by the app owner, an agency user must repeat the module setup
2. Ad trackings must include the correct **Agency ID** (parameter `af_prt` — case-sensitive) on both click and impression trackings
   - If the integration module is correctly enabled by the agency, `af_prt` will be automatically included during tracking generation
3. Your **Agency ID must be added to Criteo's allowlist** before campaign launch — ask your Criteo contact to add your Agency ID if this is your first campaign with AppsFlyer

<Warning>
  Not following one of the points above may lead to incorrect attribution from AppsFlyer to Criteo campaigns, directly impacting the results measured on both platforms.
</Warning>

For more info, see [About agency accounts](https://support.appsflyer.com/hc/en-us/articles/207034106-About-agency-accounts).

### Product Catalog for Dynamic Campaigns

A catalog feed is an XML or CSV file containing product information (name, price, deep link, image link, etc.) that allows Criteo to dynamically generate product-recommendation banners.

<Tip>
  Criteo may already have the product catalog in some cases (e.g., advertisers already live on web campaigns).
</Tip>

Criteo recommends having both **Native Deeplinks** (URI Scheme links) and **Universal/App Links** whenever possible.
